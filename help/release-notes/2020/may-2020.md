# Adobe Experience Platform 發行說明

**發行日期：2020 年 5 月 13 日**

Adobe Experience Platform 現有功能更新：

- [使用者介面更新](#ux)

## 使用者介面更新 {#ux}

Adobe Experience Platform將發佈網域和標題列的更新，以改善您的使用體驗並與其他[!DNL Experience Cloud]應用程式統一。

- 更輕鬆地在組織之間切換或切換至不同的應用程式
- 改善使用者說明，包括說明功能表中的精選文章和內容相關檔案
- 能夠提供有關[!DNL Experience Platform]和檔案支援票證的反饋

新體驗的推出是漸進的。 您可在[https://experience.adobe.com/platform](https://experience.adobe.com/platform)檢視體驗。

## [!DNL Data Science Workspace] {#dsw}

[!DNL Data Science Workspace] 運用機器學習和人工智慧，從資料中釋放見解。[!DNL Data Science Workspace]已整合至Adobe Experience Platform，可協助您跨Adobe解決方案使用內容和資料資產進行預測。 [!DNL Data Science Workspace]的其中一個實現方式是使用[!DNL JupyterLab]。 [!DNL JupyterLab] 是網路使用者介面， [[!DNL Project Jupyter]](https://jupyter.org/) 與Adobe Experience Platform緊密整合。它為資料科學家提供互動式開發環境，以便與[!DNL Jupyter]筆記型電腦、程式碼和資料搭配使用。

**新功能**

| 功能 | 說明 |
|--- | ---|
| [!DNL JupyterLab Launcher] | [!DNL JupyterLab Launcher]現在包含[!DNL Spark] 2.4筆記型電腦的開機。 [!DNL Spark] 2.3筆記型電腦開機現在已標示為已過時，並設定在後續版本中移除。 |
| [!DNL Spark] 2.4 | 新的Scala([!DNL Spark])和PySpark配方現在使用[!DNL Spark] 2.4。 |
| 內核 | Scala([!DNL Spark])筆記本現在通過Scala內核編寫。 PySpark筆記本現在是透過[!DNL Python] Kernel製作的。 [!DNL Spark]和PySpark內核已過時，並設定為在後續版本中移除。 |
| 配方 | 新的PySpark和[!DNL Spark]配方現在會遵循類似[!DNL Python]和R配方的[!DNL Docker]工作流程。 |

有關遷移筆記型電腦和配方以使用[!DNL Spark] 2.4的詳細資訊，請參閱[筆記本資料存取指南](../../data-science-workspace/jupyterlab/access-notebook-data.md)。 有關Data Science Workspace的更多一般資訊，請參閱[概觀檔案](../../data-science-workspace/home.md)。

## [!DNL Destinations] {#destinations}

在[即時客戶資料平台](../../rtcdp/overview.md)中，目標是預先建立的與目標平台的整合，這些平台可順暢地向這些合作夥伴啟用資料。

**[!DNL Facebook]**

即時CDP現在支援[!DNL Facebook]的資料啟動，讓您啟用[!DNL Facebook]促銷活動的設定檔，以便根據雜湊的電子郵件鎖定受眾、個人化和抑制受眾。

有關新功能的詳細資訊，請參閱[Facebook目標](../../destinations/catalog/social/facebook.md)頁。

## [!DNL Experience Platform Web SDK] 和 [!DNL Experience Platform Edge Network] {#edge}

[!DNL Experience Platform Web SDK]和[!DNL Experience Platform Edge Network]可讓使用者針對一般使用者裝置和瀏覽器，即時將資料傳送至Adobe Experience Platform和其他Adobe解決方案。 您可在我們的[公用路線圖](https://github.com/adobe/alloy/projects/5)中找到最新的使用案例清單，該路線圖經常更新。

**新功能**

| 功能 | 說明 |
|--- | ---|
| 支援ECID | SDK支援ECID立即可用，而不需安裝任何其他程式庫或資訊。 |
| 設定UI | 使用[!DNL Launch]中的新邊緣設定UI管理您的設定ID設定，您必須位於允許清單中才能存取。 |
| Adobe Experience Platform網頁SDK架構欄位群組 | 用於[!DNL Experience Platform web SDK]的欄位組，該欄位組包含所有受支援的欄位。 |
| 課程許可控制 | 為公司提供[!DNL Experience Platform Web SDK]的加入和退出控制權。 |
| 新[!DNL Experience Cloud Debugger Extension]中的用戶端除錯支援 | 查看來自[!DNL Experience Platform web SDK]的請求以及邊緣跟蹤，以查看資料如何通過系統流動。 |
| Adobe Analytics | 透過邊緣設定，將資料傳送至[!DNL Analytics]報表套裝。 XDM會平面化為內容資料，支援多套裝標籤。 |
| Adobe Target | 支援Adobe Target。 包括VEC、表單式撰寫器、A/B、XT、Automated Personalization、MVT。 |
| Adobe Audience Manager支援 | 支援[!DNL Audience Manager] ID同步、URL目的地和Cookie目的地。 |
| 身分同步 | 將`setCustomersIds`重新命名為`syncIdentity`，以更清楚說明。 |
| XDM Object Builder | 在啟動擴充功能中，您現在可以將XDM物件建立為資料元素。 |

有關[!DNL Platform Web SDK]和[!DNL Edge Network]的詳細資訊，請參閱[文檔](../../edge/home.md)。

## [!DNL Real-time Customer Profile] {#profile}

Adobe Experience Platform可讓您為客戶推動協調、一致且相關的體驗，不論客戶在何處或何時與您的品牌互動。 透過[!DNL Real-time Customer Profile]，您可以全面瞭解每個客戶，並結合來自多個通道的資料，包括線上、離線、CRM和第三方資料。 [!DNL Profile] 可讓您將分散的客戶資料整合為統一的檢視，提供每個客戶互動的可操作、時間戳記帳戶。

**新功能**

| 功能 | 說明 |
| -----------| ---------- |
| 新的描述檔匯出量度 | 新增度量至描述檔匯出工作，顯示每個命名空間中匯出的描述檔總數和描述檔數。 |
| 新的可觀測性見解度量 | 可觀測性前瞻分析API現在有下列用於串流擷取至[!DNL Profile]的量度：傳入請求率、成功擷取率、已收錄記錄大小。 |
| 批量GET端點 | 已將大量GET端點新增至[!DNL Real-time Customer Profile] API，以便在單一API呼叫中擷取多個結果。 您現在最多可以批量GET100個ID，以用於區段定義、區段工作和合併原則。 |
| 依身分瀏覽描述檔 | 在[!DNL Platform] UI中，您現在可以選取識別名稱空間並提供識別值，以瀏覽描述檔。 |

**錯誤修正**

- None.

**已知問題**

- 沒有。

## 來源 {#sources}

Adobe Experience Platform可以從外部來源收集資料，同時允許您使用[!DNL Platform]服務來建構、標籤和增強該資料。 您可以從多種來源收集資料，例如Adobe應用程式、雲端儲存空間、協力廠商軟體和您的CRM系統。

[!DNL Experience Platform] 提供REST風格的API和互動式UI，讓您輕鬆地為各種資料提供者設定來源連線。這些源連接允許您驗證並連接到外部儲存系統和CRM服務、設定接收運行的時間，以及管理資料接收吞吐量。

**新功能**

| 功能 | 說明 |
| ------- | ----------- |
| 雲端儲存系統的其他API和UI支援 | [!DNL Azure File Storage]的新源連接器。 |
| 其他資料庫的API和UI支援 | [!DNL Azure Data Explorer]、IBM DB2和[!DNL Oracle DB]的新源連接器。 |
| Adobe Audience Manager到[!DNL Experience Platform]資料共用 | [!DNL Audience Manager]連接器的布建過程已更新。 [!DNL Audience Manager] 的資料集 [!DNL Real-time Customer Profile] 現在預設為禁用。您可以手動選擇要升級到[!DNL Profile]的資料集。 新的預設設定不具有可回溯性，而且僅影響新[!DNL Audience Manager]連接器的設定。 請參閱[資料集使用指南](../../catalog/datasets/user-guide.md)中的更多資訊。 |

**已知問題**

- 沒有。
