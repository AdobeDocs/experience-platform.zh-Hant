## 使用者介面更新 {#ux}

Adobe Experience Platform即將推出網域和標題列更新，除了提升您的使用體驗，也與其他 [!DNL Experience Cloud] 應用程式。

- 更輕鬆地在組織之間切換，或切換至不同的應用程式
- 改善使用者說明，包括「說明」功能表中的精選文章和內容相關檔案
- 提供意見回饋的功能 [!DNL Experience Platform] 和檔案支援票證

新體驗的推出是漸進式的。 您可以在下列位置檢視體驗： [https://experience.adobe.com/platform](https://experience.adobe.com/platform).

## [!DNL Data Science Workspace] {#dsw}

[!DNL Data Science Workspace] 使用機器學習和人工智慧從您的資料中釋放見解。 整合至Adobe Experience Platform， [!DNL Data Science Workspace] 可協助您在各種Adobe解決方案中使用您的內容和資料資產進行預測。 其中一種方式 [!DNL Data Science Workspace] 這是透過使用來完成的 [!DNL JupyterLab]. [!DNL JupyterLab] 是Web型使用者介面，用於 [[!DNL Project Jupyter]](https://jupyter.org/) 並緊密整合至Adobe Experience Platform。 它為資料科學家提供互動式開發環境 [!DNL Jupyter] Notebooks、程式碼和資料。

**新功能**

| 功能 | 說明 |
|--- | ---|
| [!DNL JupyterLab Launcher] | 此 [!DNL JupyterLab Launcher] 現在包含「 」的入門者 [!DNL Spark] 2.4筆記本。 [!DNL Spark] 2.3筆記型電腦啟動程式現在已標籤為已棄用，並設定在後續版本中移除。 |
| [!DNL Spark] 2.4 | 新比例([!DNL Spark])和PySpark配方現在使用 [!DNL Spark] 2.4. |
| 核心 | Scala ([!DNL Spark])現在可透過Scala核心撰寫筆記本。 PySpark筆記型電腦現在是透過以下方式撰寫： [!DNL Python] 核心。 此 [!DNL Spark] 和PySpark核心已過時，並設定在後續版本中移除。 |
| 配方 | 新增PySpark和 [!DNL Spark] 配方現在遵循 [!DNL Docker] 類似的工作流程 [!DNL Python] 和R配方。 |

如需移轉您的筆記型電腦與配方以使用的詳細資訊 [!DNL Spark] 2.4，請參閱 [notebook資料存取指南](../../data-science-workspace/jupyterlab/access-notebook-data.md). 如需資料科學工作區的詳細一般資訊，請參閱 [概述檔案](../../data-science-workspace/home.md).

## [!DNL Destinations] {#destinations}

在 [Real-time Customer Data Platform](../../rtcdp/overview.md)，目的地是預先建立的與目的地平台的整合，可透過順暢的方式為這些合作夥伴啟用資料。

**[!DNL Facebook]**

Real-Time CDP現在支援資料啟用 [!DNL Facebook]，可讓您為啟動設定檔 [!DNL Facebook] 根據雜湊電子郵件進行對象目標定位、個人化和抑制的行銷活動。

如需新功能的詳細資訊，請參閱 [facebook目的地](../../destinations/catalog/social/facebook.md) 頁面。

| 文件 | 說明 |

## [!DNL Experience Platform Web SDK] 和 [!DNL Experience Platform Edge Network] {#edge}

此 [!DNL Experience Platform Web SDK] 和 [!DNL Experience Platform Edge Network] 允許使用者即時傳送資料至一般使用者裝置和瀏覽器的Adobe Experience Platform及其他Adobe解決方案。 最新的使用案例清單可在我們的 [公開藍圖](https://github.com/adobe/alloy/projects/5) 經常更新。

**新功能**

| 功能 | 說明 |
|--- | ---|
| 支援ECID | SDK支援立即可用的ECID，且無需安裝任何其他程式庫或資訊。 |
| 設定UI | 在中使用新的邊緣組態UI管理您的組態ID設定 [!DNL Launch]，您必須在允許清單上才能存取。 |
| Adobe Experience Platform Web SDK結構描述欄位群組 | 要搭配使用的欄位群組 [!DNL Experience Platform web SDK] 其中包含所有支援的欄位。 |
| 課程同意控制 | 讓公司可控制選擇加入和選擇退出 [!DNL Experience Platform Web SDK]. |
| 新版本中的使用者端偵錯支援 [!DNL Experience Cloud Debugger Extension] | 檢視來自以下專案的請求： [!DNL Experience Platform web SDK] 以及邊緣追蹤，以瞭解資料如何流經系統。 |
| Adobe Analytics | 傳送資料至 [!DNL Analytics] 透過edge設定的報表套裝。 XDM會平面化成內容資料，支援多套裝標籤。 |
| Adobe Target | 支援Adobe Target。 包括VEC、表單式撰寫器、A/B、XT、Automated Personalization、MVT。 |
| Adobe Audience Manager支援 | 支援 [!DNL Audience Manager] ID同步、URL目的地和Cookie目的地。 |
| 身分同步 | 已重新命名 `setCustomersIds` 至 `syncIdentity` 以更清楚呈現。 |
| XDM物件產生器 | 在Launch擴充功能中，您現在可以將XDM物件建置為資料元素。 |

如需詳細資訊，請參閱 [!DNL Platform Web SDK] 和 [!DNL Edge Network]，請參閱 [檔案](../../edge/home.md).

## [!DNL Real-Time Customer Profile] {#profile}

Adobe Experience Platform可讓您為客戶推動協調、一致且相關的體驗，無論客戶在哪裡或何時與您的品牌互動。 替換為 [!DNL Real-Time Customer Profile]，您可以看到結合來自多個管道（包括線上、離線、CRM和第三方資料）之資料的每個個別客戶的整體檢視。 [!DNL Profile] 可讓您將不同的客戶資料合併成統一的檢視，針對每個客戶互動提供可採取行動且附有時間戳記的說明。

**新功能**

| 功能 | 說明 |
| -----------| ---------- |
| 新的設定檔匯出量度 | 新增量度至設定檔匯出作業，顯示匯出的設定檔總數和每個名稱空間中的設定檔數量。 |
| 新的可觀察性深入分析量度 | 「可觀察性深入分析API」現在提供下列量度，可將擷取串流至 [!DNL Profile]：傳入請求率、成功擷取率、擷取記錄大小。 |
| 大量GET端點 | 已將大量GET端點新增至 [!DNL Real-Time Customer Profile] API可在單一API呼叫中擷取多個結果。 您現在最多可以批次GET區段定義、區段作業和合併原則的100個ID。 |
| 依身分瀏覽設定檔 | 在 [!DNL Platform] UI您現在可以選取身分名稱空間並提供身分值，以瀏覽設定檔。 |

## 來源 {#sources}

Adobe Experience Platform可從外部來源內嵌資料，同時允許您使用建構、加標籤及增強該資料 [!DNL Platform] 服務。 您可以內嵌來自各種來源的資料，例如Adobe應用程式、雲端儲存、協力廠商軟體和您的CRM系統。

[!DNL Experience Platform] 提供RESTful API和互動式UI，讓您輕鬆設定各種資料提供者的來源連線。 這些來源連線可讓您驗證並連線至外部儲存系統和CRM服務、設定擷取執行的時間，以及管理資料擷取輸送量。

**新功能**

| 功能 | 說明 |
| ------- | ----------- |
| 雲端儲存系統的額外API和UI支援 | 新的來源聯結器 [!DNL Azure File Storage]. |
| 資料庫其他API和UI支援 | 新的來源聯結器 [!DNL Azure Data Explorer]、 IBM DB2和 [!DNL Oracle DB]. |
| Adobe Audience Manager至 [!DNL Experience Platform] 資料共用 | 的布建程式 [!DNL Audience Manager] 聯結器已更新。 [!DNL Audience Manager] 的資料集 [!DNL Real-Time Customer Profile] 現在預設為停用。 您可以手動選擇要提升至哪些資料集 [!DNL Profile]. 新的預設設定不具可回溯性，只會影響新設定的布建 [!DNL Audience Manager] 聯結器。 如需詳細資訊，請參閱 [資料集使用手冊](../../catalog/datasets/user-guide.md). |

**已知問題**

- None.
