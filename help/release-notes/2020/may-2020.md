## 使用者介面更新 {#ux}

Adobe Experience Platform將發佈域和標題欄的更新，以改進您的體驗，並與其他 [!DNL Experience Cloud] 應用程式。

- 更輕鬆地在組織之間切換或切換到其他應用程式
- 改進的用戶幫助，包括「幫助」菜單中的特色文章和與上下文相關的文檔
- 提供關於 [!DNL Experience Platform] 和檔案支援票證

新經驗的推廣是漸進的。 您可以在 [https://experience.adobe.com/platform](https://experience.adobe.com/platform)。

## [!DNL Data Science Workspace] {#dsw}

[!DNL Data Science Workspace] 利用機器學習和人工智慧從資料中釋放洞見。 融入Adobe Experience Platform, [!DNL Data Science Workspace] 幫助您跨Adobe解決方案使用內容和資料資產進行預測。 方法之一 [!DNL Data Science Workspace] 通過使用 [!DNL JupyterLab]。 [!DNL JupyterLab] 是基於Web的用戶介面 [[!DNL Project Jupyter]](https://jupyter.org/) 並且緊密融入Adobe Experience Platform。 它為資料科學家提供了一個互動式開發環境 [!DNL Jupyter] 筆記本、代碼和資料。

**新功能**

| 功能 | 說明 |
|--- | ---|
| [!DNL JupyterLab Launcher] | 的 [!DNL JupyterLab Launcher] 現在包括先頭 [!DNL Spark] 2.4筆記型電腦。 [!DNL Spark] 2.3筆記型電腦啟動器現在被標籤為不建議使用，並將在後續版本中刪除。 |
| [!DNL Spark] 2.4 | 新斯卡拉([!DNL Spark])和PySpark配方 [!DNL Spark] 2.4 |
| 核 | 斯卡拉([!DNL Spark])筆記本現在通過Scala內核編寫。 PySpark筆記本現在通過 [!DNL Python] 內核。 的 [!DNL Spark] 和PySpark內核已棄用，並設定為在後續版本中刪除。 |
| 食譜 | 新PySpark和 [!DNL Spark] 現在的食譜 [!DNL Docker] 類似於 [!DNL Python] 和R菜譜。 |

有關遷移筆記本和配方以使用的詳細資訊 [!DNL Spark] 2.4，請參閱 [筆記本資料存取指南](../../data-science-workspace/jupyterlab/access-notebook-data.md)。 有關資料科學工作區的更多一般資訊，請參見 [概述文檔](../../data-science-workspace/home.md)。

## [!DNL Real-Time Customer Profile] {#profile}

Adobe Experience Platform使您能夠為您的客戶提供協調、一致和相關的體驗，無論客戶在何處或何時與您的品牌進行交互。 與 [!DNL Real-Time Customer Profile]，您可以看到每個客戶的整體視圖，該視圖將來自多個渠道的資料組合在一起，包括線上、離線、CRM和第三方資料。 [!DNL Profile] 允許您將全異的客戶資料整合到一個統一視圖中，為每個客戶交互提供一個可操作且時間戳記的帳戶。

**新功能**

| 功能 | 說明 |
| -----------| ---------- |
| 新建配置檔案導出度量 | 已向配置檔案導出作業添加度量，其中顯示每個命名空間中導出的配置檔案總數和配置檔案數。 |
| 新的可觀性透視指標 | Oncebrity Insights API現在具有以下用於將接收流式傳輸到 [!DNL Profile]:傳入請求速率、成功攝取速率、攝取的記錄大小。 |
| 批量GET端點 | 已將批量GET終結點添加到 [!DNL Real-Time Customer Profile] API，用於在單個API調用中檢索多個結果。 現在，您最多可以批量GET段定義、段作業和合併策略的100個ID。 |
| 按標識瀏覽配置檔案 | 在 [!DNL Platform] UI現在可以選擇標識命名空間並提供標識值以瀏覽配置檔案。 |

## 來源 {#sources}

Adobe Experience Platform可以從外部源接收資料，同時允許您使用 [!DNL Platform] 服務。 您可以從多種源(如Adobe應用程式、基於雲的儲存、第三方軟體和CRM系統)中接收資料。

[!DNL Experience Platform] 提供了REST風格的API和互動式UI，使您可以輕鬆地為各種資料提供程式設定源連接。 通過這些源連接，您可以驗證並連接到外部儲存系統和CRM服務，設定接收運行時間，並管理資料接收吞吐量。

**新功能**

| 功能 | 說明 |
| ------- | ----------- |
| 雲儲存系統的其他API和UI支援 | 新的源連接器 [!DNL Azure File Storage]。 |
| 對資料庫的其他API和UI支援 | 新的源連接器 [!DNL Azure Data Explorer]、IBMDB2和 [!DNL Oracle DB]。 |
| Adobe Audience Manager [!DNL Experience Platform] 資料共用 | 的預配過程 [!DNL Audience Manager] 連接器已更新。 [!DNL Audience Manager] 資料集 [!DNL Real-Time Customer Profile] 現在預設禁用。 您可以手動選擇要升級到的資料集 [!DNL Profile]。 新預設設定不具有追溯性，只影響新設定 [!DNL Audience Manager] 連接器。 請參閱 [資料集使用手冊](../../catalog/datasets/user-guide.md)。 |

**已知問題**

- None.
