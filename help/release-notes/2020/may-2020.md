## 使用者介面更新 {#ux}

Adobe Experience Platform即將推出網域和標題列更新，除了提升您的使用體驗，也與其他 [!DNL Experience Cloud] 應用程式。

- 更輕鬆地在組織之間切換，或切換到不同的應用程式
- 改善使用者說明，包括「說明」功能表中的精選文章和內容相關檔案
- 提供意見回饋的功能 [!DNL Experience Platform] 和檔案支援票證

新體驗的推出是漸進的。 您可以在 [https://experience.adobe.com/platform](https://experience.adobe.com/platform).

## [!DNL Data Science Workspace] {#dsw}

[!DNL Data Science Workspace] 使用機器學習和人工智慧，釋放資料的深入見解。 整合至Adobe Experience Platform, [!DNL Data Science Workspace] 可協助您跨Adobe解決方案使用內容和資料資產進行預測。 其中一種方法 [!DNL Data Science Workspace] 這是通過使用 [!DNL JupyterLab]. [!DNL JupyterLab] 是 [[!DNL Project Jupyter]](https://jupyter.org/) 與Adobe Experience Platform緊密整合。 它為資料科學家提供互動式開發環境，以便與 [!DNL Jupyter] 筆記本、代碼和資料。

**新功能**

| 功能 | 說明 |
|--- | ---|
| [!DNL JupyterLab Launcher] | 此 [!DNL JupyterLab Launcher] 現在包括開胃菜 [!DNL Spark] 2.4筆記型電腦。 [!DNL Spark] 2.3筆記型電腦啟動程式現已標示為已過時，並在後續版本中設定為移除。 |
| [!DNL Spark] 2.4 | 新標籤([!DNL Spark])和PySpark菜譜現已使用 [!DNL Spark] 2.4。 |
| 內核 | 縮放([!DNL Spark])筆記本現在是通過Scala內核編寫的。 PySpark筆記本現在是透過 [!DNL Python] 內核。 此 [!DNL Spark] 和PySpark內核已過時，並設定在後續版本中刪除。 |
| 訣竅 | 新PySpark和 [!DNL Spark] 方式現在請遵循 [!DNL Docker] 類似於 [!DNL Python] 和R菜譜。 |

有關遷移筆記本和配方的詳細資訊，請使用 [!DNL Spark] 2.4，請參閱 [筆記型資料存取指南](../../data-science-workspace/jupyterlab/access-notebook-data.md). 如需Data Science Workspace的更一般資訊，請參閱 [概述檔案](../../data-science-workspace/home.md).

## [!DNL Destinations] {#destinations}

在 [Real-time Customer Data Platform](../../rtcdp/overview.md)，目的地是與目的地平台預先建立的整合，可順暢地向這些合作夥伴啟用資料。

**[!DNL Facebook]**

Real-Time CDP現在支援將資料啟動至 [!DNL Facebook]，可讓您為 [!DNL Facebook] 根據雜湊電子郵件鎖定目標、個人化和隱藏對象的促銷活動。

如需新功能的詳細資訊，請參閱 [Facebook目的地](../../destinations/catalog/social/facebook.md) 頁面。

## [!DNL Experience Platform Web SDK] 和 [!DNL Experience Platform Edge Network] {#edge}

此 [!DNL Experience Platform Web SDK] 和 [!DNL Experience Platform Edge Network] 可讓使用者針對一般使用者裝置和瀏覽器，即時將資料傳送至Adobe Experience Platform和其他Adobe解決方案。 若需最近的使用案例清單，請參閱 [公共路線圖](https://github.com/adobe/alloy/projects/5) 經常更新。

**新功能**

| 功能 | 說明 |
|--- | ---|
| 支援ECID | SDK支援立即可用的ECID，且不含任何其他程式庫或安裝資訊。 |
| 配置UI | 使用 [!DNL Launch]，您必須位於允許清單中才能存取。 |
| Adobe Experience Platform Web SDK結構欄位群組 | 要與 [!DNL Experience Platform web SDK] 包含所有受支援的欄位。 |
| 課程同意控制 | 為公司提供選擇加入和選擇退出的控制 [!DNL Experience Platform Web SDK]. |
| 新的用戶端除錯支援 [!DNL Experience Cloud Debugger Extension] | 請參閱 [!DNL Experience Platform web SDK] 以及邊緣追蹤，以查看資料如何透過系統流動。 |
| Adobe Analytics | 將資料傳送至 [!DNL Analytics] 透過edge設定來報表套裝。 XDM會平面化為內容資料，支援多套裝標籤。 |
| Adobe Target | 支援Adobe Target。 包括VEC、表單式撰寫器、A/B、XT、Automated Personalization、MVT。 |
| Adobe Audience Manager支援 | 支援 [!DNL Audience Manager] ID同步、URL目的地和Cookie目的地。 |
| 身分同步 | 已重新命名 `setCustomersIds` to `syncIdentity` 讓事情更加清楚。 |
| XDM物件產生器 | 在launch擴充功能中，您現在可以將XDM物件建置為資料元素。 |

如需 [!DNL Platform Web SDK] 和 [!DNL Edge Network]，請參閱 [檔案](../../edge/home.md).

## [!DNL Real-time Customer Profile] {#profile}

Adobe Experience Platform可讓您為客戶提供協調、一致且相關的體驗，無論客戶在何處或何時與您的品牌互動。 使用 [!DNL Real-time Customer Profile]，您可以全面了解將多個管道資料結合在一起的每個客戶，包括線上、離線、CRM和協力廠商資料。 [!DNL Profile] 可讓您將不同的客戶資料整合至統一的檢視中，提供每個客戶互動的可操作、時間戳記帳戶。

**新功能**

| 功能 | 說明 |
| -----------| ---------- |
| 新的設定檔匯出量度 | 新增度量至設定檔匯出工作，顯示每個命名空間中已匯出的設定檔總數和設定檔數。 |
| 新的可觀察性前瞻分析量度 | 可觀察性前瞻分析API現在提供下列量度，可將內容串流內嵌至 [!DNL Profile]:傳入的要求率、成功的擷取率、擷取的記錄大小。 |
| 大量GET端點 | 將大量GET端點新增至 [!DNL Real-time Customer Profile] API可在單一API呼叫中擷取多個結果。 您現在最多可以批量GET100個ID，以了解區段定義、區段作業和合併原則。 |
| 按身份瀏覽配置檔案 | 在 [!DNL Platform] UI您現在可以選取身分命名空間並提供身分值，以瀏覽設定檔。 |

## 來源 {#sources}

Adobe Experience Platform可以內嵌來自外部來源的資料，同時允許您使用 [!DNL Platform] 服務。 您可以從多種來源擷取資料，例如Adobe應用程式、雲端儲存、協力廠商軟體和您的CRM系統。

[!DNL Experience Platform] 提供RESTful API和互動式UI，讓您輕鬆為各種資料提供者設定來源連線。 這些源連接允許您驗證並連接到外部儲存系統和CRM服務、設定獲取運行時間以及管理資料獲取吞吐量。

**新功能**

| 功能 | 說明 |
| ------- | ----------- |
| 雲端儲存系統的其他API和UI支援 | 新源連接器 [!DNL Azure File Storage]. |
| 資料庫的其他API和UI支援 | 新源連接器 [!DNL Azure Data Explorer]、 IBM DB2和 [!DNL Oracle DB]. |
| Adobe Audience Manager [!DNL Experience Platform] 資料共用 | 的布建程式 [!DNL Audience Manager] 連接器已更新。 [!DNL Audience Manager] 資料集 [!DNL Real-time Customer Profile] 現在預設為停用。 您可以手動選擇要促銷的資料集 [!DNL Profile]. 新的預設設定不可回溯，且只會影響新的布建 [!DNL Audience Manager] 連接器。 如需詳細資訊，請參閱 [資料集使用手冊](../../catalog/datasets/user-guide.md). |

- 無。
