# Adobe Experience Platform 發行說明

**發行日期：2021 年 7 月 28 日**

Adobe Experience Platform 現有功能更新：

- [Data Science Workspace](#dsw)
- [資料流](#destinations)
- [目的地](#destinations)
- [體驗資料模型(XDM)](#xdm)
- [查詢服務](#query)
- [來源](#sources)

## Data Science Workspace {#dsw}

Data Science Workspace使用機器學習和人工智慧從資料中建立洞見。 整合到Adobe Experience Platform的Data Science Workspace可幫助您跨Adobe解決方案使用內容和資料資產進行預測。

**新功能**

| 功能 | 說明 |
| --- | --- |
| 庫和作業系統更新 | Data Science Workspace已對庫和作業系統進行了重大更新，以改進功能和可用性。 這包括JupyterLab 1.2.20、Python 3.7、Palots 1.2.4、Tensorflow 2.4.1和CUDA 11和CUDNN 8支援等。 要瞭解如何查看JupyterLab中的可用庫，請訪問 [支援的庫](../../data-science-workspace/jupyterlab/overview.md#supported-libraries) JupyterLab筆記本概述文檔中的一節。 |

有關Data Science Workspace的更一般資訊，請參閱 [資料科學工作區概述](../../data-science-workspace/home.md)。

## 資料流 {#dataflows}

在平台中，資料被從許多不同的源中攝取，在系統內進行分析，並被激活到各種目的地。 通過向資料流提供透明性，平台使跟蹤這種潛在的非線性資料流的過程變得更容易。

資料流是跨平台移動資料的作業的表示形式。 這些資料流是跨不同的服務配置的，可幫助將資料從源連接器移動到目標資料集，然後由Identity Service和Real-Time Customer Profile利用，最終激活到目標。

**新功能**

| 功能 | 說明 |
| ------- | ----------- |
| 目標儀表板 | 現在，您可以使用監視儀表板監視目標的資料流。 要瞭解更多資訊，請閱讀上的教程 [監視UI中的目標](../../dataflows/ui/monitor-destinations.md#monitoring-destinations-dashboard) |

有關資料流的詳細資訊，請參閱 [資料流概述](../../dataflows/home.md)。 要瞭解有關目標的詳細資訊，請參閱 [目標概述](../../destinations/home.md)。

## 目的地 {#destinations}

目的地是預先構建的與目標平台的整合，這些平台允許從Adobe Experience Platform無縫激活資料。 您可以使用目標來激活跨渠道市場營銷活動、電子郵件活動、目標廣告和許多其他使用案例的已知和未知資料。

**新功能**

| 功能 | 說明 |
| --- | --- |
| [更快的增量檔案導出](../../destinations/ui/activate-batch-profile-destinations.md#export-incremental-files) | 現在，您可以計畫每3、6、8和12小時為基於檔案的目標進行增量檔案導出。 當前不支援更改已保存段的檔案導出計畫。 要使用不同的計畫重新導出段，必須建立新的目標實例。 這是將在未來版本中解決的限制。 |
| [支援重複資料消除密鑰](../../destinations/ui/activate-batch-profile-destinations.md#deduplication-keys) | 選取去重複化索引鍵，以消除匯出檔案中同一設定檔的多筆記錄。可以選擇單個命名空間或最多兩個XDM架構屬性作為重複資料消除鍵。 |

## 體驗資料模型(XDM) {#xdm}

體驗資料模型(XDM)是一種開源規範，旨在提高數字型驗的威力。 它為模式形式的資料提供了通用結構和定義，使任何應用程式都能與平台服務通信。

| 功能 | 說明 |
| --- | --- |
| 電信業過濾器 | 將欄位組添加到UI中的架構時，您現在可以按電信行業進行篩選。 查看 [電信業實體關係圖](../../xdm/schema/industries/telecom.md) 查看電信使用案例的推薦資料模型。 |

有關平台中XDM的更多一般資訊，請參閱 [XDM系統概述](../../xdm/home.md)。

## 查詢服務 {#query}

查詢服務提供了使用標準SQL查詢Adobe Experience Platform資料的能力，支援各種分析和資料管理使用案例。 它是一個無伺服器工具，允許您連接Data Lake中的資料集，並將查詢結果捕獲為新資料集，以用於報告、Data Science Workspace或接收到即時客戶配置檔案。

您可以使用查詢服務構建資料分析生態系統，建立客戶跨各種交互渠道的圖片。 這些渠道可能包括銷售點、Web、移動或CRM系統。

**新功能**

| 功能 | 說明 |
| ------- | ----------- |
| 計畫查詢 | 現在，您可以使用查詢編輯器在平台中調度查詢。 要瞭解更多資訊，請閱讀 [查詢編輯器](../../query-service/ui/user-guide.md#scheduled-queries)。 |

有關詳細資訊，請參閱 [查詢服務文檔](../../query-service/home.md)。

## 來源 {#sources}

Adobe Experience Platform可以從外部源接收資料，同時允許您使用平台服務來構建、標籤和增強資料。 您可以從多種來源(如Adobe應用程式、基於雲的儲存、第三方軟體和您的CRM系統)接收資料。

Experience Platform提供REST風格的API和互動式UI，讓您能夠輕鬆地為各種資料提供程式設定源連接。 通過這些源連接，您可以驗證並連接到外部儲存系統和CRM服務，設定接收運行時間，並管理資料接收吞吐量。

| 功能 | 說明 |
| ------- | ----------- |
| Beta源移至GA | 已將以下源從beta升級為GA: <ul><li>[[!DNL Amazon Redshift]](../../sources/connectors/databases/redshift.md)</li><li>[[!DNL Azure Table Storage]](../../sources/connectors/databases/ats.md)</li><li>[[!DNL PayPal]](../../sources/connectors/payments/paypal.md)</li></ul> |
| [!DNL Salesforce Marketing Cloud] (Beta) | 現在可以連接 [!DNL Salesforce Marketing Cloud] Experience Platform [!DNL Flow Service] API或UI。 查看 [[!DNL Salesforce Marketing Cloud] 連接器概述](../../sources/connectors/marketing-automation/salesforce-marketing-cloud.md) 的子菜單。 |

要瞭解有關源的詳細資訊，請參閱 [源概述](../../sources/home.md)。
