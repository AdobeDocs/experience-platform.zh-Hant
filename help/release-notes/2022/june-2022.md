## [!DNL Data Science Workspace] {#dsw}

Data Science Workspace使用機器學習和人工智慧，釋放資料的深入分析。 整合至Adobe Experience Platform的Data Science Workspace可協助您跨Adobe解決方案，使用內容和資料資產進行預測。 Data Science Workspace完成此工作的其中一種方式是使用JupyterLab。 JupyterLab是適用於 <a href="https://jupyter.org/" target="_blank">朱佩特專案</a> 與Adobe Experience Platform緊密整合。 它為資料科學家提供互動式開發環境，以便與Jupyter筆記型電腦、程式碼和資料合作。

| 功能 | 說明 |
| --- | --- |
| JupyterLab啟動器 | JupyterLab啟動器現在包括Spark 3.2筆記型電腦的啟動器。 Spark 2.4筆記型電腦啟動器現在由Spark 3.2筆記型電腦取代，並將成為此版本的一部分。 |
| Spark 3.2 | 新的Scala(Spark)和PySpark配方現在使用Spark 3.2 |
| 內核 | Scala(Spark)筆記本現在通過Scala內核進行製作。 PySpark筆記本現在通過Python內核製作。 Spark和PySpark內核已過時，並將在後續版本中刪除。 |
| 訣竅 | 新的PySpark和Spark配方現在遵循Docker工作流，類似於Python和R配方。 |

{style=&quot;table-layout:auto&quot;}

如需Data Science Workspace的更一般資訊，請參閱 [概述檔案](../../data-science-workspace/home.md).

## [!DNL Destinations] {#destinations}

[!DNL Destinations] 預先建置與目的地平台的整合，可順暢地從Adobe Experience Platform啟動資料。 您可以使用目的地來針對跨通路行銷活動、電子郵件行銷活動、目標廣告和其他許多使用案例，啟用已知和未知的資料。

**新功能或更新功能**

| 功能 | 說明 |
| ----------- | ----------- |
| （測試版）Destination SDK支援 [[!DNL Google Cloud Storage]](../../destinations/destination-sdk/server-and-file-configuration.md#gcs-example) 檔案型目的地和 [可配置檔案名](../../destinations/destination-sdk/file-based-destination-configuration.md#file-name-configuration). | 您現在可以透過檔案名稱巨集，使用Destination SDK建立Google雲端儲存目的地，並定義匯出檔案的自訂檔案名稱。 <br><br> Adobe Experience Platform Destination SDK中的檔案式目的地支援目前仍在測試階段。 檔案和功能可能會有所變更。 |
| 資料流中的段列運行到批處理目標 | 對於要運行到批處理目標的資料流，UI現在會顯示與每個資料流運行關聯的段的名稱。 深入了解 [資料流運行到批處理目的地](/help/dataflows/ui/monitor-destinations.md#dataflow-runs-for-batch-destinations). |

## 資料收集 {#data-collection}

Adobe Experience Platform提供一套技術，可讓您收集用戶端客戶體驗資料，並傳送至Adobe Experience Platform Edge Network，以便在中加以擴充、轉換及分發至Adobe或非Adobe目的地。

**新功能**

| 功能 | 說明 |
| --- | --- |
| [資料流的存取類型設定](../../edge/datastreams/overview.md#create) | 建立新資料流時，您現在可以選取要邊緣網路接受的請求類型： <ul><li>**[!UICONTROL 混合驗證]**:選取此選項時，邊緣網路會接受已驗證和未驗證的請求。 當您打算使用Web SDK時，請選取此選項，或 [行動SDK](https://aep-sdks.gitbook.io/docs/)，連同 [伺服器API](../../server-api/overview.md). </li><li>**[!UICONTROL 僅驗證]**:選取此選項時，邊緣網路僅接受已驗證的請求。 當您打算只使用伺服器API，並想要防止任何未驗證的請求由 [!DNL Edge Network]. </li></ul> |
| [提出主張](../../edge/personalization/rendering-personalization-content.md#applypropositions) 單頁應用程式中，而不增加量度。 | 新增的 `applyPropositions` 命令允許您呈現或執行 [!DNL Target] 整合至單頁應用程式，而不會增加 [!DNL Analytics] 和 [!DNL Target] 量度。 這會提高報表正確性。 |
| [行動對網頁和跨網域ID共用](../../edge/identity/id-sharing.md) | Adobe Experience Platform Web SDK現在支援訪客ID共用功能，可讓您在行動應用程式、行動網頁內容以及各網域之間，提供更精確的個人化體驗。 |
| [Google資料層標籤擴充功能](../../tags/extensions/web/google-data-layer/overview.md) | Google資料層擴充功能可讓您在標籤實施中使用Google資料層。 |
| [Google Ads增強轉換事件轉送擴充功能](https://partners.adobe.com/exchangeprogram/experiencecloud/exchange.details.108630.html) | Google廣告增強轉換擴充功能可讓您即時增強Google廣告轉換。 |
| [Mailchimp事件轉送擴充功能](../../tags/extensions/web/mailchimp/overview.md) | Mailchimp事件轉送擴充功能會將事件傳送至Mailchimp行銷API，以觸發Mailchimp行銷活動、歷程或交易的電子郵件。 |

如需詳細資訊，請參閱 [資料匯集概述](../../collection/home.md).

## 查詢服務 {#query-service}

查詢服務可讓您使用標準SQL在Adobe Experience Platform中查詢資料 [!DNL Data Lake]. 您可以從 [!DNL Data Lake] 並將查詢結果擷取為新資料集，以用於報表、Data Science Workspace或擷取至「即時客戶個人檔案」。

**更新功能**

| 功能 | 說明 |
| --- | --- |
| 臨機結構標籤 | 將標籤套用至透過Query Service CTAS查詢自動產生的臨機結構的資料欄位，以管理對敏感資料的存取。 您可以限制臨機結構的特定欄位或資料集的使用，以控制對敏感個人資料和個人識別資訊的存取。 透過使用屬性型存取控制功能，您可以透過Platform UI標籤隨選結構欄位。 |
| `FLATTEN` 設定 | 透過協力廠商BI工具連線至資料庫時， `FLATTEN` 設定會將巢狀資料結構拼合成單獨的列，其中屬性名稱將成為保存行值的列名。 這可改善隨選結構的可用性，並減少在不支援巢狀資料結構的BI工具中擷取、分析、轉換和報告資料所需的工作量。 |

{style=&quot;table-layout:auto&quot;}

有關Query Services的詳細資訊，請參閱 [查詢服務概述](../../query-service/home.md).

## 來源 {#sources}

Adobe Experience Platform可內嵌來自外部來源的資料，同時允許您使用Platform服務來建構、加標籤及增強該資料。 您可以內嵌來自各種來源的資料，例如Adobe應用程式、雲端儲存、協力廠商軟體和您的CRM系統。

Experience Platform提供RESTful API和互動式UI，讓您輕鬆為各種資料提供者設定來源連線。 這些源連接允許您驗證並連接到外部儲存系統和CRM服務、設定獲取運行時間以及管理資料獲取吞吐量。

| 功能 | 說明 |
| --- | --- |
| 測試版 [!DNL Mixpanel] 來源 | 您現在可以使用 [!DNL Mixpanel] 來源：從 [!DNL Mixpanel] 帳戶Experience Platform。 請參閱 [[!DNL Mixpanel] 來源檔案](../../sources/connectors/analytics/mixpanel.md) 以取得更多資訊。 |

若要進一步了解來源，請參閱 [來源概觀](../../sources/home.md).
