## 儀表板 {#dashboards}

Adobe Experience Platform提供多個 [!DNL dashboards] 通過它可以查看有關組織資料的重要資訊，如每日快照時捕獲的。

### 帳戶設定檔控制面板

「帳戶設定檔」控制面板會顯示來自多個來源的統一帳戶資訊快照，這些來源涵蓋您的行銷管道，以及您的組織目前用來儲存客戶帳戶資訊的不同系統。

**更新功能**

| 功能 | 說明 |
| --- | --- |
| 按行業Widget列出的帳戶總數 | 此Widget會以單一量度顯示帳戶總數，並使用環圈圖來說明構成整體數量之產業的計數比例大小。 |
| 帳戶設定檔新增的介面工具集 | 此介面工具集使用色彩編碼的長條圖，來說明指定時段內新增至帳戶的設定檔數，以及構成這些新增設定檔的不同產業的比例。 |

{style=&quot;table-layout:auto&quot;}

請參閱 [Real-Time CDP, B2B版概述](../../rtcdp/b2b-overview.md) 若要進一步了解可用的B2B功能，或 [端對端教學課程](../../rtcdp/b2b-tutorial.md) 進一步了解如何在B2B工作流程中建立帳戶設定檔。

如需可用來視覺化帳戶設定檔相關量度的小工具的詳細資訊，請參閱 [帳戶設定檔小工具檔案](../../dashboards/guides/account-profiles.md#standard-widgets).

### 設定檔控制面板

「設定檔」控制面板會顯示您的組織在「設定檔存放區」內Experience Platform的屬性（記錄）資料快照。

**更新功能**

| 功能 | 說明 |
| --- | --- |
| 對應對象Widget | 此介面工具集顯示可啟動至從「設定檔」控制面板下拉式清單中選取之目的地的對應對象總數。 |

如需「設定檔」控制面板的詳細資訊，請參閱 [設定檔控制面板概觀](../../dashboards/guides/profiles.md).

### 目的地控制面板

「目標」控制面板會顯示貴組織已在Experience Platform中啟用的目標的快照。

**更新功能**

| 功能 | 說明 |
| --- | --- |
| 受眾介面工具集 | 此介面工具集會根據套用至設定檔資料的所選合併原則，提供可啟動的區段總數。 |

{style=&quot;table-layout:auto&quot;}

若要進一步了解「目的地」控制面板，請參閱 [目的地控制面板概觀](../../dashboards/guides/destinations.md).

## Experience Data Model(XDM) {#xdm}

XDM是開放原始碼規格，可針對匯入Adobe Experience Platform的資料提供通用結構和定義（結構）。 遵循XDM標準，所有客戶體驗資料皆可整合至通用表示法，以更快速、更整合的方式提供深入分析。 您可以從客戶動作中獲得寶貴的深入分析、透過區段定義客戶受眾，以及將客戶屬性用於個人化目的。

**新功能**

| 功能 | 說明 |
| --- | --- |

## Real-Time Customer Data Platform B2B 版本 {#b2b}

Real-Time CDP B2B Edition以Real-time Customer Data Platform(Real-Time CDP)為基礎，專為以企業對企業服務模式運作的行銷人員所打造。 它匯集了來自多個來源的資料，並將其結合為人員和帳戶設定檔的單一檢視。 此統一的資料可讓行銷人員精確鎖定特定對象，並參與所有可用管道中的這些對象。

**新功能**

| 功能 | 說明 |
| --- | --- |
| 導致帳戶匹配 | 帳戶比對的潛在客戶可讓您將已知的人員設定檔加入帳戶設定檔。 然後，您就可以在B2B內容（如帳戶或商機）中劃分和定位資料。 每日執行的工作會同時使用決定性和可能性因素，來比對尚未與任何帳戶相關聯的人員設定檔與最符合的帳戶。 然後，您可以決定是否在區段定義中包含這類符合項目。 <br><br>如需詳細資訊，請參閱 [導致帳戶匹配](../../rtcdp/b2b-ai-ml-services/lead-to-account-matching.md). 如需如何設定銷售機會至帳戶比對的指示，請參閱 [帳戶設定檔UI指南](../../rtcdp/account/../accounts/account-profile-ui-guide.md?lang=en#configure-lead-to-account-matching).</li> |
| 預測性銷售機會和帳戶計分 | 預測性銷售機會和帳戶分數使用基於樹的（隨機林/梯度提升）機器學習方法，該方法包括從機會階段轉換事件中學習並預測該事件，並將人員活動聚合到帳戶級別以生成帳戶分數。 匯總和單位層級也提供最具影響力的因素，協助B2B行銷人員更清楚了解促成分數的因素。 <br><br>如需詳細資訊，請參閱 [預測性銷售機會和帳戶計分](../../rtcdp/b2b-ai-ml-services/predictive-lead-and-account-scoring.md). 如需如何管理分數的詳細資訊，請參閱 [在Real-time Customer Data Platform B2B版中管理預測性潛在客戶和帳戶計分。](../../rtcdp/b2b-ai-ml-services/manage-predictive-lead-and-account-scoring.md) |

如需如何監控設定檔擴充的指南，請參閱 [在UI中監控設定檔擴充](../../dataflows/ui/b2b/monitor-profile-enrichment.md). 若要進一步了解Real-Time CDP B2B版，請參閱 [Real-Time CDP B2B概述](../../rtcdp/overview.md).

## 即時客戶個人檔案 {#profile}

Adobe Experience Platform可讓您為客戶提供協調、一致且相關的體驗，無論客戶在何處或何時與您的品牌互動。 透過即時客戶個人檔案，您可以全面了解各個客戶，其中結合來自多個管道的資料，包括線上、離線、CRM和第三方資料。 設定檔可讓您將客戶資料併入統一檢視中，提供每個客戶互動的可操作、時間戳記帳戶。

| 功能 | 說明 |
| ------- | ----------- |
| 孤立的設定檔邊緣屬性清除（有限發行） | 如果您的組織可以存取此功能，Profile Service現在會每天移除使用者活動區域剩餘的邊緣屬性，以更精確地呈現您在系統中的設定檔。 指定設定檔的所有設定檔片段都遭刪除後，就會進行此清理，這應該會影響從其中合併的資料集中的設定檔 `com_adobe_aep_profile_region_dataset` 標示為true。 這可能會在授權使用控制面板中顯示「可定址的受眾」量度的下降，也可能在設定檔控制面板中顯示「設定檔計數」量度的下降，因為這些量度包含此版本之前剩餘的邊緣屬性片段。 |

{style=&quot;table-layout:auto&quot;}

## 來源 {#sources}

Adobe Experience Platform可內嵌來自外部來源的資料，同時允許您使用Platform服務來建構、加標籤及增強該資料。 您可以內嵌來自各種來源的資料，例如Adobe應用程式、雲端儲存、協力廠商軟體和您的CRM系統。

Experience Platform提供RESTful API和互動式UI，讓您輕鬆為各種資料提供者設定來源連線。 這些源連接允許您驗證並連接到外部儲存系統和CRM服務、設定獲取運行時間以及管理資料獲取吞吐量。
