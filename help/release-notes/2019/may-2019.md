## [!DNL Data Ingestion] {#ingestion}

Adobe Experience Platform提供了豐富的功能集，可接收任何類型和任何延遲的資料。 您可以使用批處理或流式API，使用Adobe構建的連接器、資料整合合作夥伴或Adobe Experience PlatformUI進行接收。

有關產品文檔，請參見 [資料接收概述](../../ingestion/home.md)。

### 批次擷取

使用批處理接收API將任何類型的資料接收到Adobe Experience Platform。 批量接收API提供了一種安全且可擴展的機制來接收符合XDM或非XDM架構的資料。

| 功能 | 說明 |
| -----------| ---------- |
| 富格式支援 | 支援JSON、Parke、CSV的攝取 |
| 資料驗證 | 對資料執行三個驗證級別：架構、資料類型和約束 |
| 移動資料加密 | 在跨資料網路傳輸資料時保護資料並保護其安全。 |
| 多部分上載 | 支援256MB或更大檔案的大檔案上載 |
| 自動分區 | 資料自動分區以提高可擴充性和優化大型資料集的效能。 |

**已知問題**

None.

有關產品文檔，請參見 [批處理接收概述](../../ingestion/batch-ingestion/overview.md)。

### 連接器

Adobe Experience Platform連接器可幫助您輕鬆從多個源接收資料，使您能夠使用 [!DNL Experience Platform] 服務。 [!DNL Platform] 連接器允許您驗證到基於雲的儲存系統和CRM服務，設定接收運行時間，並管理資料接收吞吐量。

| 功能 | 說明 |
| -----------| ---------- |
| 基於雲的儲存連接器 | 從中接收資料 [!DNL Microsoft Azure Blob] 和 [!DNL Amazon S3] 雲儲存到Adobe Experience Platform |
| [!DNL Microsoft Dynamics CRM] &amp; [!DNL Salesforce CRM] | 發現中的所有可用資料 [!DNL Microsoft Dynamics] 或 [!DNL Salesforce] CRM。 從中接收任何對象、標準或自定義 [!DNL Microsoft Dynamics] 或 [!DNL Salesforce] CRM。 增量資料可以按小時或每天接收。 |  |

**已知問題**

* 基於雲的儲存連接器只能通過API調用使用。


有關產品文檔，請參見 [連接器概述](../../sources/home.md)。

### 串流擷取

流式接收將資料發送到 [!DNL Experience Platform] 從客戶端和伺服器端設備即時獲取。 可以對 [!DNL Real-Time Customer Profiles] 支援經驗證的資料收集和多記錄負載的亞秒延遲。

| 功能 | 說明 |
| -----------| ---------- |
| 同步驗證 | 允許立即發現JSON負載中的錯誤，從而提高開發人員速度。 |
| 端到端監控 | 監視從接收到消耗的流資料 [!DNL Data Lake] 和 [!DNL Real-Time Customer Profile]。 |
| 已驗證的資料收集 | 開啟或關閉選項以要求客戶端提供身份驗證令牌。 允許Adobe Experience Platform服務區分來自受信任源和不受信任源的記錄。 |
| 即時通過 [!DNL Launch] | 使用中的Adobe Experience Platform擴展 [!DNL Launch] 以便從客戶端設備即時發送XDM格式的資料。 |

None.

## [!DNL Experience Data Model] (XDM) {#xdm}

標準化和互操作性是其背後的關鍵概念 [!DNL Experience Platform]。 [!DNL Experience Data Model] (XDM)在Adobe的推動下，致力於標準化客戶體驗資料並定義客戶體驗管理模式。

XDM是一種公開記錄的規範，旨在提高數字型驗的威力。 它為任何與Adobe Experience Platform服務通信的應用程式提供了共同的結構和定義。 通過遵守XDM標準，所有客戶體驗資料都可以納入到一個通用的表示形式中，以更快、更整合的方式提供洞察力。 您可以從客戶操作中獲得有價值的見解，通過細分市場定義客戶受眾，並將客戶屬性用於個性化目的。

XDM是允許 [!DNL Experience Clou]d由Adobe Experience Platform提供動力，在正確的時刻，在正確的頻道向正確的人傳遞正確的資訊。

方法 [!DNL Experience Platform] 是建的， [!DNL XDM System] 操作 [!DNL Experience Data Model] 方案使用者 [!DNL Experience Platform] 元件。

| 功能 | 說明 |
| -----------| ---------- |
| [!DNL Schema Registry] | 的 [!DNL Schema Registry] 提供用戶介面([!DNL Schema Editor])和REST風格的API，從中可以查看和管理所有與架構相關的資源。 的 [!DNL Schema Editor] 和 [!DNL Schema Registry] API還可用於建立和管理每個組織唯一的新架構和資源。 |
| [!DNL Schema Library] | 的 [!DNL Schema Library] 包含全部 [!DNL Schema Registry] 資源，包括由單個組織定義的資源和按Adobe提供給所有用戶的資源， [!DNL Experience Platform] 與其應用程式整合的合作夥伴和供應商 [!DNL Experience Platform]。 |

**已知問題**

None.

瞭解有關使用XDM的更多資訊 [!DNL Schema Registry] API和 [!DNL Schema Editor]，請閱讀 [XDM系統文檔](../../xdm/home.md)。

## [!DNL Real-Time Customer Profile] {#profile}

[!DNL Profile] 是通用查找實體儲存，它合併各種企業資料資產中的資料，並提供以即時使用者配置檔案和相關時間系列事件（稱為聯合視圖）的形式訪問該資料。 統一視圖有助於通過各種渠道通知和授權操作， [!DNL Platform]和Adobe解決方案整合。 而且， [!DNL Profile] 用來推動機器學習和Sensei。 [!DNL Profile] API還可以直接用於豐富第三方解決方案、 CRM系統和專有解決方案的功能。

| 功能 | 說明 |
| -----------| ---------- |
| [!DNL Real-time customer profile] | [!DNL Profile] 通過資料輸入接收資料，將其與配置檔案儲存中的現有資料合併，並幾乎即時地建立或修改統一視圖。 這些資料會隨著您客戶的體驗而不斷變化，可用於提供個性化的建議。 |
| 聯合視圖 | C.C.C.C.C. [!DNL Profile] 是統一多通道資料的能力。 [!DNL Profile] 可以為您的消費者群中的個人提供合併視圖、跨資料集和連結標識（稱為union視圖）。 |
| Edge | 邊緣服務是低延遲資料收集、可插拔計算和跨所有可定址通道快速資料激活的框架。 邊緣服務可幫助即時個性化。 |

**已知問題**

None.

有關產品文檔，請參見 [配置檔案概述](../../profile/home.md)。

## 身份識別服務 {#identity}

Identity Service解決了客戶的不連接身份帶來的根本難題，這是由於客戶資料分散的特性在您組織的生態系統中的各個不同系統之間產生。 它通過跨設備和跨消費者使用的各種系統橋接身份來實現這一點。 這樣，Identity Service便於全面瞭解您的客戶及其行為，因此您可以即時提供有影響的數字型驗。

| 功能 | 說明 |
| -----------| ---------- |
| 識別圖 | 標識圖是標識之間關係的映射，該映射可近即時更新客戶活動。 [!DNL Identity Service] 管理僅由您的組織可見的標識圖，並基於您的資料構建，稱為專用圖。 [!DNL Identity Service] 當接收的資料記錄包含多個標識時，將擴展您的私有圖表，並添加這些標識之間的關係。 |

**已知問題**

None.

有關產品文檔，請參見 [Identity Service概述](../../identity-service/home.md)。

## [!DNL Segmentation Service] {#segmentation}

[!DNL Segmentation Service] 從配置檔案儲存中定義特定的配置檔案子集，描述區分配置檔案儲存中可銷售人員組的標準。 段可以基於記錄資料（如人口統計資訊）或代表客戶與您品牌接觸點的時間系列事件。

例如，在一封名為「你忘記買運動鞋了嗎？」的電子郵件中，你可能希望在過去30天內搜索跑鞋但沒有完成購買的所有用戶的受眾群。 或者，您可能希望將Web內容目標為僅向獎勵計畫中具有特定評級的用戶顯示。

| 功能 | 說明 |
| -----------| ---------- |
| 分段API | REST風格的服務使您能夠以寫程式方式訪問構建客戶細分市場。 |
| 區段產生器 | 段構建器是一個強大且直觀的工作區，可從您的 [!DNL Profile] 資料。 |

**已知問題**

None.

有關產品文檔，請參見 [分段服務概述](../../segmentation/home.md)

## 資料治理 {#governance}

Adobe Experience Platform允許公司將來自多個企業系統的資料整合在一起，以便營銷人員能夠識別、理解和吸引客戶。 [!DNL Experience Platform] 包括端到端的資料治理基礎架構，以確保正確使用 [!DNL Platform] 和系統間共用時。

資料治理是用於管理客戶資料和確保遵守適用於資料使用的法規、限制和策略的一系列策略和技術。 它在 [!DNL Experience Platform] 包括編目、資料沿襲、資料使用標籤、資料存取策略和對資料的訪問控制，以執行市場營銷操作。

開始資料治理需要對適用於您的客戶資料的法規、合同義務和公司策略有透徹的瞭解。 從那裡，可以通過應用適當的資料使用標籤來對資料進行分類，並且可以通過定義資料使用策略來控制其使用。

Data Governance框架通過以下幾項簡化並優化了資料分類和建立資料使用策略的過程： [!DNL Experience Platform] 用戶介面和 [!DNL Policy Service] API。

| 功能 | 說明 |
| -----------| ---------- |
| 標籤 [!DNL Experience Platform] 實體 | 基於預定義的分類標籤在連接、資料集和資料集欄位級別應用和管理資料使用標籤。 |
| 建立資料使用策略 | 根據標籤和請求的市場營銷操作的存在來定義和管理資料使用策略。 |
| 檢查策略違規 | 評估在執行市場營銷操作時是否違反了定義的使用策略 [!DNL Platform] 資料。 |

**已知問題**

* None.

有關詳細資訊，請參閱 [資料治理概述](../../data-governance/home.md))。
