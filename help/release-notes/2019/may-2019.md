## [!DNL Data Ingestion] {#ingestion}

Adobe Experience Platform提供一組豐富的功能，可吸收任何類型和任何延遲的資料。 您可以使用Adobe內建的連接器、資料整合合作夥伴或Adobe Experience Platform UI，使用批次或串流API進行收錄。

如需產品檔案，請參閱資 [料擷取概觀](../../ingestion/home.md)。

### 批次擷取

使用批次擷取API將任何類型的資料擷取至Adobe Experience Platform。 Bulk Ingestion API提供安全且可擴充的機制，來擷取符合XDM或非XDM架構的資料。

| 功能 | 說明 |
| -----------| ---------- |
| 多樣化格式支援 | 支援擷取JSON、Parke、CSV |
| 資料驗證 | 對資料執行三個驗證層級： 方案、資料類型和約束 |
| 動態資料加密 | 在跨資料網路傳輸資料時保護和保護資料。 |
| 多部分上傳 | 檔案256MB或以上支援大型檔案上傳 |
| 自動分區 | 資料會自動分區，以提高可擴充性並優化大型資料集的效能。 |

**已知問題**

無.

如需產品檔案，請參閱批次 [擷取概觀](../../ingestion/batch-ingestion/overview.md)。

### 連接器

Adobe Experience Platform Connectors可協助您輕鬆從多個來源擷取資料，讓您使用服務來建構、標示並增強資 [!DNL Experience Platform] 料。 [!DNL Platform] 連接器可讓您驗證至雲端儲存系統和CRM服務、設定擷取執行的時間，以及管理資料擷取吞吐量。

| 功能 | 說明 |
| -----------| ---------- |
| 雲端儲存空間連接器 | 將資料從雲 [!DNL Microsoft Azure Blob] 端儲 [!DNL Amazon S3] 存空間內嵌至Adobe Experience Platform |
| [!DNL Microsoft Dynamics CRM] &amp; [!DNL Salesforce CRM] | 探索或 [!DNL Microsoft Dynamics] CRM中的所有 [!DNL Salesforce] 資料。 從或 [!DNL Microsoft Dynamics][!DNL Salesforce] CRM中收錄任何物件、標準或自訂。 增量資料可以每小時或每日接收。 |  |

**已知問題**

如需產品檔案，請參 [閱連接器概觀](../../sources/home.md)。

## [!DNL Experience Data Model] (XDM) {#xdm}

標準化和互操作性是其背後的關鍵概念 [!DNL Experience Platform]。 [!DNL Experience Data Model] (XDM)是由Adobe推動，旨在標準化客戶體驗資料並定義客戶體驗管理的架構。

XDM是公開記載的規格，旨在改善數位體驗的強大功能。 它提供任何應用程式的通用結構和定義，以便與Adobe Experience Platform上的服務通訊。 遵循XDM標準，所有客戶體驗資料都可整合在以更快速、更整合的方式提供見解的通用表現形式中。 您可以從客戶行動中獲得寶貴見解，透過細分定義客戶受眾，並將客戶屬性用於個人化目的。

XDM是一種機制，可讓 [!DNL Experience Clou]d在Adobe Experience Platform的支援下，在正確的時刻，在正確的通道上向適當的人傳遞正確的訊息。

構建的方 [!DNL Experience Platform] 法可操作 [!DNL XDM System] 方 [!DNL Experience Data Model] 案供元件使 [!DNL Experience Platform] 用。

| 功能 | 說明 |
| -----------| ---------- |
| [!DNL Schema Registry] | 提供 [!DNL Schema Registry] 了一個用戶介面([!DNL Schema Editor])和REST風格的API，從中可以查看和管理所有與架構相關的資源。 此 [!DNL Schema Editor] 和 [!DNL Schema Registry] API也可用來建立和管理每個IMS組織專屬的新架構和資源。 |
| [!DNL Schema Library] | 其中 [!DNL Schema Library] 包含所 [!DNL Schema Registry] 有資源，包括由個別IMS組織定義的資源，以及由Adobe、合作夥伴和廠商（其應用程式已整合）提供給所有使用者的資源 [!DNL Experience Platform][!DNL Experience Platform]。 |

**已知問題**

無.

若要進一步瞭解如何使用 [!DNL Schema Registry] API和XDM [!DNL Schema Editor]，請閱讀 [XDM系統檔案](../../xdm/home.md)。

## [!DNL Real-time Customer Profile] {#profile}

[!DNL Profile] 是一般查閱實體儲存，可合併各種企業資料資產的資料，並以即時使用者描述檔和相關時間系列事件（稱為聯合檢視）的形式提供該資料的存取。 統一的檢視有助於通過各個通道、Adobe解決方案整合來 [!DNL Platform]通知並授權行動。 此外，還 [!DNL Profile] 被用來推動機器學習和Sensei。 [!DNL Profile] API也可直接用於豐富協力廠商解決方案、CRM系統和專屬解決方案的功能。

| 功能 | 說明 |
| -----------| ---------- |
| [!DNL Real-time customer profile] | [!DNL Profile] 透過資料入口收集資料、將其與描述檔儲存區中的現有資料合併，並建立或修改統一檢視。 這些資料會隨著客戶體驗而及時發展，可用來提供個人化建議。 |
| 聯合視圖 | 多通道資料的統 [!DNL Profile] 一性是其關鍵特點之一。 [!DNL Profile] 可以提供消費者群、跨資料集和連結身分的個人合併檢視，稱為聯合檢視。 |
| Edge | Edge Service可做為低延遲資料收集、可插拔運算和快速啟動所有可定址通道的架構。 Edge Service可協助即時個人化。 |

**已知問題**

無.

如需產品檔案，請參 [閱描述檔概述](../../profile/home.md)。

## Identity 服務 {#identity}

Identity Service可解決客戶不連貫的身分所帶來的根本挑戰，因為客戶資料在您組織的生態系統中的各個不同系統上分散。 它可跨裝置橋接身分，並跨消費者使用的各種系統，與品牌互動。 如此，Identity Service可協助您全面瞭解客戶及其行為，以便您即時提供具影響力的數位體驗。

| 功能 | 說明 |
| -----------| ---------- |
| 身分圖 | 身分圖是身份之間關係的映射，可即時隨客戶活動更新。 [!DNL Identity Service] 管理僅由您的組織可見的識別圖形，並根據您的資料（稱為私用圖形）建立。 [!DNL Identity Service] 當收錄的資料記錄包含多個身分時，增加您的私人圖表，並在這些身分之間新增關係。 |

**已知問題**

無.

如需產品檔案，請參 [閱Identity Service概觀](../../identity-service/home.md)。

## [!DNL Segmentation Service] {#segmentation}

[!DNL Segmentation Service] 定義描述檔存放區中描述檔的特定子集，並說明區分描述檔存放區中可銷售人員群組的標準。 區段可以根據記錄資料（例如人口統計資訊）或代表客戶與品牌接觸點的時間系列事件來劃分。

例如，在名為「您忘記購買運動鞋嗎？」的電子郵件促銷活動中，您可能想要在過去30天內搜尋跑鞋但未完成購買的所有使用者的對象區段。 或者，您可能想要將網頁內容定位為只顯示給獎勵方案中具有特定評分的使用者。

| 功能 | 說明 |
| -----------| ---------- |
| 區段API | REST風格的服務可讓您以程式化方式存取建立客戶細分。 |
| 區段產生器 | 區段產生器是強穩且直覺式的工作區，可從您的資料建立客戶 [!DNL Profile] 區段。 |

**已知問題**

無.

如需產品檔案，請參閱區 [段服務概觀](../../segmentation/home.md)

## [!DNL Data Governance] {#governance}

Adobe Experience Platform可讓公司將來自多個企業系統的資料整合在一起，讓行銷人員更能識別、瞭解並吸引客戶。 [!DNL Experience Platform] 包括端對端資料治理基礎架構(包括 [!DNL Data Usage Labeling and Enforcement] (DULE))，以確保在系統內和系統間共 [!DNL Platform] 享時正確使用資料。

資料治理是用於管理客戶資料並確保遵守適用於資料使用的法規、限制和政策的一系列策略和技術。 它在各個層級中都扮演了關 [!DNL Experience Platform] 鍵角色，包括編目、資料傳承、資料使用標籤、資料存取政策，以及對行銷動作資料的存取控制。

開始使用資料治理需要對適用於客戶資料的法規、合約義務和公司政策有深入的瞭解。 在此基礎上，通過應用適當的資料使用標籤對資料進行分類，並通過定義資料使用策略來控制其使用。

DULE框架通過用戶介面和DULE [!DNL Experience Platform][!DNL Policy Service] API簡化了資料分類和建立資料使用策略的過程。

| 功能 | 說明 |
| -----------| ---------- |
| 標籤實 [!DNL Experience Platform] 體 | 根據預先定義的分類標籤，在連線、資料集和資料集欄位層級套用並管理資料使用標籤。 |
| 建立資料使用原則 | 根據標籤和要求的行銷動作，定義並管理資料使用政策。 |
| 檢查策略違規 | 評估對資料執行行銷動作時是否違反定義的使用 [!DNL Platform] 原則。 |

**已知問題**

* 無.

如需詳細資訊，請參閱資 [料治理概觀](../../data-governance/home.md))。
