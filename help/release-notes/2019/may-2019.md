## [!DNL Data Ingestion] {#ingestion}

Adobe Experience Platform提供一組豐富的功能，可擷取任何類型和任何資料延遲。 您可以使用批次或串流API、Adobe建置的連接器、資料整合合作夥伴或Adobe Experience Platform UI來內嵌。

如需產品檔案，請參閱 [資料擷取概觀](../../ingestion/home.md).

### 批次擷取

使用批次擷取API將任何類型的資料內嵌至Adobe Experience Platform。 大量擷取API提供安全且可擴充的機制，可擷取符合XDM或非XDM結構的資料。

| 功能 | 說明 |
| -----------| ---------- |
| 豐富格式支援 | 支援擷取JSON、Parquet、CSV |
| 資料驗證 | 對資料執行三個層級的驗證：結構、資料類型和限制 |
| 移動中的資料加密 | 在資料網路間傳輸時保護和保護資料。 |
| 多部分上載 | 256MB或更大檔案支援的大檔案上載 |
| 自動分割 | 資料會自動分割，以改善擴充性並最佳化大型資料集的效能。 |

**已知問題**

None.

如需產品檔案，請參閱 [批次匯入概觀](../../ingestion/batch-ingestion/overview.md).

### 連接器

Adobe Experience Platform連接器可協助您輕鬆從多個來源擷取資料，讓您使用來建構、加上標籤，以及增強資料 [!DNL Experience Platform] 服務。 [!DNL Platform] 連接器可讓您驗證雲端儲存系統和CRM服務、設定擷取執行的時間，以及管理資料擷取吞吐量。

| 功能 | 說明 |
| -----------| ---------- |
| 雲端儲存連接器 | 從內嵌資料 [!DNL Microsoft Azure Blob] 和 [!DNL Amazon S3] 雲端儲存空間至Adobe Experience Platform |
| [!DNL Microsoft Dynamics CRM] &amp; [!DNL Salesforce CRM] | 探索 [!DNL Microsoft Dynamics] 或 [!DNL Salesforce] CRM。 擷取任何物件、標準或自訂 [!DNL Microsoft Dynamics] 或 [!DNL Salesforce] CRM。 增量資料可以每小時或每天擷取。 |  |

**已知問題**

* 雲端型儲存連接器只能透過API呼叫使用。


如需產品檔案，請參閱 [連接器概述](../../sources/home.md).

### Adobe解決方案

Adobe提供從Adobe解決方案到Adobe Experience Platform的受管資料傳輸。 透過此版本，資料傳輸可從下列解決方案使用。

| 功能 | 說明 |
| -----------| ---------- |
| Adobe Analytics | 來自Adobe Analytics的報表套裝資料位於 [!DNL Platform]. |
| Adobe Audience Manager | 來自Adobe Audience Manager的資料收集記錄、已上線資料、特徵和區段資料，可在 [!DNL Platform]. |
| [!DNL Advertising Cloud] | DSP記錄來自 [!DNL Advertising Cloud] 可在 [!DNL Platform]. |
| Adobe Campaign Standard | 來自Adobe Campaign Standard的設定檔和事件資料可在 [!DNL Platform]. |
| [!DNL Target] (預覽) | 來自Adobe Target的事件資料可在 [!DNL Platform]. |


**已知問題**

None.

### 串流擷取

串流內嵌會將資料傳送至 [!DNL Experience Platform] 即時從用戶端和伺服器端裝置傳送。 可對 [!DNL Real-Time Customer Profiles] 在次秒延遲，支援已驗證的資料收集和多記錄負載。

| 功能 | 說明 |
| -----------| ---------- |
| 同步驗證 | 可立即發現JSON裝載中的錯誤，提高開發人員速度。 |
| 端對端監控 | 監控從擷取到耗用的串流資料 [!DNL Data Lake] 和 [!DNL Real-Time Customer Profile]. |
| 已驗證的資料收集 | 開啟或關閉選項，以要求用戶端提供驗證Token。 允許Adobe Experience Platform服務區分來自受信任源和不受信任源的記錄。 |
| 透過即時傳送資料 [!DNL Launch] | 在中使用Adobe Experience Platform擴充功能 [!DNL Launch] 可輕鬆即時從用戶端裝置傳送XDM格式的資料。 |

**已知問題**

None.

如需產品檔案，請參閱 [串流獲取概觀](../../ingestion/streaming-ingestion/overview.md).

### 監控

即時監控從用戶端和伺服器端裝置批次和串流內嵌至Adobe Experience Platform。

| 功能 | 說明 |
| -----------| ---------- |
| 監控串流資料 | 端到端監控流資料到 [!DNL Data Lake], [!DNL Real-Time Customer Profile]和 [!DNL Identity Service]. |

**已知問題**

None.

## [!DNL Experience Data Model] (XDM) {#xdm}

標準化和互操作性是背後的關鍵概念 [!DNL Experience Platform]. [!DNL Experience Data Model] (XDM)受Adobe推動，致力標準化客戶體驗資料並定義客戶體驗管理的結構。

XDM是公開記錄的規格，旨在改善數位體驗的強大功能。 它為與Adobe Experience Platform上的服務通訊的任何應用程式提供共同的結構和定義。 遵循XDM標準，所有客戶體驗資料皆可整合至以更快速、更整合的方式提供深入分析的通用表示法中。 您可以從客戶動作中獲得寶貴的深入分析、透過區段定義客戶受眾，以及將客戶屬性用於個人化目的。

XDM是可讓 [!DNL Experience Clou]d由Adobe Experience Platform提供技術支援，在正確的時間，在正確的通道向正確的人傳遞正確的訊息。

方法 [!DNL Experience Platform] 是建的， [!DNL XDM System] 操作 [!DNL Experience Data Model] 供使用的結構 [!DNL Experience Platform] 元件。

| 功能 | 說明 |
| -----------| ---------- |
| [!DNL Schema Registry] | 此 [!DNL Schema Registry] 提供使用者介面([!DNL Schema Editor])和RESTful API，可從中檢視和管理所有架構相關資源。 此 [!DNL Schema Editor] 和 [!DNL Schema Registry] API也可用來建立及管理每個IMS組織特有的新結構和資源。 |
| [!DNL Schema Library] | 此 [!DNL Schema Library] 包含全部 [!DNL Schema Registry] 資源，包括由個別IMS組織定義的資源，以及由Adobe提供給所有使用者的資源， [!DNL Experience Platform] 將應用程式與 [!DNL Experience Platform]. |

**已知問題**

None.

若要進一步了解如何使用XDM，請使用 [!DNL Schema Registry] API和 [!DNL Schema Editor]，請閱讀 [XDM系統檔案](../../xdm/home.md).

## [!DNL Real-Time Customer Profile] {#profile}

[!DNL Profile] 是一般查詢實體存放區，可合併各種企業資料資產中的資料，並以即時消費者設定檔和相關時間序列事件（稱為聯合檢視）的形式提供對該資料的存取。 統一的檢視有助於為各管道的動作提供相關資訊並賦予其權力， [!DNL Platform]，以及Adobe解決方案整合。 此外， [!DNL Profile] 可用來支援機器學習和Sensei。 [!DNL Profile] API也可直接用來擴充協力廠商解決方案、CRM系統和專屬解決方案的功能。

| 功能 | 說明 |
| -----------| ---------- |
| [!DNL Real-time customer profile] | [!DNL Profile] 透過資料匯入點內嵌資料、將其與設定檔存放區中的現有資料合併，以及幾乎即時建立或修改統一檢視。 這些資料會隨著您客戶的體驗而及時演化，可用來提供個人化建議。 |
| 聯合視圖 | 其中一項主要功能 [!DNL Profile] 是整合多通道資料的能力。 [!DNL Profile] 可提供消費者基礎、跨資料集和連結身分的合併檢視（稱為聯合檢視）。 |
| Edge | 邊緣服務是所有可定址通道中低延遲資料收集、可插拔計算和快速資料激活的框架。 邊緣服務可促進即時個人化。 |

**已知問題**

None.

如需產品檔案，請參閱 [設定檔概述](../../profile/home.md).

## 身份識別服務 {#identity}

Identity服務解決了由客戶的不連結身份構成的根本難題，這是由客戶資料在您組織的生態系統中所扮演的每個不同系統上分散的性質所造成。 它可透過跨裝置以及消費者與品牌互動所使用的各種系統橋接身分來達成此目的。 這樣，Identity Service便利了對客戶及其行為的完整了解，讓您能夠即時提供具影響力的數位體驗。

| 功能 | 說明 |
| -----------| ---------- |
| 身分圖 | 身分圖表是身分之間關係的地圖，會近乎即時隨客戶活動更新。 [!DNL Identity Service] 管理僅由您的組織可見且以您的資料為基礎（稱為私密圖表）建置的身分圖表。 [!DNL Identity Service] 當擷取的資料記錄包含多個身分時，增加這些身分之間的關係，以擴大您的私人圖表。 |

**已知問題**

None.

如需產品檔案，請參閱 [Identity服務概述](../../identity-service/home.md).

## [!DNL Segmentation Service] {#segmentation}

[!DNL Segmentation Service] 會從您的設定檔存放區定義特定的設定檔子集，並說明區分設定檔存放區中可行銷人員群組的條件。 區段可以根據記錄資料（例如人口統計資訊）或時間系列事件，代表客戶與您品牌的接觸點。

例如，在名為「您忘了購買運動鞋嗎？」的電子郵件行銷活動中，您可能想要有一個對象區段，該區段包含過去30天內搜尋跑鞋，但未完成購買的所有使用者。 或者，您可能想要將網頁內容定位為只向獎勵方案中具有特定評分的使用者顯示。

| 功能 | 說明 |
| -----------| ---------- |
| 區段API | RESTful服務可讓您程式化地存取以建立客戶區段。 |
| 區段產生器 | 區段產生器是功能強大且直覺的工作區，可讓您從 [!DNL Profile] 資料。 |

**已知問題**

None.

如需產品檔案，請參閱 [區段服務概觀](../../segmentation/home.md)

## 資料治理 {#governance}

Adobe Experience Platform可讓公司將來自多個企業系統的資料匯整在一起，讓行銷人員更能識別、了解客戶，並與其互動。 [!DNL Experience Platform] 包括端到端的資料治理基礎架構，以確保在中正確使用資料 [!DNL Platform] 和在系統之間共用時。

資料控管是一系列策略和技術，用於管理客戶資料並確保符合適用於資料使用的法規、限制和政策。 在 [!DNL Experience Platform] 在各個層級，包括編目、資料處理、資料使用標籤、資料存取策略，以及針對行銷動作的資料存取控制。

若要開始使用資料控管功能，您必須全面了解適用於客戶資料的法規、合約義務和公司政策。 從那裡，資料可以通過應用適當的資料使用標籤來分類，而資料的使用可以通過定義資料使用策略來控制。

資料控管架構透過 [!DNL Experience Platform] 使用者介面和 [!DNL Policy Service] API。

| 功能 | 說明 |
| -----------| ---------- |
| 標籤 [!DNL Experience Platform] 實體 | 根據預先定義的分類標籤，在連線、資料集和資料集欄位層級套用及管理資料使用量標籤。 |
| 建立資料使用原則 | 根據是否有標籤和請求的行銷動作，定義和管理資料使用政策。 |
| 檢查策略違規 | 評估在執行行銷動作時是否違反定義的使用政策 [!DNL Platform] 資料。 |

**已知問題**

* None.

如需詳細資訊，請參閱 [資料控管概觀](../../data-governance/home.md))。
