# Adobe Experience Platform 發行說明

**發行日期：2019 年 6 月 28 日**

Adobe Experience Platform的新功能：

* [[!DNL Data Science Workspace]](#dsw)
* [[!DNL Decisioning Service]](#decisioning)
* [[!DNL Query Service]](#query)

更新現有功能：

* [[!DNL Experience Data Model (XDM)]](#xdm)
* [[!DNL Segmentation Service]](#segmentation)

## [!DNL Data Science Workspace] {#dsw}

Adobe Experience Platform[!DNL Data Science Workspace]是[!DNL Experience Platform]內的完整管理服務，可讓資料科學家建立並執行機器學習模型，從Adobe解決方案和協力廠商系統的資料和內容順暢地獲得見解。 [!DNL Data Science Workspace] 與端對端資 [!DNL Platform] 料科學生命週期緊密整合，包括探索和準備XDM資料，然後開發和運作模型以自動豐富機器學習 [!DNL Real-time Customer Profile] 見解。

**主要功能**

| 功能 | 說明 |
| -----------| ---------- |
| 資源調配和計算隔離 | 為資料科學家提供所需的專用計算資源，使其能夠以安全方式在[!DNL Experience Platform]內執行不受信任的代碼。 |
| 首次使用者體驗 | 包含各種機器學習架構和語言（例如[!DNL Python]、R、PySpark和Scala [!DNL Spark]）的現成可用範例。 |
| 筆記型電腦 | 為[!DNL Jupyter Notebooks]支援的資料科學家／資料工程師量身打造的環境，讓他們能夠準備資料、擷取功能，並使用精選的資料庫清單和常用的機器學習架構來開發ML模型。 |
| 資料探索 | 順暢地存取內嵌至與[!DNL Platform Data Access] SDK整合之[!DNL Platform]的XDM資料。 |
| 資料視覺化 | 能夠在[!DNL Jupyter Notebooks]中執行SQL查詢以加速資料準備和功能工程。 |
| 特徵管線 | API/SDK for Scala/PySpark部署功能工程管道，將核心XDM資料轉換為功能架構。 |
| 製作模型 | 範本和執行時期，讓資料科學家可專注於模型開發，而不需建置存取資料和計算資源的基礎架構程式碼。 您可以匯入模型程式碼並加以運作，以便從[!DNL Platform]中的資料衍生前瞻分析。 |
| 企業模型管理 | 支援多租用戶資料模型，以追蹤模型版本和相關的超參陣列態，為合作夥伴生態系統提供基礎。 |
| 模型評估 | 評估並最佳化[!DNL Python]、PySpark、R和Scala中的回歸與分類模型。 |
| 模型部署 | 能夠比較多個實驗執行中的評估度量和設定，並將最佳模型發佈為服務。 |
| 批次計分 | 運用機器學習深入資訊豐富[!DNL Real-time Customer Profile]，或將它們寫回[!DNL Platform]資料集 |
| 排程 | 與[!DNL Platform]協調服務整合，透過API將模型訓練、計分和功能管道與使用者定義的排程自動化。 |

**已知問題**

* 排程和功能管道目前僅透過API提供，而UI將在未來版本中新增。

如需詳細資訊，請造訪[資料科學工作區概觀](../../data-science-workspace/home.md)。

## [!DNL Decisioning Service] {#decisioning}

Adobe Experience Platform[!DNL Decisioning Service]提供以程式設計方式智慧地從一組可供特定個人使用的選項中選取「下一個最佳體驗」、將這些選項傳送至任何通道或應用程式，以及執行報告和分析的功能。

預先建立的豐富資料模型可讓「下一個最佳選件」決策案例不受通道限制。

**主要功能**

| 功能 | 說明 |
| -----------| ---------- |
| 業務對象儲存庫 | 由JSON結構描述模型驅動的儲存庫可讓開發人員建立、讀取、更新和刪除各種商業物件。 儲存庫提供通用、表達能力強的查詢API以及模式感知搜索。 |
| 儲存庫容器 | 在業務對象儲存庫中，開發人員可以將其關注點與項目、業務或組織單位或項目生命週期階段（例如，在開發和整合、接移或用於即時生產）隔離開來。 這些隔離稱為儲存庫容器。 |
| 角色與權限 | 使用[!DNL Admin Console]，組織可以建立和管理配置檔案，以按類型、訪問操作和容器授予對資源的目標訪問權。 用戶可以添加到這些訪問配置檔案中，並自動根據這些策略計算有效訪問權限。 |
| 預先建立的選件物件模型 | 不需先建立資料模型，[!DNL Platform]開發人員可運用預先建立的JSON結構描述和關係來建立選件目錄、定義決策規則和限制，以及組合選件集合以進行決策。 |
| 基於配置檔案和非配置檔案資料的決策規則 | 與[!DNL Real-time Customer Profile]的緊密整合可讓開發人員建立運用描述檔資料的決策規則。 不僅可以使用描述檔屬性做出決策，而且可以根據描述檔的體驗事件歷史記錄以及與使用者身分無關的業務實體（例如流量條件、產品庫存）做出決策。 任何[!DNL Experience Data Model](XDM)實體（其架構存在於[!DNL Schema Registry]中）都可用於決策規則。 規則是第一類實體，可重複用於任何決策選項和活動。 |
| 排名與上限設定 | 對滿足給定用戶所有資格和其他約束的決策選項進行排名，並選擇最佳選項。 可使用額外的每位使用者和全域封閉限制來限制可用選項的曝光度，從而在考量資源限制和使用者疲勞時提供個人化。 |
| [!DNL Decisioning] REST API | [!DNL Decisioning Service]可使用簡單的REST API來呼叫，以取得特定個人的下一個最佳選件。 量度API可用來檢查即時選件提案和限定值。 |
| 將決策事件串流至[!DNL Data Lake]和[!DNL Query Service] | [!DNL Decisioning Service]會自動建立資料集，將所有XDM決策事件自動串流至[!DNL Data Lake]。 然後，這些資料集便可使用[!DNL Query Service]進行分析和報告。 |
| 開發人員支援 | 自助選擇加入，並提供Adobe I/O檔案，包括各種主題的教學課程。 |

**已知問題**

* 選件資料模型不會透過[!DNL Schema Registry]公開，因此只能以有限的方式延伸。 模型架構具有內建結構，允許附加自訂資料。 未來，您將可擴充基本XDM模型類別，以定義您自己的自訂決策網域。
* 您必須已布建「選件管理」網域模型，而且必須在此產品內容中管理使用者和整合。

## [!DNL Query Service] {#query}

[!DNL Query Service] 提供使用標準SQL查詢Adobe Experience Platform資料的能力，以支援許多不同的分析和資料管理使用案例。它是一種無伺服器工具，允許您將[!DNL Data Lake]中的任何資料集連接起來，並將查詢結果捕獲為用於報告、[!DNL Data Science Workspace]或用於向[!DNL Profile Service]提取的新資料集。

您可以使用[!DNL Query Service]來建立資料分析生態系統，以建立消費者在不同互動通道中的形象。 這些渠道可能包括：

* 銷售點系統
* Web
* 行動
* CRM系統

**主要功能**

| 功能 | 說明 |
| -----------| ---------- |
| 查詢編輯器 | 使用線上工具來編寫、驗證、測試和執行查詢。 它包括一個控制台，用於提供有關查詢執行的詳細資訊以及預覽查詢結果的功能。 |
| 資料集建立 | 通過標準SQL語法在[!DNL Experience Platform]上建立資料集。 |
| Adobe定義函式 | 運用捷徑功能來處理常見工作，例如識別工作階段或設定歸因。 |
| BI工具連接 | 使用常見BI工具中的PostgreSQL(Postgres)驅動程式連線至[!DNL Query Service]以建立報表和視覺化。 支援的工具包括：[!DNL Tableau]、[!DNL Power BI]和[!DNL Looker]。 在「認證」標籤上尋找驗證資訊。 |
| 資料庫管理工具連接 | 將[!DNL Aqua Data Studio]或[!DNL DB Visualizer]連線至[!DNL Query Service]，以取得資料探索和資料集建立功能。 [!DNL Query Service] 還支援從R Studio連接。在「認證」標籤上尋找驗證資訊。 |
| 命令列查詢工具 | 連接PSQL以能夠從命令行運行查詢。 |
| 查詢日誌 | 保留由[!DNL Query Service]執行的查詢歷史記錄，並使您能夠查找以前的SQL，以便編輯、執行或在結果中建立資料集。 |
| 查詢排程API | 透過此API排程週期性執行的查詢。 |

**已知問題**

* [!DNL Query Editor] 顯示查詢結果的100行示例。為了保存完整的結果集，請使用查詢日誌中的資料集建立功能。
* 近期發行將新增對「檢視」的支援，以及套用排程至查詢的UI。

如需[!DNL Query Service]的詳細資訊，請參閱[產品檔案](../../query-service/home.md)。

## [!DNL Experience Data Model] (XDM) {#xdm}

標準化和互操作性是[!DNL Experience Platform]背後的關鍵概念。 [!DNL Experience Data Model] (XDM)由Adobe驅動，旨在標準化客戶體驗資料並定義客戶體驗管理的架構。

XDM是公開記載的規格，旨在改善數位體驗的強大功能。 它為與Adobe Experience Platform服務通訊的任何應用程式提供共同的架構和定義。 遵循XDM標準，所有客戶體驗資料都可整合在以更快速、更整合的方式提供見解的通用表現形式中。 您可以從客戶行動中獲得寶貴見解，透過細分定義客戶受眾，並將客戶屬性用於個人化目的。

XDM是一種機制，允許[!DNL Experience Cloud](由Adobe Experience Platform提供支援)在正確的時間，在正確的頻道上向適當的人傳遞正確的訊息。

建立[!DNL Experience Platform]的方法， [!DNL XDM System]操作[!DNL Experience Data Model]方案，以便由[!DNL Experience Platform]元件使用。

**新功能**

| 功能 | 說明 |
| ---------- | ------------ |
| JSON結構描述限制 | 下列資料類型現在在使用者介面中有其他選項可定義限制：`string` —— 最小／最大長度、模式、預設值、格式（如[JSON結構描述草稿-6](https://tools.ietf.org/html/draft-wright-json-schema-01)中所定義）和`double` —— 最小／最大，預設值。 |
| 自訂 `$id` | 現在，在POST請求中建立資源時，您可以提供自己的`$id`值。 |
| 架構註冊表效能改進 | 最佳化聯合架構產生，並增強架構快取，以大幅改善API回應時間。 |

**錯誤修正**

* 將identityMap欄位移出上下文／描述檔並移入其自己的架構欄位群組，讓定義身分更直覺。
* 已修補所有基於上下文／描述檔且具有上下文/identitymap的現有結構。
* 修正未提供版本時的錯誤訊息。
* 已修正[!DNL Schema Registry]為描述檔結合結構調用提供隨機回應的錯誤。
* 修正了聯合結構描述未在[!DNL Schema Registry]中顯示正確欄位的錯誤。
* 已修正身分描述子有時無法使用有效名稱空間建立的錯誤。
* 修正物件使用`properties`而非`allOf`時的取消參照問題。

**已知問題**

* 無法通過添加欄位來擴展[!DNL Platform]提供的欄位組。
* 從模式組合中刪除欄位組時，不刪除描述符。
* 無法建立沒有標籤的枚舉欄位。

要瞭解有關使用[!DNL Schema Registry] API和[!DNL Schema Editor]使用XDM的更多資訊，請閱讀[ XDM系統文檔](../../xdm/home.md)。

## [!DNL Segmentation Service] {#segmentation}

[!DNL Segmentation Service] 定義描述檔存放區中描述檔的特定子集，並說明區分描述檔存放區中可銷售人員群組的標準。區段可以根據記錄資料（例如人口統計資訊）或代表客戶與品牌接觸點的時間系列事件來劃分。

例如，在以跑鞋為主的電子郵件促銷活動中，您可以使用過去30天內搜尋跑鞋但未完成購買的所有使用者的對象區段。 另一個例子是使用區段來定位網站內容，以便僅顯示給屬於您獎勵方案特定層次的訪客。

**新功能**

| 功能 | 說明 |
| -----------| ---------- |
| 相對時間規則 | 您現在可以選擇滾動時間窗口，例如14天前、3到5小時前等。 |
| XDM欄位摘要 | 對於左側導軌上的「屬性」，現在提供摘要，以檢視您的基礎資料。 |
| 左側搜尋 | 已改善左側邊欄區段部分的搜尋功能。 |
| eVar好記的名稱 | 改善對好記名稱的支援，讓您更輕鬆地查看從Adobe Analytics的自訂事件和維度擷取的資訊。 |
| 合併原則支援 | 您現在可以使用簡單的下拉式清單，選擇要套用至其區段定義的合併原則。 |

**錯誤修正**

* 修正造成左側導軌中屬性和事件建立區塊載入緩慢的間歇性問題。
* 修正導致估計器傳回「NaN」回應的錯誤。
* 修正某些欄位開啟錯誤規則建立畫布的錯誤。

**已知問題**

* None.

如需詳細資訊，請參閱[分段服務概觀](../../segmentation/home.md)。
