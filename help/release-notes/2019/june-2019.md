# Adobe Experience Platform 發行說明

**發行日期：2019 年 6 月 28 日**

Adobe Experience Platform的新功能：

* [[!DNL Data Science Workspace]](#dsw)
* [[!DNL Decisioning Service]](#decisioning)
* [[!DNL Query Service]](#query)

對現有功能的更新：

* [[!DNL Experience Data Model (XDM)]](#xdm)
* [[!DNL Segmentation Service]](#segmentation)

## [!DNL Data Science Workspace] {#dsw}

Adobe Experience Platform [!DNL Data Science Workspace] 是內部的完全托管服務 [!DNL Experience Platform] 使資料科學家能夠通過構建和操作機器學習模型，跨Adobe解決方案和第三方系統無縫地從資料和內容中生成洞見。 [!DNL Data Science Workspace] 與 [!DNL Platform] 並為端到端資料科學生命週期提供動力，包括探索和準備XDM資料，然後開發和實施模型以自動豐富 [!DNL Real-time Customer Profile] 和機器學習的見解。

**主要功能**

| 功能 | 說明 |
| -----------| ---------- |
| 資源調配和計算隔離 | 配置所需的專用計算資源，使資料科學家能夠在內部執行不受信任的代碼 [!DNL Experience Platform] 以安全的方式。 |
| 首次用戶體驗 | 包括各種機器學習框架和語言(如 [!DNL Python]、R、PySpark和Scala [!DNL Spark]。 |
| 筆記本 | 為資料科學家/資料工程師提供由 [!DNL Jupyter Notebooks] 使他們能夠準備資料、提取特徵，並使用已建立的庫清單和流行的機器學習框架開發ML模型。 |
| 資料勘探 | 無縫訪問接收到的XDM資料 [!DNL Platform] 整合 [!DNL Platform Data Access] SDK。 |
| 資料可視化 | 能夠在中執行SQL查詢 [!DNL Jupyter Notebooks] 加快資料準備和功能工程。 |
| 特徵管線 | 用於Scala/PySpark的API/SDK，用於部署功能工程管線，以將核心XDM資料轉換為功能模式。 |
| 模型創作 | 使資料科學家能夠專注於模型開發而無需實施訪問資料和計算資源的基礎架構代碼的模板和運行時。 您可以導入模型代碼並對其進行操作，以從中的資料中獲取Insights [!DNL Platform]。 |
| 企業模型管理 | 支援多租戶資料模型以跟蹤模型版本和相關的超參數配置，為合作夥伴生態系統提供基礎。 |
| 模型評估 | 評價和優化回歸和分類模型 [!DNL Python]、PySpark、R和Scala。 |
| 模型部署 | 能夠跨多個實驗運行比較評估度量和配置，並將最佳模型作為服務發佈。 |
| 批記分 | 豐 [!DNL Real-time Customer Profile] 或將其作為資料集寫回 [!DNL Platform] |
| 計畫 | 與 [!DNL Platform] 業務流程服務，通過API使用用戶定義的計畫自動化模型培訓、評分和功能管道。 |

**已知問題**

* 當前僅通過API提供調度和功能管線，在將來的版本中添加一個UI。

有關詳細資訊，請訪問 [資料科學工作區概述](../../data-science-workspace/home.md)。

## [!DNL Decisioning Service] {#decisioning}

Adobe Experience Platform [!DNL Decisioning Service] 提供了以寫程式方式智慧地為給定個人從一組可用選項中選擇「下一最佳體驗」、將它們交付到任何渠道或應用程式以及執行報告和分析的能力。

預構建的豐富資料模型支援以通道不可知的方式確定「下一個最佳服務」的使用案例。

**主要功能**

| 功能 | 說明 |
| -----------| ---------- |
| 業務對象儲存庫 | 由JSON架構模型驅動的儲存庫允許開發人員建立、讀取、更新和刪除各種業務對象。 儲存庫提供通用、有表達力的查詢API以及模式感知搜索。 |
| 儲存庫容器 | 在業務對象儲存庫中，開發人員可以將其關注點與項目、業務或組織單位或項目生命週期階段（例如，在開發和整合、試運行或用於即時生產使用中）隔離開來。 這些隔離稱為儲存庫容器。 |
| 角色與權限 | 使用 [!DNL Admin Console]，組織可以建立和管理配置檔案，以按類型、訪問操作和容器授予對資源的目標訪問權限。 可以將用戶添加到這些訪問配置檔案中，並根據這些策略自動計算有效訪問權限。 |
| 預構建的服務對象模型 | 無需首先構建資料模型， [!DNL Platform] 開發人員可以利用預構建的JSON架構和關係建立聘用目錄、定義決策規則和約束，以及匯編聘用集合以進行決策。 |
| 基於簡檔和非簡檔資料的決策規則 | 與 [!DNL Real-time Customer Profile] 允許開發人員建立利用配置檔案資料的決策規則。 不僅可以使用配置檔案屬性做出決策，而且還基於配置檔案的體驗事件歷史以及基於與用戶身份無關的業務實體（例如，交通狀況、產品庫存）。 任意 [!DNL Experience Data Model] (XDM)中存在模式的實體 [!DNL Schema Registry] 可用於決策規則。 規則是一類實體，可重新用於任何決策選項和活動。 |
| 排名和上限設定 | 對滿足給定用戶的所有資格和其它約束的決策選項進行排序，並選擇最佳選項。 可使用附加的每用戶和全局封閉約束來限制可用選項的暴露，從而允許在考慮資源約束和用戶疲勞的情況下實現個性化。 |
| [!DNL Decisioning] REST API | 的 [!DNL Decisioning Service] 可以使用簡單的REST API調用以獲得給定個人的下一最佳優惠。 度量API可用於檢查即時提供命題和上限值。 |
| 將決策事件流式傳輸到 [!DNL Data Lake] 和 [!DNL Query Service] | 的 [!DNL Decisioning Service] 自動建立資料集以自動將所有XDM決策事件流入 [!DNL Data Lake]。 然後，資料集可用於使用 [!DNL Query Service]。 |
| 開發人員支援 | 自助服務選擇加入，並提供Adobe I/O文檔，包括各種主題的教程。 |

**已知問題**

* 服務資料模型未通過 [!DNL Schema Registry] 因此只能以有限的方式進行擴展。 模型架構具有內置結構以允許附加自定義資料。 將來，您將能夠擴展基本XDM模型類以定義自己的自定義決策域。
* 必須為您設定「優惠管理」域模型，並且必須在此產品上下文中管理用戶和整合。

## [!DNL Query Service] {#query}

[!DNL Query Service] 提供了使用標準SQL查詢Adobe Experience Platform資料以支援多種不同分析和資料管理使用案例的能力。 它是一個無伺服器工具，允許您在 [!DNL Data Lake] 並將查詢結果捕獲為新資料集以用於報告， [!DNL Data Science Workspace]或者，攝入 [!DNL Profile Service]。

您可以使用 [!DNL Query Service] 構建資料分析生態系統，建立消費者跨各種交互渠道的圖景。 這些渠道可能包括：

* 銷售點系統
* Web
* 行動
* CRM系統

**主要功能**

| 功能 | 說明 |
| -----------| ---------- |
| 查詢編輯器 | 使用基於Web的工具來編寫、驗證、test和執行查詢。 它包括一個控制台，用於提供有關查詢執行的詳細資訊以及預覽查詢結果的功能。 |
| 資料集建立 | 在上建立資料集 [!DNL Experience Platform] 通過標準SQL語法。 |
| Adobe定義函式 | 利用快捷方式功能執行常見任務，如標識會話或設定屬性。 |
| BI工具連接 | 使用常用BI工具中找到的PostgreSQL(Postgres)驅動程式連接到 [!DNL Query Service] 建立報告和可視化。 支援的工具包括： [!DNL Tableau]。 [!DNL Power BI], [!DNL Looker]。 在「憑據」頁籤上查找驗證資訊。 |
| 資料庫管理工具連接 | 連接 [!DNL Aqua Data Studio] 或 [!DNL DB Visualizer] 至 [!DNL Query Service] 用於資料勘探和資料集建立功能。 [!DNL Query Service] 還支援來自R Studio的連接。 在「憑據」頁籤上查找驗證資訊。 |
| 命令行查詢工具 | 連接PSQL以能夠從命令行運行查詢。 |
| 查詢日誌 | 保留由執行的查詢的歷史記錄 [!DNL Query Service] 並使您能夠查找先前的SQL以從結果中編輯、執行或建立資料集。 |
| 查詢計畫API | 計畫通過此API執行的查詢。 |

**已知問題**

* [!DNL Query Editor] 顯示查詢結果的100行示例。 要保留完整的結果集，請使用「查詢日誌」中的資料集建立功能。
* 短期版本將添加對視圖和UI的支援，以將計畫應用到查詢。

有關 [!DNL Query Service]，請參見 [產品文檔](../../query-service/home.md)。

## [!DNL Experience Data Model] (XDM) {#xdm}

標準化和互操作性是其背後的關鍵概念 [!DNL Experience Platform]。 [!DNL Experience Data Model] (XDM)在Adobe的推動下，致力於標準化客戶體驗資料並定義客戶體驗管理模式。

XDM是一種公開記錄的規範，旨在提高數字型驗的威力。 它為任何與Adobe Experience Platform服務通信的應用程式提供了共同的結構和定義。 通過遵守XDM標準，所有客戶體驗資料都可以納入到一個通用的表示形式中，以更快、更整合的方式提供洞察力。 您可以從客戶操作中獲得有價值的見解，通過細分市場定義客戶受眾，並將客戶屬性用於個性化目的。

XDM是允許 [!DNL Experience Cloud]在Adobe Experience Platform的支援下，在正確的時間，在正確的頻道向正確的人傳遞正確的資訊。

方法 [!DNL Experience Platform] 是建的， [!DNL XDM System] 操作 [!DNL Experience Data Model] 方案使用者 [!DNL Experience Platform] 元件。

**新功能**

| 功能 | 說明 |
| ---------- | ------------ |
| JSON架構約束 | 以下資料類型現在在用戶介面中具有其他選項來定義約束： `string`  — 最小/最大長度，模式，預設值（定義如下） [JSON架構draft-6](https://tools.ietf.org/html/draft-wright-json-schema-01)) `double`  — 最小值/最大值，預設值。 |
| 自訂 `$id` | 你現在可以自己提供 `$id` 在POST請求中建立資源時的值。 |
| 架構註冊表效能改進 | 優化聯合模式生成和增強的模式快取，以大大提高API響應時間。 |

**錯誤修正**

* 將identityMap欄位移出上下文/配置檔案並移入其自己的架構欄位組，以使定義標識更直觀。
* 已為基於上下文/配置檔案的所有現有架構打補丁，並帶有上下文/標識映射。
* 未提供版本時出現修復錯誤消息。
* 修復Bug [!DNL Schema Registry] 正在為配置檔案聯合架構調用提供隨機響應。
* 修復了聯合架構未在中顯示正確欄位的錯誤 [!DNL Schema Registry]。
* 修復錯誤，其中偶爾無法使用有效的命名空間建立標識描述符。
* 如果對象使用固定取消引用問題 `properties` 而不是 `allOf`。

**已知問題**

* 無法擴展 [!DNL Platform] — 通過添加欄位提供的欄位組。
* 當從架構組合中刪除欄位組時，不刪除描述符。
* 無法建立無標籤的枚舉欄位。

瞭解有關使用XDM的更多資訊 [!DNL Schema Registry] API和 [!DNL Schema Editor]，請閱讀 [XDM系統文檔](../../xdm/home.md)。

## [!DNL Segmentation Service] {#segmentation}

[!DNL Segmentation Service] 從配置檔案儲存中定義特定的配置檔案子集，並描述區分配置檔案儲存中有價人員組的標準。 段可以基於記錄資料（如人口統計資訊）或代表客戶與您品牌接觸點的時間系列事件。

例如，在以跑鞋為重點的電子郵件活動中，您可以使用過去30天內搜索跑鞋但未完成購買的所有用戶的受眾部分。 另一個示例可能是使用段來目標站點內容，以便它只顯示給屬於獎勵計畫特定層的訪問者。

**新功能**

| 功能 | 說明 |
| -----------| ---------- |
| 相對時間規則 | 您現在可以選擇滾動時間窗口，如14天前、3到5小時前等。 |
| XDM欄位摘要 | 對於左滑軌上的「屬性」，現在可以提供查看底層資料的摘要。 |
| 左軌搜索 | 改進了左滑軌段部分的搜索能力。 |
| eVar友好名 | 改進了對友好名稱的支援，使您能夠更輕鬆地查看在來自Adobe Analytics的自定義事件和維中捕獲到的資訊。 |
| 合併策略支援 | 現在，您可以使用簡單下拉清單選擇要應用於其段定義的合併策略。 |

**錯誤修正**

* 修復了間歇性問題，導致在左滑軌中緩慢載入屬性和事件構建塊。
* 修復了導致估計器返回「NaN」響應的錯誤。
* 修復了某些欄位開啟錯誤規則生成畫布的錯誤。

**已知問題**

* None.

有關詳細資訊，請參見 [分段服務概述](../../segmentation/home.md)。
