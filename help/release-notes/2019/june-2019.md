# Adobe Experience Platform 發行說明

**發行日期：2019 年 6 月 28 日**

Adobe Experience Platform的新功能：

* [[!DNL Data Science Workspace]](#dsw)
* [[!DNL Decisioning Service]](#decisioning)
* [[!DNL Query Service]](#query)

更新現有功能：

* [[!DNL Experience Data Model (XDM)]](#xdm)
* [[!DNL Segmentation Service]](#segmentation)

## [!DNL Data Science Workspace] {#dsw}

Adobe Experience Platform [!DNL Data Science Workspace] 是 [!DNL Experience Platform] 這可讓資料科學家通過構建和操作機器學習模型，從Adobe解決方案和第三方系統的資料和內容無縫地生成見解。 [!DNL Data Science Workspace] 緊密整合於 [!DNL Platform] 並支援端對端資料科學的生命週期，包括探索和準備XDM資料，接著開發並實施模型以自動豐富 [!DNL Real-Time Customer Profile] 以及機器學習深入分析。

**主要功能**

| 功能 | 說明 |
| -----------| ---------- |
| 布建和計算隔離 | 提供專用的計算資源，使資料科學家能夠在內執行不受信任的代碼 [!DNL Experience Platform] 以安全的方式。 |
| 首次使用者體驗 | 包含各種機器學習架構和語言(例如 [!DNL Python]、R、PySpark和Scala [!DNL Spark]. |
| 筆記本 | 為資料科學家/由提供技術支援的資料工程師提供的定製環境 [!DNL Jupyter Notebooks] 讓他們透過已整理的程式庫清單和熱門的機器學習架構，準備資料、擷取功能，以及開發ML模型。 |
| 資料探索 | 無縫存取內嵌至的XDM資料 [!DNL Platform] 整合 [!DNL Platform Data Access] SDK. |
| 資料視覺效果 | 能夠在 [!DNL Jupyter Notebooks] 加速資料準備和功能工程。 |
| 特徵管線 | 適用於Scala/PySpark的API/SDK可部署功能工程管道，將核心XDM資料轉換為功能結構。 |
| 模型製作 | 使資料科學家能夠專注於模型開發而無須實施用於存取資料和計算資源的基礎架構代碼的模板和運行時間。 您可以匯入模型代碼並加以運算，以便從中的資料衍生出前瞻分析 [!DNL Platform]. |
| 企業模型管理 | 支援多租用戶資料模型，以追蹤模型版本和相關的超參數設定，為合作夥伴生態系統提供基礎。 |
| 模型評估 | 在 [!DNL Python], PySpark, R和Scala。 |
| 模型部署 | 能夠比較跨多個實驗執行的評估量度和設定，並以服務形式發佈最佳模型。 |
| 批次計分 | 豐富 [!DNL Real-Time Customer Profile] 有機器學習的深入分析，或以資料集形式寫回 [!DNL Platform] |
| 排程 | 與整合 [!DNL Platform] 協調服務，透過API使用者定義的排程，自動化模型訓練、計分和功能管道。 |

**已知問題**

* 排程和功能管道目前僅透過API提供，並在未來版本中新增UI。

如需詳細資訊，請造訪 [Data Science Workspace概述](../../data-science-workspace/home.md).

## [!DNL Decisioning Service] {#decisioning}

Adobe Experience Platform [!DNL Decisioning Service] 提供以程式設計方式智慧地從指定個人的一組可用選項中選取「下一個最佳體驗」、將其傳遞至任何通道或應用程式，以及執行報告和分析的功能。

預先建立的豐富資料模型可讓您以不受通道限制的方式，執行「下一個最佳優惠方案」決策的使用案例。

**主要功能**

| 功能 | 說明 |
| -----------| ---------- |
| 業務對象儲存庫 | 由JSON結構模型驅動的存放庫可讓開發人員建立、讀取、更新和刪除各種商業物件。 存放庫提供一般用途、具表達力的查詢API，以及結構感知搜尋。 |
| 存放庫容器 | 在業務對象儲存庫內，開發人員可以隔離其對項目、業務或組織單位或項目生命週期階段（例如在開發和整合、測試或即時生產使用中）的關注。 這些隔離稱為儲存庫容器。 |
| 角色與權限 | 使用 [!DNL Admin Console]，組織可以建立和管理設定檔，以依類型、存取操作和容器授予資源的目標存取權。 可以將用戶添加到這些訪問配置檔案中，並從這些策略自動計算有效訪問權限。 |
| 預先建立的選件物件模型 | 不需要先建立資料模型， [!DNL Platform] 開發人員可善用預先建置的JSON結構和關係來建立優惠方案目錄、定義決策規則和限制，以及組合優惠方案集合以供決策。 |
| 根據設定檔和非設定檔資料的決策規則 | 與 [!DNL Real-Time Customer Profile] 可讓開發人員建立運用設定檔資料的決策規則。 決策不僅可使用設定檔屬性，而且還基於設定檔的體驗事件歷史記錄，以及與使用者身分無關的業務實體（例如流量條件、產品詳細目錄）。 任何 [!DNL Experience Data Model] (XDM)實體，其中架構存在於 [!DNL Schema Registry] 可用於決策規則。 規則是第一類實體，可重複用於任何決策選項和活動。 |
| 排名和上限設定 | 對滿足給定用戶的所有資格和其他約束的決策選項進行排名，並選擇最佳選項。 可使用額外的每位使用者和全域限定限制來限制可用選項的曝光量，從而在考慮資源限制和使用者疲勞的情況下實現個人化。 |
| [!DNL Decisioning] REST API | 此 [!DNL Decisioning Service] 可使用簡單的REST API叫用，以取得特定個人的下一個最佳選件。 量度API可用來檢查即時選件主張和限定值。 |
| 將決策事件串流至 [!DNL Data Lake] 和 [!DNL Query Service] | 此 [!DNL Decisioning Service] 自動建立資料集，自動將所有XDM決策事件串流至 [!DNL Data Lake]. 然後，資料集即可供使用 [!DNL Query Service]. |
| 開發人員培訓 | 自助服務選擇加入並提供Adobe I/O相關檔案，包括各種主題的教學課程。 |

**已知問題**

* 選件資料模型不會透過 [!DNL Schema Registry] 因此，只能以有限的方式擴展。 模型架構具有內建結構，可允許附加自訂資料。 未來，您將能擴充基本XDM模型類別，以定義您自己的自訂決策網域。
* 您必須布建Offer Management網域模型，且使用者和整合必須在此產品內容中管理。

## [!DNL Query Service] {#query}

[!DNL Query Service] 提供在Adobe Experience Platform中使用標準SQL查詢資料的功能，以支援許多不同的分析和資料管理使用案例。 這是無伺服器工具，可讓您加入 [!DNL Data Lake] 並將查詢結果擷取為新資料集以用於報表， [!DNL Data Science Workspace]，或擷取至 [!DNL Profile Service].

您可以使用 [!DNL Query Service] 建立資料分析生態系統，建立消費者在不同互動管道中的形象。 這些管道可能包括：

* 銷售點系統
* Web
* 行動
* CRM系統

**主要功能**

| 功能 | 說明 |
| -----------| ---------- |
| 查詢編輯器 | 使用基於Web的工具來編寫、驗證、測試和執行查詢。 它包括一個控制台，用於提供有關查詢執行的詳細資訊以及預覽查詢結果的功能。 |
| 資料集建立 | 在上建立資料集 [!DNL Experience Platform] 通過標準SQL語法。 |
| Adobe定義的函式 | 運用快速鍵功能執行常見工作，例如識別工作階段或設定歸因。 |
| BI工具連接 | 使用在通用BI工具中找到的PostgreSQL(Postgres)驅動程式連接到 [!DNL Query Service] 來建立報表和視覺效果。 支援的工具包括： [!DNL Tableau], [!DNL Power BI]，和 [!DNL Looker]. 在「憑據」頁簽上查找身份驗證資訊。 |
| 資料庫管理工具連接 | Connect [!DNL Aqua Data Studio] 或 [!DNL DB Visualizer] to [!DNL Query Service] 資料探索和資料集建立功能。 [!DNL Query Service] 還支援R Studio的連接。 在「憑據」頁簽上查找身份驗證資訊。 |
| 命令列查詢工具 | 連接PSQL以能夠從命令行運行查詢。 |
| 查詢記錄 | 保留由執行的查詢的歷史記錄 [!DNL Query Service] 並使您能夠查找以前的SQL，以便編輯、執行或從結果中建立資料集。 |
| 查詢排程API | 透過此API排程週期性執行的查詢。 |

**已知問題**

* [!DNL Query Editor] 顯示查詢結果的100行示例。 若要保留完整的結果集，請使用「查詢記錄」中的資料集建立功能。
* 近期版本將新增檢視支援，以及將排程套用至查詢的UI。

如需 [!DNL Query Service]，請參閱 [產品檔案](../../query-service/home.md).

## [!DNL Experience Data Model] (XDM) {#xdm}

標準化和互操作性是背後的關鍵概念 [!DNL Experience Platform]. [!DNL Experience Data Model] (XDM)受Adobe推動，致力標準化客戶體驗資料並定義客戶體驗管理的結構。

XDM是公開記錄的規格，旨在改善數位體驗的強大功能。 它為與Adobe Experience Platform上的服務通訊的任何應用程式提供共同的結構和定義。 遵循XDM標準，所有客戶體驗資料皆可整合至以更快速、更整合的方式提供深入分析的通用表示法中。 您可以從客戶動作中獲得寶貴的深入分析、透過區段定義客戶受眾，以及將客戶屬性用於個人化目的。

XDM是可讓 [!DNL Experience Cloud]，由Adobe Experience Platform提供技術支援，在正確的時間，透過正確的管道將適當的訊息傳送給適當的人員。

方法 [!DNL Experience Platform] 是建的， [!DNL XDM System] 操作 [!DNL Experience Data Model] 供使用的結構 [!DNL Experience Platform] 元件。

**新功能**

| 功能 | 說明 |
| ---------- | ------------ |
| JSON結構限制 | 下列資料類型現在在使用者介面中有其他選項可定義限制： `string`  — 最小/最大長度，模式，預設值，格式(定義於 [JSON結構描述草稿–6](https://tools.ietf.org/html/draft-wright-json-schema-01))和 `double`  — 最小值/最大值，預設值。 |
| 自訂 `$id` | 您現在可以提供自己的 `$id` 值。 |
| 架構註冊表效能改善 | 最佳化聯合架構產生，並增強架構快取，大幅改善API回應時間。 |

**錯誤修正**

* 將identityMap欄位移出內容/設定檔，並移入其專屬的架構欄位群組，讓定義身分的方式更符合使用者的直覺。
* 已使用context/identitymap，根據內容/設定檔修補所有現有結構。
* 修正未提供版本時的錯誤訊息。
* 修正 [!DNL Schema Registry] 正在為設定檔聯合結構呼叫提供隨機回應。
* 修正聯合結構未顯示正確欄位的錯誤，位於 [!DNL Schema Registry].
* 修正有時候無法以有效命名空間建立身分描述元的錯誤。
* 修正物件若使用 `properties` 而非 `allOf`.

**已知問題**

* 無法擴展 [!DNL Platform] — 通過添加欄位提供的欄位組。
* 從架構組合中刪除欄位組時，不會刪除描述符。
* 無法建立沒有標籤的枚舉欄位。

若要進一步了解如何使用XDM，請使用 [!DNL Schema Registry] API和 [!DNL Schema Editor]，請閱讀 [XDM系統檔案](../../xdm/home.md).

## [!DNL Segmentation Service] {#segmentation}

[!DNL Segmentation Service] 會從您的設定檔存放區定義特定的設定檔子集，說明區分設定檔存放區中可行銷之人員群組的條件。 區段可以根據記錄資料（例如人口統計資訊）或時間系列事件，代表客戶與您品牌的接觸點。

例如，在以跑鞋為重點的電子郵件促銷活動中，您可以使用過去30天內搜尋跑鞋但未完成購買之所有使用者的對象區段。 另一個範例可能是使用區段來定位網站內容，使其只顯示給屬於您獎勵方案特定階層的訪客。

**新功能**

| 功能 | 說明 |
| -----------| ---------- |
| 相對時間規則 | 您現在可以選擇滾動時間窗口，例如14天前、3到5小時前等。 |
| XDM欄位摘要 | 若為左側邊欄的「屬性」，現在提供摘要，供您檢視基礎資料。 |
| 左側邊欄搜尋 | 改善左側邊欄區段部分的搜尋功能。 |
| eVar好記名稱 | 改善對易記名稱的支援，讓您更輕鬆查看自訂事件和Adobe Analytics維度中擷取的資訊。 |
| 合併策略支援 | 您現在可以使用簡單的下拉式清單，選擇要套用至其區段定義的合併原則。 |

**錯誤修正**

* 修正造成左側邊欄中屬性和事件建立區塊載入緩慢的間歇性問題。
* 修正導致估算程式傳回「NaN」回應的錯誤。
* 修正某些欄位開啟錯誤規則建立畫布的錯誤。

**已知問題**

* None.

如需詳細資訊，請參閱 [區段服務概觀](../../segmentation/home.md).
