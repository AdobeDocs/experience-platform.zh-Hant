# Adobe Experience Platform 術語 {#adobe-experience-platform-glossary}

## A

**存取控制**：角色型存取控制可讓管理員將存取權和許可權指派給Experience Platform的使用者。 許可權包括檢視和/或使用Experience Platform功能，例如建立沙箱、定義結構描述和管理資料集的能力。

**存取金鑰ID**：存取金鑰ID是與 [!DNL Amazon] S3秘密存取金鑰。 存取金鑰ID和秘密存取金鑰會一起用來簽署 [!DNL Amazon Web Services] (AWS)要求。

**動作**：在標籤的內容中，動作是特定型別的規則元件，可定義在事件發生以及評估和傳遞條件後應該發生的事情。

**啟動**：啟動是使用者為了將區段或設定檔對應到目的地而採取的動作，例如 [!DNL Oracle Eloqua]， [!DNL Google]，或 [!DNL Salesforce Marketing Cloud].

**活動**：在 [!DNL Offer Decisioning]，活動包含通知優惠方案選擇的邏輯。

**管理員**：貴組織中可設定和自訂Adobe Admin Console中Experience Platform許可權的一或多名個人。

**Adobe Admin Console**： Adobe Admin Console是管理貴組織Adobe產品權益和存取許可權的集中位置。 透過主控台，管理員可以授予使用者群組各種Platform功能的存取許可權，例如「管理資料集」、「檢視資料集」或「管理設定檔」。

**Adobe Experience Platform**： Adobe Experience Platform可標準化整個企業的資料和內容、提供即時消費者個人檔案、運用資料科學，以及加速內容速度，藉此推動客戶歷程中的體驗個人化。

**Adobe Experience Platform查詢服務**：讓資料分析師查詢事件和設定檔，以便用於分析和機器學習。 透過查詢服務，資料科學家和分析人員可以提取他們儲存在Experience Platform中的所有資料集(包括行為資料以及銷售點(POS)、客戶關係管理(CRM)等)，並查詢這些資料集以回答有關資料的特定問題。

**Adobe Experience Platform Segmentation Service**：可讓您從即時客戶設定檔資料建立區段及產生對象。 然後，這些對象可以匯出至他們在資料湖中的資料集。

**Adobe智慧型服務**：Attribution AI和Customer AI等智慧型服務是機器學習、人工智慧為基礎的模型，專門建置並需要Experience Platform才能執行和運作。

**Adobe I/O**：Adobe I/O是Experience Platform的一部分，可讓開發人員存取整合、擴充及自訂Platform所需的一切，包括API、事件、開發人員控制檯及實用工具。

**Adobe Sensei**：Adobe Sensei是支援Experience Platform的智慧架構。 此外，也提供了一組AI服務，讓品牌得以增強其能力，提供即時、個人化的客戶體驗。

**Amazon S3貯體**： [!DNL Amazon S3] 貯體是儲存在中的資料的基本容器。 [!DNL Amazon] 生態系統。 貯體包含物件，每個物件都是使用唯一的開發人員指派的索引鍵來儲存和擷取。

**Amazon S3聯結器**：此 [!DNL Amazon] S3聯結器可讓Experience Platform客戶安全地連線並存取其 [!DNL Amazon] S3資料。

**APA**：此 [[!DNL Australia Privacy Act (Privacy Act)]](https://www.oaic.gov.au/privacy/the-privacy-act) 促進及保護個人隱私，並規範澳洲政府機構及組織處理個人資訊的方式。 此 [!DNL Privacy Act] 包含適用於私營部門組織的原則。 例如，個人有權瞭解為何收集個人資訊以及如何加以使用，以及存取、清除其資料和更正個人資訊的能力。

**附加儲存策略**：「附加」儲存策略是指定透過連線擷取的協力廠商資料，並在資料集結尾附加任何新資料或列時使用的選項。 先前擷取的列保持不變，只有自上次排程執行以來建立的列才會擷取到Experience Platform。 在Experience Platform時，在來源系統中變更的任何列都會保持不變。

**陣列**：陣列用於具有相同資料型別的已排序元素。

**人工智慧**：人工智慧是電腦系統的理論和開發，能夠執行通常需要人類智慧的工作，例如視覺感知、語音識別、決策和語言之間的翻譯。

**屬性**：屬性是指表示設定檔的指定特性。

**屬性合併**：使用即時客戶設定檔API定義合併原則時， `attributeMerge` 物件會指出當資料衝突時，合併原則優先處理設定檔屬性的方式。 這等同於選取 [!UICONTROL 合併方法] 在平台UI中定義合併原則時。

**Attribution AI**： [!DNL Attribution AI] 是一項由Adobe Sensei提供支援的智慧型服務，可在整個客戶生命週期中提供演演算法多管道歸因功能。

**對象**：受眾是符合區段定義准則的結果設定檔集。

**對象人數**：對象人數是符合區段定義條件並符合對象成員資格的設定檔總數。

**對象快照**：對象快照會擷取細分時符合區段條件的所有設定檔。

## B

**回填**：對於已排程的來源，回填選項可擷取歷史資料。

**回填期間**：回填期間可讓您選擇透過來源連線來設定擷取協力廠商歷史資料的時間長度。 選取「永久」的回填期間將會擷取來源資料的整個歷程記錄以供Experience Platform。

**批次**：批次是一段時間內收集並作為單一單元一起處理的一組資料。 資料集是由多個批次組成。

**批次識別碼**：批次ID是Adobe為批次資料產生的識別碼。

**批次擷取**：批次內嵌可讓您將資料以批次檔案的形式內嵌到Experience Platform中。 批次是多個資料單位，由一或多個要作為一個單位進行內嵌的檔案所組成。

**批次細分**：批次細分是持續資料選擇流程的替代方法，並會透過區段定義一次移動所有設定檔資料以產生相應的受眾。 建立後，會儲存並儲存此區段，以便將其匯出以供使用。

**建置**：在標籤的內容中，組建是一個檔案或一組檔案，其中包含執行程式庫內包含的業務邏輯所需的所有設定和程式碼，可讓您將該程式庫部署在網站或行動應用程式上。

**商業智慧工具**：商業智慧(BI)工具主要與整合 [!DNL Experience Platform Query Service]. BI工具是一種應用程式軟體，可收集和處理來自內部和外部系統的大量非結構化資料。

## C

**上限**：在 [!DNL Offer Decisioning]，上限（也稱為頻率上限）會用於決策規則，以定義選件的顯示次數。 有兩種上限：可在合併的目標受眾中建議多少次選件（稱為「全域上限」），以及可向相同一般使用者建議多少次選件（稱為「設定檔上限」）。

**目錄**：在來源和目的地的內容中，目錄是具備與Adobe應用程式和協力廠商技術可用連線的相簿。 不要混淆 [!DNL Catalog Service].

**[!DNL Catalog Service]**： [!DNL Catalog Service] (有時稱為 [!DNL Catalog])是Adobe Experience Platform中資料位置和譜系的記錄系統。 雖然所有內嵌至Experience Platform的資料都會以檔案和目錄的形式儲存在Data Lake中， [!DNL Catalog] 儲存這些檔案和目錄的中繼資料和說明，用於查詢、監控和資料治理。

**CCPA**：此 [[!DNL California Consumer Privacy Act (CCPA)]](https://oag.ca.gov/privacy/ccpa) 加強美國加州居民的隱私權和消費者保護。 CCPA為加州居民提供了新的資料隱私權，包括存取和刪除其個人資料的權利、知悉其個人資料是否遭到販售或揭露（以及對象是誰）的權利，以及選擇退出將其資料販售予第三方的權利。

**類別**：在Experience Data Model (XDM)中，類別會定義用於建立結構描述的最小欄位集，並定義結構描述所代表的商業物件的基本行為。

**使用者端**：使用者端是連線的外部工具或應用程式 [!DNL Query Service] 透過 [!DNL PostgreSQL] 通訊協定或HTTP API。

**集合**：在 [!DNL Offer Decisioning]，集合是優惠方案的子集，根據行銷人員定義的預先定義條件，例如優惠方案類別。

**與PII行銷動作結合**：結合任何個人識別資訊(PII)與匿名資料的行銷動作。 來自廣告網路、廣告伺服器和協力廠商資料提供者的資料合約通常包含明確的合約禁令，禁止將此類資料與可直接識別的資料一起使用。

**命令列介面**：命令列介面是可用來連線的文字型工具 [!DNL Query Service] 用於原始查詢執行。

**組合**：組成是一組共同組成結構的元件。

**條件**：在標籤的內容中，條件為規則元件，會評估必須傳回的邏輯陳述式 `true` 或 `false`. 所有條件都必須評估為 `true` 且所有例外條件都必須評估為 `false` 執行規則上的任何動作之前。

**主控台**：在 [!DNL Query Service]，主控台會提供有關查詢狀態和操作的資訊。 主控台顯示連線至 [!DNL Query Service]、正在執行的查詢作業，以及因這些查詢而產生的任何錯誤訊息。

**合約(「C」)標籤**：合約(「C」)資料使用標籤可用來對具有合約義務或與貴組織的資料控管原則相關的資料進行分類。

**CPRA**：此 [[!DNL California Consumer Privacy Rights Act (CPRA)]](https://cppa.ca.gov/regulations/consumer_privacy_act.html) 展開並修正部分 [!DNL California Consumer Privacy Act (CCPA)]. 此 [!DNL CPRA] 提升消費者權益，並透過更廣義的敏感個人資訊定義，擴大資料型別，確立加州消費者資料隱私權的新基準。 此外， [!DNL CPRA] 成立加州隱私保護局，專責實作及執行資料隱私權規則的新機構。

**C1合約標籤**：A `C1` 合約資料使用標籤指定資料只能以彙整形式從Adobe Experience Cloud匯出，而不包含個人或裝置識別碼。 例如，源自社交網路的資料。

**C2合約標籤**：A `C2` 合約資料使用標籤會指定無法匯出至協力廠商的資料。 有些資料提供者的合約中有條款，禁止匯出原本收集的資料。 例如，社交網路合約通常會限制您從合約中接收的資料傳輸。 C2比C1更嚴格，C1只需要彙總和匿名資料。

**C3合約標籤**：A `C3` 合約資料使用標籤會指定無法結合或搭配直接可識別資訊使用的資料。 有些資料提供者在其合約中訂有條款，禁止將該資料與可直接識別的資訊合併或使用。 例如，來自廣告網路、廣告伺服器和第三方資料提供者的資料合約通常包含明確的合約禁令，禁止使用可直接識別的資料。

**C4合約標籤**：A `C4` 合約資料使用標籤指定資料不可用於任何廣告或內容的目標定位，無論是在網站上還是跨網站。 C4是最嚴格的標籤，因為它包含C5、C6和C7標籤。

**C5合約標籤**：A `C5` 合約資料使用標籤指定資料不可用於基於興趣的內容或廣告的跨網站目標定位。 如果符合下列三個條件，就會發生以興趣為基礎的目標定位或個人化：網站所收集的資料用於推斷使用者的興趣；用於其他情境中，例如其他網站或應用程式上；並用於根據這些推斷選取提供哪些內容或廣告。

**C6合約標籤**：A `C6` 合約資料使用標籤指定資料不能用於網站上的廣告目標定位。 站上廣告目標定位包括在您組織的網站或應用程式上選擇和投放廣告，或衡量此類廣告的投放和有效性。 這包括使用先前收集到的網站上的資料，瞭解使用者對選擇廣告的興趣、處理有關顯示哪些廣告、何時在何處顯示，以及使用者是否採取任何與廣告相關的動作，例如選擇廣告或購買。

**C7合約標籤**：A `C7` 合約資料使用標籤指定資料不能用於網站上的內容目標定位。 網站上的內容鎖定目標包括選擇和傳送您組織網站或應用程式上的內容，或測量此類內容的傳送和有效性。 這包括先前收集的使用者對選取內容之興趣的相關資訊、處理顯示的內容、顯示的頻率或時間長度、顯示時間和位置，以及使用者是否採取與內容相關的任何動作，例如選取內容。

**C8合約標籤**：A `C8` 合約資料使用標籤指定資料不可用於測量貴組織的網站或應用程式。 這不包括以興趣為基礎的鎖定目標，這是您使用此服務來後續個人化內容和/或其他內容中的廣告的相關資訊集合。

**C9合約標籤**：A `C9` 合約資料使用標籤指定資料不能用於資料科學工作流程。 有些合約包含明確禁令，禁止資料科學所用的資料。 有時候，這些措辭會禁止將資料用於人工智慧(AI)、機器學習(ML)或模型。

**C10合約標籤**：A `C10` 合約資料使用標籤指定資料不能用於拼接身分啟用。 有些資料使用原則會限制將拼接的身分資料用於個人化。 此 `C10` 如果區段的合併原則使用「私密圖表」選項，系統會自動將標籤套用至區段。

**「建立日期」欄**：透過來源連線指定協力廠商資料時，選取建立日期欄是一個選項。 當選取附加儲存策略且資料集結構描述包含多個日期欄位時，您必須從可用的結構描述中選擇以指定「建立日期」索引鍵欄。 選取覆寫儲存策略時，「建立日期」選項無法使用。

**建立表格為選取**：建立表格為選取(CTAS)是一項SQL命令，當作為完整且有效的SQL查詢的一部分執行時，會指示執行此命令 [!DNL Query Service] 將查詢結果保留在資料集中。 您可以建立新的結果集、覆寫先前的結果或附加至先前的結果。

**跨網站資料**：跨網站資料是來自多個網站的資料組合，包括站上資料和站外資料的組合，或來自多個站外來源的資料組合。

**跨網站目標定位行銷動作**：此行銷動作會將資料用於跨網站廣告目標定位。 來自多個網站的資料組合（包括站上資料和站外資料的組合，或來自多個站外來源的資料組合）稱為跨網站資料。 通常會收集和處理跨網站資料，以推斷客戶的興趣。

**自訂身分名稱空間**：您的組織可以建立自訂身分名稱空間，以代表特定組織或業務案例的身分。

**自訂標籤**：自訂資料使用標籤可讓您建立特定標籤，並將其套用至符合特定業務需求的資料欄位。

**Customer AI**： Customer AI是一項由Adobe Sensei提供支援的智慧型服務，它透過AI傾向豐富了客戶設定檔，並強化了客戶細分和目標定位工作。

## D

**每日**：在排程檔案匯出的內容中，排程每天從開始日期到結束日期在使用者指定的時間匯出一次完整或增量檔案匯出。

**資料字典**：在標籤的內容中，資料字典（也稱為資料對應）是屬性內定義的一組資料元素。

**資料元素**：在標籤的內容中，資料元素是規則和擴充功能中使用的指標，可指向使用者端裝置上存在的特定資料片段。

**資料擷取**：資料擷取是將資料從來源新增至Experience Platform的程式。 資料可透過多種方式內嵌至Platform，包括串流、批次，或透過來源聯結器新增。

**資料層**：在標籤的內容中，資料層是存在於使用者端裝置上的資料結構，包含檢視頁面或畫面之內容的相關中繼資料。

**資料控管**：資料控管涵蓋了用於確保資料在資料使用方面符合法規和組織政策的策略和技術。

**資料整合合作夥伴**：資料整合合作夥伴可簡化並自動化從200多個來源的大量資料載入和轉換作業，以進行Experience Platform，而不需撰寫程式碼。

**資料集標籤**：資料使用標籤可以新增到資料集。 該資料集內的所有欄位將會繼承資料集的標籤。

**資料科學工作區**： [!DNL Data Science Workspace] 透過Experience Platform，客戶可利用Platform和Adobe應用程式的資料，建立機器學習模型，以建立智慧型區段、產生見解並提供預測，讓您大幅提升一般使用者的數位體驗。

**資料來源**：資料來源是使用者指定的資料來源。 資料來源的範例包括行動應用程式、設定檔及/或體驗事件、網站設定檔事件或CRM。

**資料管理員**：資料管理員是負責管理、監督及執行組織資料資產的人員。 資料管理員也可確保資料治理政策受到保障並保持符合政府法規和組織政策。

**資料串流**：資料串流是共用相同結構描述且由相同來源傳送的訊息集或集合。

**資料型別**：資料型別是可重複使用的XDM資源，其定義在階層式表示中包含多個屬性的物件型別欄位。

**資料流**：資料流是從來源流入Platform並流出到目的地的虛擬資料管道。

**資料流執行**：資料流執行是根據使用者指定排程進入Experience Platform的資料流。

**資料集**：資料集是資料集合的儲存和管理結構，通常是包含結構（欄）和欄位（列）的表格。

**資料集ID**：內嵌資料集的Adobe產生識別碼。

**資料集輸出**：資料集輸出提供一種機制，用於決定哪個「建立表格為選取」選項將用於特定 [!DNL Query Service] 執行。

**重複資料刪除索引鍵**：使用者定義的主索引鍵，可決定使用者要透過哪種身分來為其設定檔進行重複資料刪除&#x200B;。

**目的地**：目的地是任何端點的一般辭彙，例如Adobe應用程式、廣告平台、雲端儲存服務或行銷服務，其中會啟用和傳送對象。

**目的地類別**：目的地類別是指一組具有類似特性的目的地。

**目的地目錄**：目的地目錄是Experience Platform中可用目的地的清單。

**直接呼叫規則**：在標籤的內容中，直接呼叫規則是直接從頁面呼叫時會執行的規則，略過事件偵測和查詢系統。
