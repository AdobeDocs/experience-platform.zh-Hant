# Adobe Experience Platform 術語 {#adobe-experience-platform-glossary}

## A

**存取控制**:角色型存取控制可讓管理員將存取權和權限指派給Experience Platform的使用者。 權限包括檢視和/或使用Experience Platform功能，例如建立沙箱、定義結構描述及管理資料集。

**訪問密鑰ID**:存取金鑰ID是與 [!DNL Amazon] S3機密存取金鑰。 訪問密鑰ID和秘密訪問密鑰用於簽名 [!DNL Amazon Web Services] (AWS)要求。

**動作**:在標籤的內容中，動作是特定類型的規則元件，可定義事件發生後應發生的事件，以及評估和傳遞條件。

**啟動**:「啟動」是使用者為將區段或設定檔對應至目的地(例如 [!DNL Oracle Eloqua], [!DNL Google]，或 [!DNL Salesforce Marketing Cloud].

**活動**:在 [!DNL Offer Decisioning]，活動包含通知選件選取的邏輯。

**管理員**:貴組織中的一或多個個人，可以設定和自訂在Adobe Admin Console中Experience Platform的權限。

**Adobe Admin Console**:Adobe Admin Console提供管理Adobe產品權益和組織存取權限的集中位置。 透過Console，管理員可以針對各種Platform功能（例如「管理資料集」、「檢視資料集」或「管理設定檔」），授與使用者群組的存取權限。

**Adobe Experience Platform**:Adobe Experience Platform可標準化整個企業的資料和內容、提供即時的消費者個人檔案、運用資料科學，以及加快內容速度，以推動客戶歷程中的體驗個人化。

**Adobe Experience Platform查詢服務**:可讓資料分析人員查詢事件和設定檔，以用於分析和機器學習。 透過Query Service，資料科學家和分析師可以提取儲存在Experience Platform中的所有資料集(包括行為資料、銷售點(POS)、客戶關係管理(CRM)等)，並查詢這些資料集以回答有關資料的特定問題。

**Adobe Experience Platform區段服務**:可讓您建立區段，並從即時客戶設定檔資料產生受眾。 然後，這些對象便可匯出至Data Lake中自己的資料集。

**AdobeIntelligent Services**:Attribution AI和客戶人工智慧等智慧服務是機器學習、基於人工智慧的模型，專門構建，需要Experience Platform運行和操作。

**Adobe I/O**:Adobe I/O是Experience Platform的一部分，可讓開發人員存取整合、擴充及自訂平台所需的一切，包括API、事件、開發人員主控台和實用工具。

**Adobe Sensei**:Adobe Sensei是支援Experience Platform的情報框架。 它還提供一組AI服務，讓品牌能夠增強其提供即時個人化客戶體驗的能力。

**Amazon S3貯體**: [!DNL Amazon S3] 貯體是儲存在 [!DNL Amazon] 生態系統。 貯體包含物件，系統會使用開發人員指派的唯一金鑰來儲存及擷取每個物件。

**Amazon S3連接器**:此 [!DNL Amazon] S3連接器可讓Experience Platform的客戶安全地連線及存取其 [!DNL Amazon] S3資料。

**附加儲存策略**:「附加」儲存策略是指定第三方資料以透過連線內嵌，以及在資料集結尾附加任何新資料或列時使用的選項。 先前擷取的列不會受到影響，且只會擷取自上次排程執行後建立的列至Experience Platform。 源系統中更改的任何行在Experience Platform時保持不變。

**陣列**:陣列用於具有相同資料類型的有序元素。

**人工智慧**:人工智慧是電腦系統的理論和發展，能夠執行通常需要人類智慧的任務，如視覺感知、語音識別、決策和語言之間的翻譯。

**屬性**:屬性是代表設定檔的指定特徵。

**屬性合併**:使用即時客戶個人檔案API定義合併原則時， `attributeMerge` 對象指示在發生資料衝突時合併策略優先處理配置檔案屬性的方式。 這等同於選取 [!UICONTROL 合併方法] 在Platform UI中定義合併原則時。

**Attribution AI**: [!DNL Attribution AI] 是採用Adobe Sensei技術的智慧型服務，可在整個客戶生命週期中提供演算法多通道歸因功能。

**對象**:對象是產生的一組符合區段定義條件的設定檔。

**對象大小**:對象大小是符合區段定義條件且符合對象成員資格資格的設定檔總數。

**受眾快照**:受眾快照會擷取在細分時符合區段條件的所有設定檔。

## B

**回填**:對於排程的來源，回填選項可啟用歷史資料的擷取。

**回填期間**:回填期間是可設定透過來源連線擷取第三方歷史資料的時間長度的選項。 選擇「forever」的回填期間，會將源資料的整個歷史記錄內嵌到Experience Platform中。

**批次**:批次是一段時間內收集並以單一單位一起處理的一組資料。 資料集由多個批次組成。

**批次ID**:批次ID是Adobe產生的資料批次識別碼。

**批次內嵌**:批次內嵌可讓您將資料以批次檔案的形式內嵌至Experience Platform。 批次是多個資料單位，由一或多個要作為一個單位進行內嵌的檔案所組成。

**批次分段**:批次分段是持續資料選取程式的替代方法，可一次透過區段定義移動所有設定檔資料，以產生對應的對象。 此區段一經建立便會儲存並儲存，以便匯出供使用。

**建置**:在標籤內容中，組建是檔案或一組檔案，其中包含執行程式庫內包含之業務邏輯所需的所有設定和程式碼，可讓您將該程式庫部署在網站或行動應用程式上。

**商業智慧工具**:商業智慧(BI)工具主要與 [!DNL Experience Platform Query Service]. BI工具是一種應用程式軟體類型，用於收集和處理來自內部和外部系統的大量非結構化資料。

## C

**限定**:在 [!DNL Offer Decisioning]，決策規則會使用限定（也稱為頻率限定）來定義顯示選件的次數。 帽子有兩種類型：可在合併的目標對象中建議某個選件的次數（稱為「全域上限」），以及可向相同一般使用者建議某個選件的次數（稱為「設定檔上限」）。

**目錄**:在來源和目的地的內容中，目錄是一個圖庫，可與Adobe應用程式和第三方技術建立連線。 不要與 [!DNL Catalog Service].

**[!DNL Catalog Service]**: [!DNL Catalog Service] (有時稱為 [!DNL Catalog])是Adobe Experience Platform內用於資料位置和歷程的記錄系統。 雖然所有擷取到Experience Platform中的資料都會以檔案和目錄的形式儲存在資料湖中， [!DNL Catalog] 保存這些檔案和目錄的元資料和說明，以用於查閱、監控和資料控管。

**類別**:在Experience Data Model(XDM)中，類別會定義用來建立結構的最小欄位集，並定義結構所代表之商業物件的基本行為。

**用戶端**:客戶端是連接到的外部工具或應用程式 [!DNL Query Service] 通過PostgreSQL協定或HTTP API。

**集合**:在 [!DNL Offer Decisioning]，集合是根據行銷人員定義之預先定義條件（例如優惠方案的類別）之選件的子集。

**結合PII行銷動作**:將任何個人識別資訊(PII)與匿名資料結合的行銷動作。 來自廣告網路、廣告伺服器和第三方資料提供者的資料合約通常包含特定合約禁止將這類資料與可直接識別的資料搭配使用。

**命令列介面**:命令列介面是文字型工具，可用來連線至 [!DNL Query Service] 用於原始查詢執行。

**組合物**:組合是組成架構的一組元件。

**條件**:在標籤的內容中，條件是評估必須傳回之邏輯陳述式的規則元件 `true` 或 `false`. 所有條件都必須評估為 `true` 所有例外條件都必須評估 `false` 執行規則上的任何動作之前。

**主控台**:在 [!DNL Query Service]，控制台會提供查詢狀態和操作的相關資訊。 主控台會顯示的連線狀態 [!DNL Query Service]、正在執行的查詢操作，以及由這些查詢產生的任何錯誤訊息。

**合同(&quot;C&quot;)標籤**:合約(「C」)資料使用標籤可用來分類有合約義務或與客戶資料控管政策相關的資料。

**C1合同標籤**:A `C1` 合約資料使用標籤指定資料只能以匯總形式從Adobe Experience Cloud匯出，不含個別或裝置識別碼。 例如，源自社交網路的資料。

**C2合同標籤**:A `C2` 合約資料使用標籤會指定無法匯出至協力廠商的資料。 有些資料提供者在其合約中訂有條款，禁止從最初收集到的資料的地方導出資料。 例如，社交網路合約通常會限制您從這些合約收到的資料傳輸。 C2比C1更嚴格，後者僅需要聚合和匿名資料。

**C3合同標籤**:A `C3` 「合約資料使用方式」標籤指定無法結合或與可直接識別資訊搭配使用的資料。 有些資料提供者在其合約中訂有條款，禁止將該資料與可直接識別資訊合併或使用。 例如，來自廣告網路、廣告伺服器和第三方資料提供者的資料合約通常包含使用可直接識別資料的特定合約禁止。

**C4合同標籤**:A `C4` 合約資料使用標籤會指定資料無法用於定位任何廣告或內容，無論是站上或跨網站皆然。 C4是限制最嚴格的標籤，因為它包含C5、C6和C7標籤。

**C5合同標籤**:A `C5` 合約資料使用標籤會指定資料無法用於興趣內容或廣告的跨網站定位。 如果符合下列三個條件，就會發生興趣型鎖定目標或個人化：現場收集的資料用於推斷用戶的興趣；用於其他內容，例如其他網站或應用程式；和可用來根據這些參照來選取提供哪些內容或廣告。

**C6合同標籤**:A `C6` 「合約資料使用量」標籤會指定資料無法用於站上廣告鎖定目標。 站上廣告定位包括在您組織的網站或應用程式上選擇和投放廣告，或評估此類廣告的投放和效果。 這包括使用先前收集的關於使用者興趣的站上資料來選取廣告、處理關於顯示哪些廣告、顯示時間和位置的資料，以及使用者是否採取了與廣告相關的動作，例如選取廣告或購買。

**C7合同標籤**:A `C7` 「合約資料使用量」標籤會指定資料無法用於內容的站上定位。 現場內容目標定位包括選擇和傳遞您組織網站或應用程式上的內容，或測量此類內容的傳遞和有效性。 這包括先前收集的關於使用者對選取內容的興趣、處理關於顯示內容、顯示頻率或時間長度、顯示時間和位置的資料，以及使用者是否採取了與內容相關的任何動作（包括選取內容）的資訊。

**C8合同標籤**:A `C8` 合約資料使用標籤會指定資料無法用於貴組織網站或應用程式的測量。 這不包括以興趣為基礎的鎖定目標，這是關於您使用此服務後續個人化其他內容和/或廣告的資訊的集合。

**C9合同標籤**:A `C9` 「合約資料使用情況」標籤指定資料無法用於資料科學工作流程。 有些合同明確禁止資料科學所使用的資料。 有時，這些措辭的措辭禁止將資料用於人工智慧(AI)、機器學習(ML)或建模。

**C10合同標籤**:A `C10` 「合約資料使用情況」標籤指定資料無法用於匯整身分啟用。 有些資料使用原則會限制使用連結的身分資料來進行個人化。 此 `C10` 如果區段的合併原則使用「專用圖表」選項，則會自動套用標籤。

**建立日期欄**:透過來源連線指定第三方資料時，選取「建立日期」欄是選項。 選取附加儲存策略且資料集架構包含多個日期欄位時，您必須從可用架構中選擇，以指定「建立日期」索引鍵欄。 選擇覆蓋保存策略時，「建立日期」選項不可用。

**將表建立為選擇**:將表建立為選擇(CTAS)是SQL命令，在作為完整且有效的SQL查詢的一部分執行時，它將指示 [!DNL Query Service] 將查詢結果保留在資料集中。 您可以建立新的結果集、覆蓋以前的結果或附加到以前的結果。

**跨網站資料**:跨網站資料是來自數個網站的資料組合，包括站上資料和離站資料的組合，或來自數個離站來源的資料組合。

**跨網站目標定位行銷動作**:使用資料進行跨網站廣告鎖定的行銷動作。 來自多個網站的資料組合（包括站上資料和離站資料的組合，或來自多個離站來源的資料組合）稱為跨網站資料。 系統通常會收集並處理跨網站資料，以便對客戶的興趣做出推論。

**自訂身分命名空間**:您的組織可建立自訂身分識別命名空間，以代表特定組織或業務案例的身分識別。

**自訂標籤**:自訂資料使用量標籤可讓您建立特定標籤，並套用至符合特定業務需求的資料欄位。

**Customer AI**:Customer AI是Adobe Sensei提供技術支援的智慧型服務，能運用AI傾向豐富客戶個人檔案，並執行客戶細分和目標定位工作。

## D

**每日**:在排程檔案匯出的內容中，排程完整或增量檔案匯出每天一次，從開始日期到使用者指定時間的結束日期。

**資料字典**:在標籤的內容中，資料字典（也稱為資料地圖）是在屬性中定義的一組資料元素。

**資料元素**:在標籤內容中，資料元素是規則和擴充功能內使用的指標，用於指向用戶端裝置上存在的特定資料片段。

**資料擷取**:資料擷取是將資料從來源新增至Experience Platform的程式。 資料可透過多種方式擷取至Platform，包括串流、批次或透過來源連接器新增。

**資料層**:在標籤的內容中，資料層是存在於用戶端裝置上的資料結構，包含與檢視頁面或畫面的內容相關的中繼資料。

**資料控管**:資料控管涵蓋的策略與技術，可確保資料符合與資料使用相關的法規與組織政策。

**資料整合合作夥伴**:資料整合合作夥伴可簡化並自動化從200多個來源到Experience Platform的大量資料的載入和轉換，無需編寫代碼。

**資料集標籤**:資料使用量標籤可新增至資料集。 該資料集內的所有欄位都會繼承資料集的標籤。

**Data Science Workspace**: [!DNL Data Science Workspace] Experience Platform內可讓客戶運用跨平台和Adobe應用程式的資料建立機器學習模型，以建立智慧區段、產生深入分析和提供預測，讓您大幅提升使用者數位體驗。

**資料來源**:資料來源是使用者指定的資料來源。 資料來源的範例包括行動應用程式、設定檔及/或體驗事件、網站設定檔事件或CRM。

**資料管理員**:資料管理員是負責管理、監督及執行組織資料資產的人員。 資料管理員還可確保資料管理政策受到保護並得到維護，以符合政府法規和組織政策。

**資料流**:資料流是一組或一組共用相同架構且由相同來源傳送的訊息。

**資料類型**:資料類型是可重複使用的XDM資源，它定義一個對象類型欄位，該欄位在層次表示中包含多個屬性。

**資料使用量標籤**:資料使用標籤可讓您分類反映隱私權相關考量事項和符合法規和公司政策的合約條件的資料。 新增至資料集的資料使用量標籤會繼承或套用至該資料集內的所有欄位。 資料使用量標籤也可直接套用至欄位。

**資料流**:資料流是從源流入Platform並流出到目的地的資料的虛擬管道。

**資料流運行**:資料流運行是一種資料流，它根據用戶指定的時間表Experience Platform。

**資料集**:資料集是資料集合（通常為表格）的儲存和管理結構，其中包含結構（欄）和欄位（列）。

**資料集ID**:Adobe產生的擷取資料集識別碼。

**資料集輸出**:資料集輸出提供一種機制，可決定「以選取方式建立表格」選項將用於特定 [!DNL Query Service] 執行。

**重複資料刪除密鑰**:使用者定義的主要金鑰，可決定使用者要依哪個身分來刪除其設定檔的重複&#x200B;性。

**增量列**:差值欄可讓您選取來源資料欄位，以表示增量擷取的時間戳記。

**增量保存策略**:增量儲存策略是透過來源連線擷取第三方資料的選項。 選項可讓使用者指定將新或變更的來源資料列擷取至Experience Platform。 資料集的結尾會新增新列，而變更的列會在Experience Platform時更新資料集。

**描述符**:在Experience Data Model(XDM)中，描述元是描述欄位特定行為的額外結構相關中繼資料集。 Experience Platform可使用描述符來了解預期的架構行為，如兩個架構之間的關係。

**方式**:配方是模型規範的Adobe術語，是表示特定機器學習流程、AI算法、處理邏輯和配置參數的頂級容器，這些參數是構建和執行受訓模型所需的，因此有助於解決特定業務問題。

**記錄**:記錄是資料集中持續存在的列資料。

**記錄資料**:提供主題屬性的相關資訊。 主題可以是組織或個人。

**週期**:在 [!DNL Query Service]，週期會定義查詢排程僅執行一次還是循環執行。

**表示**:在 [!DNL Offer Decisioning]，表示法是管道用來顯示選件的資訊，例如位置或語言。

**資源**:在標籤內容中，資源是一般術語，指的是標籤使用者可在用戶端環境中設定的選項，包括擴充功能、資料元素和規則。

**基於角色的訪問控制**:角色型存取控制可讓管理員將存取權和權限指派給Experience Platform的使用者。 權限包括檢視和/或使用Experience Platform功能，例如建立沙箱、定義結構描述及管理資料集。

**規則**:在標籤內容中，規則是元件的集合，定義應以邏輯分組的特定事件、條件和動作集。

**規則元件**:在標籤的內容中，規則元件是組成規則的事件、條件和動作。

**執行階段**:運行時指定機器學習配方的運行時環境。 [!DNL Python], R, [!DNL Spark]、PySpark和Tensorflow運行時，您可以為配方源輸入Docker影像的URL。

## S

**範例資料**:範例資料是資料檔案的預覽，通常是前100列，讓資料科學家或工程師了解資料檔案中的架構或資料。

**沙箱**:沙箱是一種虛擬結構，可將單一Platform例項分割為個別的虛擬環境，以協助開發及改進數位體驗應用程式。

**沙箱重設**:沙箱重設會刪除沙箱內的所有資料，包括資料、設定檔和區段。 沙箱重設可能會影響連線至內部或外部目的地的資料。

**沙箱切換器**:Experience Platform中的沙箱切換器控制項可讓使用者在可存取的沙箱之間導覽。 切換沙箱會變更所有內容，並可能根據權限變更功能存取。

**排程**:排程是使用者定義的規格，說明從協力廠商資料來源擷取資料至Adobe Experience Platform的頻率或頻度。

**分數**:計分是使用經過訓練的模型，從資料產生深入分析的程式。

**結構**:結構是一組規則，可代表及驗證資料的結構和格式。 結構由類別和選用欄位群組組成，可用來建立資料集和資料流。 結構可以包括行為屬性、時間戳、身份、屬性定義、關係等。

**架構欄位組**:在Experience Data Model(XDM)中，結構欄位群組可讓使用者擴充可重複使用的欄位，以定義要包含在結構中的一或多個屬性。

**結構庫**:結構庫包含由Adobe提供的業界標準XDM資源，以及貴組織定義的自訂資源。

**架構註冊**:架構註冊表提供用於查看和管理架構庫中所有與架構相關的資源的用戶介面和RESTful API。

**秘密訪問密鑰**:機密存取金鑰是 [!DNL Amazon] 與存取鍵值ID搭配使用以簽署AWS請求的S3鍵值。

**區段**:區段是一組規則，包含屬性和事件資料，讓許多設定檔有資格成為受眾。

**區段產生器**:此 [!DNL Segment Builder] 是視覺化開發環境，用於建立區段定義。 它是所有使用Experience Platform細分服務的應用程式的共同元件。

**區段定義**:區段定義是用來說明目標受眾的主要特徵或行為的規則集。 概念化後，區段定義中概述的規則將用於決定區段的合格受眾成員。

**區段評估方法**:區段評估方法有兩種：排程和隨選。 排程評估可啟用在特定時間執行匯出作業的循環排程，而隨需評估則包括建立區段作業以立即建立對象。

**區段匯出**:區段匯出是Experience Platform中兩種目的地之一。 透過區段匯出，您可以傳送符合資格且已對應至目的地的設定檔。 使用區段和使用者ID以及假名資料，且通常會與社交網路及其他數位媒體目標平台整合。

**區段ID**:區段ID是與區段相關聯的自動產生識別碼。

**區段成員資格**:區段成員資格會顯示設定檔目前屬於哪個區段。

**區段規則**:區段規則會定義用以判斷設定檔是否符合區段的條件。

**區段**:區段是將大量客戶、潛在客戶或消費者劃分為具有類似屬性且將對特定行銷策略作出類似回應的較小群組的程式。

**Sensei ML架構**:Sensei ML Framework是統一的機器學習(ML)框架，它利用Experience Platform資料，讓資料科學家能夠以更快、可擴展和可重複使用的方式開發由ML驅動的智慧服務。

**敏感(「S」)標籤**:敏感(「S」)標籤可用來分類被視為敏感的資料，例如您想要標示為敏感的不同行為或地理資料類型。

**服務**:利用Adobe智慧服務實施AI和ML服務的強大框架。 服務提供即時、個人化的客戶體驗，或實施自訂智慧服務。

**單一身分個人化行銷動作**:使用資料進行站上內容個人化的行銷動作。 Onsite個人化是用於推斷使用者興趣的任何資料，用來根據這些推斷選取提供哪些內容或廣告。

**S1資料使用量標籤**:安 `S1` 資料使用標籤可用來分類指定經緯度的資料，這些經緯度可用來判斷裝置的精確位置。

**S2資料使用量標籤**:安 `S2` 資料使用標籤可用來分類可用於判斷廣泛定義地理序列區域的資料。

**來源**:來源是Platform中任何輸入連接器的一般辭彙。 另請參閱：源連接器

**源屬性**:來源屬性是來源資料集中的欄位。 源屬性映射到目標架構欄位。

**源目錄**:源目錄是Experience Platform中可用的源連接器清單。

**來源類別**:源類別是具有相似特徵的源的分組。

**源連接器**:來源連接器（也稱為來源）可協助使用者輕鬆內嵌來自多個來源的資料，以利使用Experience Platform服務來建構、標示及增強資料。 資料可從多種來源擷取，例如雲端儲存、協力廠商軟體和CRM系統。

**串流連線**:串流連線是Adobe提供的唯一端點，並與客戶的IMS組織系結，可將資料串流至Experience Platform。

**標準身分命名空間**:標準身分識別命名空間是由Adobe提供的預先定義身分識別命名空間，代表業界標準解決方案，通常用於識別客戶。

**串流內嵌**:串流內嵌可讓您從用戶端和伺服器端裝置傳送資料以即時Experience Platform。

**串流細分**:串流區段是持續進行的資料選取程式，會根據使用者活動更新區段。 建立並儲存區段後，會對傳入的資料套用區段定義至 [!DNL Real-time Customer Profile]. 會定期處理區段新增和移除，確保目標受眾仍具相關性。

**系統視圖**:「系統視圖」是流經的來源資料集的視覺表示法 [!DNL Real-time Customer Profile] 目的地。

## X

**XDM**:請參閱Experience Data Model(XDM)。

**XDM決策事件**:XDM決策事件是以時間序列為基礎的類別，用來擷取關於決策活動結果和內容的觀察結果。 這包括關於如何作出決定、何時作出決定、提出（和選擇）哪些備選方案，以及在決策過程中存在哪些影響決定或可以觀察到的背景狀態的資訊。

**XDM ExperienceEvent**:XDM ExperienceEvent是以時間序列為基礎的類別，用於在發生事件（或一組事件）時擷取系統狀態，包括相關主體的時間點和身分。 另請參閱：體驗事件

**XDM個別設定檔**:XDM [!DNL Individual Profile] 是基於記錄的類，它對已識別和部分已識別的主題的屬性形成奇異的表示。 高度識別的配置檔案可用於個人通信或目標性參與，並且可以包含詳細的個人資訊，如姓名、性別、出生日期、位置和聯繫資訊，包括電話號碼和電子郵件地址。

**XDM系統**:XDM System代表可運作XDM結構以用於下游Experience Platform服務的架構。
