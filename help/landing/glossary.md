# Adobe Experience Platform 術語 {#adobe-experience-platform-glossary}

## A

**存取控制**:基於角色的存取控制可讓管理員將存取權和權限指派給Experience Platform的使用者。權限包括檢視和／或使用Experience Platform功能的能力，例如建立沙盒、定義結構描述以及管理資料集。

**存取金鑰ID**:存取金鑰ID是與 [!DNL Amazon] S3機密存取金鑰相關聯的唯一識別碼。訪問密鑰ID和秘密訪問密鑰一起用於簽署[!DNL Amazon Web Services](AWS)請求。

**動作**:在中， [!DNL Platform Launch]動作是一種特定類型的規則元件，可定義事件發生後應發生的事件，並評估和傳遞條件。

**啟動**:「啟動」是使用者將區段或描述檔對應至目標(例如、 [!DNL Oracle Eloqua]或 [!DNL Google])的動 [!DNL Salesforce Marketing Cloud]作。

**活動**:在 [!DNL Offer Decisioning]中，活動包含通知選件選擇的邏輯。

**管理員**:您組織中的一或多個個人，可以設定和自訂在Adobe Admin ConsoleExperience Platform的權限。

**Adobe Admin Console**:Adobe Admin Console提供一個集中位置，用於管理Adobe產品權益，並存取您的組織。管理員可以通過控制台授予用戶組對各種平台功能的訪問權限，如「管理資料集」、「查看資料集」或「管理配置檔案」。

**Adobe Experience Platform**:Adobe Experience Platform將整個企業的資料和內容標準化，提供即時的消費者個人檔案，提供資料科學，並加速內容速度，以推動客戶體驗的個人化。

**Adobe Experience Platform Launch**: [!DNL Platform Launch] 是標籤和SDK管理生態系統，與Experience Platform和應用程式 [!DNL Experience Cloud] 整合。[!DNL Platform Launch] 提供部署、統一及管理分析、行銷和廣告整合的工具，以在所有用戶端裝置上提供相關客戶體驗。

**Adobe Experience Platform查詢服務**:可讓資料分析人員查詢事件和設定檔，以便用於分析和機器學習。使用查詢服務，資料科學家和分析師可以提取其儲存在Experience Platform中的所有資料集(包括行為資料和銷售點(POS)、客戶關係管理(CRM)等)，並查詢這些資料集以回答有關資料的特定問題。

**Adobe Experience Platform區段服務**:可讓您建立細分，並從即時客戶個人檔案資料產生受眾。然後，這些觀眾可匯出至資料湖中的專屬資料集。

**Adobe智慧服務**:智慧型服務(例如Attribution AI和客戶人工智慧)是機器學習、以人工智慧為基礎的模型，專門建立並需要Experience Platform來執行和運作。

**Adobe I/O**:Adobe I/O是Experience Platform的一部分，可讓開發人員存取整合、擴充和自訂平台所需的一切，包括API、事件、開發人員主控台和實用工具。

**Adobe Sensei**:Adobe Sensei是支援Experience Platform的情報框架。它還提供一組AI服務，讓品牌能夠增強提供即時、個人化客戶體驗的能力。

**AmazonS3桶**: [!DNL Amazon S3] 儲存桶是儲存在生態系統中的資料的基本 [!DNL Amazon] 容器。儲存桶包含對象，每個對象都使用由開發人員指派的唯一密鑰進行儲存和檢索。

**AmazonS3連接器**:S3連 [!DNL Amazon] 接器可讓Experience Platform客戶安全地連接和存取 [!DNL Amazon] S3資料。

**附加儲存策略**:「附加」儲存策略是指定第三方資料，以透過連線進行內嵌，並在資料集結尾附加任何新資料或列時，所使用的選項。先前收錄的列保持不變，只有自上次排程執行後建立的列會收錄至Experience Platform。 源系統中更改的任何行在Experience Platform上保持不變。

**陣列**:陣列用於具有相同資料類型的有序元素。

**人工智慧**:人工智慧是電腦系統的理論和發展，能夠執行通常需要人類智慧的任務，如視覺感知、語音識別、決策和語言間的翻譯。

**屬性**:屬性是指定的特徵，表示配置檔案。

**屬性合併**:使用即時客戶描述檔API定義合併原則時，物件會 `attributeMerge` 指出在發生資料衝突時，合併原則將依何種方式來排列描述檔屬性的優先順序。它等同於在平台UI中定義合併策略時選擇[!UICONTROL Merge method]。

**Attribution AI**: [!DNL Attribution AI] 是由Adobe Sensei提供的智慧型服務，可在整個客戶生命週期中提供演算法多通道歸因功能。

**觀眾**:對象是符合區段定義條件的設定檔產生集。

**受眾規模**:對象大小是符合區段定義標準且符合對象會籍資格的設定檔總數。

**觀眾快照**:觀眾快照會擷取區段時符合區段條件的所有設定檔。

## B

**回填**:對於排程的來源，回填選項可讓您擷取歷史資料。

**回填期間**:回填期間是一個選項，用於設定通過源連接接收第三方歷史資料的時間長度。選取「永久」的回填期間，會將來源資料的整個歷史記錄收錄至Experience Platform。

**批**:批是一組在一段時間內收集並作為單個單位一起處理的資料。資料集由多個批組成。

**批次ID**:批次ID是Adobe產生的資料批次識別碼。

**批次擷取**:批次擷取可讓您將資料以批次檔案的形式內嵌至Experience Platform。批次是多個資料單位，由一或多個要作為一個單位進行內嵌的檔案所組成。

**批次分段**:批次分段是持續資料選擇程式的替代選擇，並透過區段定義一次移動所有描述檔資料，以產生對應的觀眾。建立後，會儲存此區段，以便匯出以供使用。

**構建**:在中 [!DNL Platform Launch]，組建是包含執行程式庫內所含商業邏輯所需的所有組態和程式碼的檔案或檔案組，可讓您將程式庫部署在您的網站或行動應用程式上。

**商業智慧工具**:商業智慧(BI)工具主要與整合 [!DNL Experience Platform Query Service]。BI工具是一種應用程式軟體類型，可從內部和外部系統收集和處理大量非結構化資料。

## C

**上限**:在 [!DNL Offer Decisioning]決策規則中，封閉（也稱為頻率封閉）會用來定義選件的顯示次數。帽子有兩種類型：在合併的目標對象中建議選件的次數（稱為「全域上限」），以及可建議給相同使用者的選件的次數（稱為「設定檔上限」）。

**目錄**:在來源和目的地方面，型錄是一個圖庫，可與Adobe應用程式和協力廠商技術連線。不要與[!DNL Catalog Service]混淆。

**[!DNL Catalog Service]**: [!DNL Catalog Service] (有時稱 [!DNL Catalog]為)是Adobe Experience Platform境內資料位置和世系的記錄系統。雖然所有吸收到Experience Platform中的資料都會以檔案和目錄的形式儲存在資料湖中，但[!DNL Catalog]會保留這些檔案和目錄的中繼資料和說明，以供查閱、監控和資料管理之用。

**類別**:在Experience Data Model(XDM)中，類定義了用於構建方案的最小欄位集，並定義了方案所表示的業務對象的基本行為。

**客戶**:客戶端是通過PostgreSQL協定或HTTP API連 [!DNL Query Service] 接到的外部工具或應用程式。

**系列**:在 [!DNL Offer Decisioning]中，系列是根據行銷人員定義的預先定義條件（例如選件類別）來劃分選件的子集。

**結合PII行銷動作**:將任何個人識別資訊(PII)與匿名資料結合的行銷動作。來自廣告網路、廣告伺服器和第三方資料供應商的資料合約通常包含特定合約禁止在直接可識別資料的情況下使用此類資料。

**命令行介面**:命令列介面是文字型工具，可用來連接至原始查詢 [!DNL Query Service] 執行。

**構圖**:構成是組成架構的一組元件。

**條件**:在 [!DNL Platform Launch]中，條件是評估必須返回或的邏輯語句的規則 `true` 元件 `false`。所有條件都必須評估為`true`，所有例外條件必須評估為`false`，才能執行規則上的任何操作。

**控制台**:在中， [!DNL Query Service]控制台提供有關查詢狀態和操作的資訊。控制台顯示到[!DNL Query Service]的連接狀態、正在執行的查詢操作以及這些查詢產生的任何錯誤消息。

**合約(「C」)標籤**:合約(「C」)資料使用標籤用於分類具有合約義務或與客戶資料管理政策相關的資料。

**C1合同標籤**:合約 `C1` 資料使用標籤指定資料只能匯出自Adobe Experience Cloud的匯整形式，而不包含個別或裝置識別碼。例如，源自社交網路的資料。

**C2合同標籤**:合約 `C2` 資料使用標籤會指定無法匯出至第三方的資料。有些資料提供者在其合約中有條款禁止從原始收集位置匯出資料。 例如，社交網路合約通常會限制您從其接收到的資料傳輸。 C2的限制比C1更嚴格，因為C1隻需要聚合和匿名資料。

**C3合同標籤**:合約 `C3` 資料使用標籤會指定無法結合或與直接可識別資訊搭配使用的資料。有些資料提供者在其合約中有條款禁止將資料與直接可識別資訊合併或使用。 例如，源自廣告網路、廣告伺服器和第三方資料供應商的資料合約通常包含使用直接可識別資料的特定合約禁止。

**C4合同標籤**:合 `C4` 約資料使用標籤指定無法將資料用於定位任何廣告或內容，不論是現場或跨網站。C4是最嚴格的標籤，因為它包含C5、C6和C7標籤。

**C5合約標籤**:合約 `C5` 資料使用標籤指定資料無法用於興趣型內容或廣告的跨網站定位。符合下列三個條件時，就會發生喜好式定位或個人化：現場收集的資料用於推斷用戶的興趣；用於其他內容，例如其他網站或應用程式；並用來根據這些推論來選擇提供哪些內容或廣告。

**C6合同標籤**:合約 `C6` 資料使用標籤會指定無法將資料用於現場廣告定位。臨場廣告定位包括在您組織的網站或應用程式上選擇和發佈廣告，或評估此類廣告的發佈效果。 這包括使用先前收集到的有關使用者興趣的現場資料來選擇廣告、處理有關所顯示廣告的資料、顯示時間和位置，以及使用者是否採取與廣告相關的任何動作，例如選擇廣告或購買。

**C7合約標籤**:合 `C7` 約資料使用標籤指定資料無法用於內容的現場定位。定位網站內容包括選擇和傳送您組織網站或應用程式上的內容，或測量內容的傳送和效果。 這包括先前收集的使用者對選取內容的興趣、處理顯示內容的資料、顯示頻率或時間長度、顯示時間和位置，以及使用者是否採取任何與內容相關的動作，包括例如選取內容。

**C8合同標籤**:合約 `C8` 資料使用標籤會指定資料無法用於測量貴組織的網站或應用程式。這並不包括喜好式定位，即您使用本服務後來個人化內容和／或廣告的資訊收集。

**C9合同標籤**:合約 `C9` 資料使用標籤會指定資料無法用於資料科學工作流程。有些合約明確禁止用於資料科學的資料。 有時，這些措辭的措辭會禁止使用資料用於人工智慧(AI)、機器學習(ML)或模型。

**C10合約標籤**:合約 `C10` 資料使用標籤指定無法將資料用於銜接身分啟動。某些資料使用政策會限制使用銜接身分資料進行個人化。 如果`C10`標籤的合併策略使用「專用圖表」選項，則會自動套用至區段。

**「建立日期」列**:透過來源連線指定第三方資料時，選取「建立日期」欄是一個選項。當選擇附加保存策略且資料集模式包含多個日期欄位時，必須從可用模式中選擇以指定「建立日期」鍵列。 選擇覆蓋保存策略時，「建立日期」選項不可用。

**按選擇建立表**:Create Table as Select(CTAS)是SQL命令，作為完整且有效的SQL查詢的一部分執行時，該命令將指 [!DNL Query Service] 示將查詢結果保存在資料集中。您可以建立新的結果集、覆寫先前的結果，或附加至先前的結果。

**跨網站資料**:跨網站資料是來自多個網站的資料組合，包括現場資料與非現場資料的組合，或來自數個非現場來源的資料組合。

**跨網站定位行銷動作**:使用資料進行跨網站廣告定位的行銷動作。數個網站的資料組合（包括現場資料與非現場資料的組合，或數個非現場來源的資料組合）稱為跨網站資料。 跨網站資料通常會收集並處理，以推論客戶的興趣。

**自訂身分名稱空間**:您的組織可以建立自訂身分名稱空間，以代表特定組織或商業案例的身分。

**自訂標籤**:自訂資料使用標籤可讓您建立特定標籤，並套用至符合特定業務需求的資料欄位。

**客戶人工智慧**:客戶AI是由Adobe Sensei提供的智慧型服務，以AI為基礎的傾向豐富客戶個人檔案，並協助客戶細分和鎖定目標。

## D

**資料字典**:在 [!DNL Platform Launch]中，資料字典（又稱資料映射）是在屬性中定義的一組資料元素。

**資料元素**:在 [!DNL Platform Launch]中，資料元素是指在規則和擴充功能中用來指向用戶端裝置上特定資料片段的指標。

**資料擷取**:資料擷取是將資料從來源新增至Experience Platform的程式。資料可透過多種方式傳入Platform，包括串流、批次或透過來源連接器新增。

**資料層**:在 [!DNL Platform Launch]中，資料層是存在於用戶端裝置上的資料結構，包含檢視頁面或畫面之上下文的中繼資料。

**資料治理**:資料治理包括用於確保資料遵守有關資料使用的法規和組織政策的戰略和技術。

**資料整合合作夥伴**:資料整合合作夥伴可簡化並自動化從超過200個來源將大量資料載入和轉換為Experience Platform，毋需編寫程式碼。

**資料集標籤**:資料使用量標籤可以新增至資料集。該資料集內的所有欄位都將繼承該資料集的標籤。

**資料科學工作區**: [!DNL Data Science Workspace] 在Experience Platform中，客戶可以利用跨平台的資料建立機器學習模型，並Adobe應用程式來建立智慧型細分、產生見解並提供預測，讓您大幅提升使用者數位體驗。

**資料來源**:資料源是用戶指定的資料源。資料來源的範例包括行動應用程式、描述檔和／或體驗事件、網站描述檔事件或CRM。

**資料管理員**:資料管理員是負責管理、監督和執行組織資料資產的人員。資料管理員也可確保資料管理政策受到保護並維持，以符合政府法規和組織政策。

**資料流**:資料流是一組或一組消息，這些消息共用相同的模式，並由同一源發送。

**資料類型**:資料類型是可重用的XDM資源，它定義了在分層表示中包含多個屬性的對象類型欄位。

**資料使用標籤**:資料使用標籤可讓您將反映隱私權相關考量和合約條件的資料分類，以符合法規和公司政策。新增至資料集的資料使用標籤會繼承或套用至該資料集內的所有欄位。 資料使用標籤也可以直接套用至欄位。

**資料流**:資料流是從源流到目標流入平台的虛擬資料流。

**資料流運行**:資料流運行是一種資料流，根據用戶指定的時間表Experience Platform。

**資料集**:資料集是資料集合的儲存和管理結構，通常是包含結構（欄）和欄（列）的表格。

**資料集ID**:Adobe產生的用於所攝取資料集的標識符。

**資料集輸出**:資料集輸出提供一種機制，可決定「以選取方式建立表格」選項將用於特定執 [!DNL Query Service] 行。

**Delta欄**:增量列允許您選擇源資料欄位，以表示增量接收的時間戳。

**增量儲存策略**:增量保存策略是用於通過源連接接收第三方資料的選項。此選項可讓使用者指定新或變更的來源資料列會收錄至Experience Platform。 新行會新增至資料集的結尾，而變更的行則會在Experience Platform的資料集中更新。

**描述符**:在Experience Data Model(XDM)中，描述符是描述欄位特定行為的另一組模式相關元資料。Experience Platform可以使用描述符來瞭解預期的模式行為，如兩個模式之間的關係。

**目標**:目標是任何端點的一般術語，例如Adobe應用程式、廣告平台、雲端儲存服務或行銷服務，在此處會啟動並傳遞對象。

**目標類別**:目標類別是具有相似特徵的目標分組。

**目標目錄**:目標目錄是Experience Platform中可用目標的清單。

## S

**範例資料**:範例資料是資料檔案的預覽，通常是前100列，讓資料科學家或工程師瞭解資料檔案中的架構或資料。

**沙盒**:沙盒是一種虛擬結構，可將單一平台實例分割為獨立的虛擬環境，以協助開發和發展數位體驗應用程式。

**沙盒重設**:沙盒重設會刪除沙盒內的所有資料，包括資料、描述檔和區段。沙盒重設可能會影響連接至內部或外部目的地的資料。

**沙盒切換器**:Experience Platform中的沙盒切換器控制項可讓使用者在他們可以存取的沙盒之間導覽。切換沙盒會變更所有內容，並可能會根據權限變更功能存取。

**排程**:排程是使用者定義的規格，說明從協力廠商資料來源擷取資料至Adobe Experience Platform的頻率或速度。

**計分**:計分是使用訓練好的模型，從資料產生見解的程式。

**架構**:架構是一組規則，用於表示和驗證資料的結構和格式。架構由類和可選欄位組組成，用於建立資料集和資料流。 一種方案可包括行為屬性、時間戳、身份、屬性定義、關係等。

**方案欄位組**:在Experience Data Model(XDM)中，架構欄位群組可讓使用者擴充可重複使用的欄位，以定義要包含在架構中的一或多個屬性。

**架構庫**:架構庫包含由Adobe提供的業界標準XDM資源，以及由您的組織定義的自訂資源。

**方案註冊**:方案註冊表提供用於查看和管理方案庫中所有方案相關資源的用戶介面和REST風格的API。

**機密存取金鑰**:機密存取金鑰是 [!DNL Amazon] S3金鑰，與存取金鑰ID搭配使用以簽署AWS要求。

**區段**:區段是一組規則，包含屬性和事件資料，可限定數個設定檔成為觀眾。

**區段產生器**:是 [!DNL Segment Builder] 用來建立區段定義的視覺化開發環境。它是所有使用「Experience Platform分段服務」之應用程式的通用元件。

**區段定義**:區段定義是用於描述目標對象之主要特性或行為的規則集。概念化後，區段定義中概述的規則會用來決定區段的合格讀者成員。

**區段評估方法**:分段評估方法有兩種：排程和隨選。排程評估可啟用定期排程，以便在特定時間執行匯出工作，而隨選評估則包括建立區段工作以立即建立觀眾。

**區段匯出**:區段匯出是Experience Platform中的兩種目的地之一。透過區段匯出，您可以傳送符合資格且已映射至目標的描述檔。 使用群體和使用者ID以及假名資料，並通常與社交網路和其他數位媒體目標平台整合。

**區段ID**:區段ID是自動產生的識別碼，與區段相關聯。

**區段會籍**:區段成員資格會顯示描述檔目前屬於哪些區段。

**區段規則**:區段規則會定義條件，以決定描述檔是否符合區段的資格。

**區段**:區段是將大量客戶、潛在客戶或消費者分成小型群組的程式，這些群組具有相似的屬性，並會以類似方式回應特定的行銷策略。

**Sensei ML框架**:Sensei ML架構是一個統一的機器學習(ML)架構，它利用Experience Platform資料，讓資料科學家以更快速、可擴充且可重複使用的方式，開發ML導向的智慧服務。

**敏感(「S」)標籤**:敏感(「S」)標籤可用來分類被視為敏感的資料，例如您想要標示為敏感的不同行為或地理資料類型。

**服務**:利用Adobe智慧服務實現AI和ML服務的強大框架。服務可提供即時、個人化的客戶體驗，或將自訂智慧服務付諸實作。

**單一身分個人化行銷動作**:使用資料進行站上內容個人化的行銷動作。Onsite個人化是用於推論使用者興趣的任何資料，並用於根據這些推論選擇提供哪些內容或廣告。

**S1資料使用標籤**:資 `S1` 料使用標籤用來分類指定緯度和經度的資料，可用來判斷裝置的精確位置。

**S2資料使用標籤**:數 `S2` 據使用標籤用於分類資料，該資料可用於確定廣義的地理區域。

**來源**:來源是「平台」中任何輸入連接器的一般術語。另請參閱：源連接器

**來源屬性**:來源屬性是來源資料集中的欄位。源屬性映射到目標方案欄位。

**源目錄**:源目錄是Experience Platform中可用源連接器的清單。

**來源類別**:來源類別是一組具有相似特徵的來源。

**源連接器**:來源連接器（也稱為來源）可協助使用者輕鬆從多個來源擷取資料，允許使用Experience Platform服務來建構、標籤和增強資料。資料可從多種來源擷取，例如雲端儲存空間、協力廠商軟體和CRM系統。

**串流連線**:串流連線是由Adobe提供的獨特端點，並系結至客戶的IMS組織，以串流資料至Experience Platform。

**標準識別名稱空間**:標準身分名稱空間是由Adobe提供的預先定義的身分名稱空間，代表業界標準解決方案，通常用來識別客戶。

**串流擷取**:串流擷取可讓您從用戶端和伺服器端裝置即時傳送資料至Experience Platform。

**串流區段**:串流區段是持續不斷的資料選擇程式，可因應使用者活動而更新區段。建立並儲存區段後，區段定義會套用至[!DNL Real-time Customer Profile]的傳入資料。 區段新增和移除作業會定期處理，確保目標受眾保持相關性。

**系統視圖**:「系統視圖」是源資料集的視覺化表示，這些資料集流 [!DNL Real-time Customer Profile] 向目標。

## T

**目標功能**:在特徵映射中，目標特徵是由模型預測的特徵。

**時間系列資料**:時間序列資料提供了記錄主體直接或間接採取操作時系統的快照。

**訓練好的模型**:訓練模型表示模型訓練過程的可執行輸出，其中一組訓練資料被應用到模型實例。經過訓練的模型將保留對任何由其建立的智慧Web服務的引用。 訓練好的模型適合於評分和建立智慧Web服務。

**代號**:Token是一種雙因素驗證安全性，可用來授權搭配使用電腦服務 [!DNL Query Service]。

## U

**聯合模式**:聯合模式是共用相同類且已啟用的模式的合併 [!DNL Real-time Customer Profile]。組織可以存在多個聯合模式，但每個類只能有一個聯合模式。

## X

**XDM**:請參閱體驗資料模型(XDM)。

**XDM決策事件**:XDM決策事件是基於時間序列的類，用於捕獲有關決策活動結果和上下文的觀察結果。這包括關於決策的制定方式、決策發生時、建議（和選擇）的選項，以及決策過程中影響決策或可以觀察到的情境狀態的資訊。

**XDM ExperienceEvent**:XDM ExperienceEvent是以時間序列為基礎的類別，用於在發生事件（或事件集）時擷取系統狀態，包括所涉主題的時間點和身分。另請參閱：體驗事件

**XDM個人資料**:XDM是 [!DNL Individual Profile] 一種基於記錄的類，它對已識別和部分識別的對象的屬性形成單一表示。高度識別的個人檔案可用於個人通訊或目標互動，且可包含詳細的個人資訊，例如姓名、性別、出生日期、地點和聯絡資訊，包括電話號碼和電子郵件地址。

**XDM系統**:XDM System代表一個框架，用於操作XDM模式，以便用於下游Experience Platform服務。
