# 合併策略使用手冊

Adobe Experience Platform可讓您從多個來源匯整資料並加以匯整，以全面瞭解每個客戶。 將這些資料整合在一起時，合併原則是平台用來決定資料的優先順序以及將哪些資料合併以建立統一檢視的規則。

使用REST風格的API或用戶介面，您可以建立新的合併策略、管理現有策略，並為組織設定預設的合併策略。 本指南提供使用Adobe Experience Platform使用者介面處理合併原則的逐步指示。

如果您希望使用即時客戶描述檔API來處理合併原則，請依照合併原則API教學課程中 [的說明進行](../api/merge-policies.md)。

## 快速入門

本指南需要對合併策略涉及的各種Experience Platform服務有充分的瞭解。 在開始本教學課程之前，請先閱讀下列服務的檔案：

* [即時客戶個人檔案](../home.md):根據來自多個來源的匯整資料，提供統一、即時的消費者個人檔案。
* [身分服務](../../identity-service/home.md):借由橋接來自不同資料來源的身分，並將之收錄至平台，以建立即時客戶個人檔案。
* [體驗資料模型(XDM)](../../xdm/home.md):平台組織客戶體驗資料的標準化架構。

## 查看合併策略

在Experience Platform使用者介面中，您可以開始使用合併原則，並透過按一下左側導軌中的「描述檔」，然後選取「合併原則」標籤，查看組織現有合併原則的清單 ******** 。

![合併策略登錄頁](../images/merge-policies/landing.png)

在登陸頁面上會顯示組織可用之每個合併原則的詳細資訊，包括「原則名 *稱*」、「 *預設合併原則*」和「 *架構*」。

要選擇顯示哪些詳細資訊，或向顯示添加其他列，請選擇右側的列選擇器表徵圖，然後按一下列名以添加或從視圖中刪除列。

![](../images/merge-policies/adjust-view.png)

## 建立合併策略

要建立新的合併策略，請按一下「合 **並策略** 」頁籤右上角的「創 **建合併策略** 」。

![合併策略登錄頁](../images/merge-policies/create-new.png)

此時 **會出現「建立合併原則** 」畫面，可讓您提供新合併原則的重要資訊。

![](../images/merge-policies/create.png)

* **名稱**:合併策略的名稱應具有描述性，但應簡明扼要。
* **架構**:與合併策略關聯的架構。 這指定為其建立此合併策略的XDM模式。 組織可以為每個方案建立多個合併策略。
* **ID拼接**:此欄位定義如何確定客戶的相關身份。 有兩個可能的值：
   * **無**:不執行身份聯繫。
   * **專用圖**:根據您的個人身分圖表執行身分識別接合。
* **屬性合併**:描述檔片段是個別客戶現有身分清單中僅有一個身分的描述檔資訊。 當使用身份圖形類型導致多個身份時，存在衝突的配置檔案屬性值和優先順序的可能性。 使用 *屬性合併* ，可讓您指定發生合併衝突時要排定優先順序的資料集描述檔值。 有兩個可能的值：
   * **排序時間戳**:在發生衝突時，請優先處理最近更新的設定檔。
   * **資料集優先順序** :根據個人資料片段的來源資料集，為描述檔片段提供優先順序。 選擇此選項時，必須選擇相關資料集及其優先順序順序。 如需詳細資訊，請參 [閱以下資料集](#dataset-precedence) 優先順序的詳細資訊。
* **預設合併策略**:切換按鈕，可讓您選擇此合併原則是否為您組織的預設原則。 如果選擇器已開啟且新原則已儲存，您先前的預設原則會自動更新為不再為預設原則。

### 資料集優先順序 {#dataset-precedence}

在選擇屬 *性合併值時* ，可以選擇 *Dataset優先順序* ，該優先順序允許您根據配置檔案片段的來源資料集為它們指定優先順序。

例如，如果貴組織在一個資料集中顯示的資訊比另一個資料集中的資料更偏好或受信任，則使用案例會是範例。

選擇數 *據集優先順序*，將開啟一個單獨的面板，要求您從「可用資料集 ** 」（或使用複選框選擇所有）將包含哪些資料集。 然後，您可以將這些資料集拖放到「選定的 *資料集* 」面板中，並將它們拖放到正確的優先順序順序中。 優先順序最高的資料集，其次是次高的資料集，依此類推。

![](../images/merge-policies/dataset-precedence.png)

建立完合併策略後，按一下「保 **存** 」返回「合併策略 ** 」頁籤，新合併策略現在出現在策略清單中。

## 編輯合併原則

通過按一下要編輯的合併策略的 *Policy Name* （策略名稱），可以通過「 *Merge policies* 」（合併策略）頁籤修改現有合併策略。

![合併策略登錄頁](../images/merge-policies/select-edit.png)

當出現「 *Edit merge policy* 」(編輯合併策略 *)螢幕時，您可以對* Name *、* Name *、***** Intring、ID Attribute mergeType、ID Suppliting AttributeType以及選擇此策略是否將Oracle Default Merge策略合併策略以合併到您的組織中時。

>[!Note]
>您無法編輯合併策略ID，如編輯螢幕頂部所示。 這是唯讀、系統產生的ID，無法變更。

![](../images/merge-policies/edit-screen.png)

完成必要的更改後，按一下「保 **存** 」返回到「合併策略 ** 」頁籤，此時將顯示更新的合併策略資訊。

![](../images/merge-policies/edited.png)

## 違反資料治理政策

建立或更新合併策略時，會執行檢查以確定合併策略是否違反您組織定義的任何資料使用策略。 資料使用政策是Adobe Experience Platform資料治理的一部分，也是描述您可在特定平台資料上執行或受限制之行銷動作類型的規則。 例如，如果合併策略用於建立激活到第三方目標的段，而您的組織有阻止將特定資料導出到第三方的資料使用策略，則在嘗試保存合併策略時，您會收到「檢測到資料治理策略違規」通知。

此通知包括已違反的資料使用策略清單，允許您通過從清單中選擇策略來查看違規的詳細資訊。 在選擇違反的策略時，「資料世系 *」頁籤提供* 違規原因和「受影響」激活 ****，每個頁籤都提供了違反資料使用策略的更詳細資訊。

若要進一步瞭解如何在Adobe Experience Platform中執行資料治理，請先閱讀資料治 [理概觀](../../data-governance/home.md)。

![](../images/merge-policies/policy-violation.png)
