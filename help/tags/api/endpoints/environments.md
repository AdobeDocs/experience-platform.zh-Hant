# 環境端點

當 [資料庫](./libraries.md) 編譯到 [建置](./builds.md) 在Reactor API中，組建的確切內容取決於環境設定和程式庫中包含的資源。 具體而言，環境會決定下列項目：

1. **目的地**:您希望部署組建的位置。 您可以選取 [主機](./hosts.md) 供環境使用。
1. **封存**:您可以選擇以可部署的檔案集來擷取組建，或以封存格式壓縮組建。 這是由 `archive` 設定。

由環境設定的目的地和封存格式會變更您參考應用程式中組建(該參考為 [內嵌程式碼](../../ui/publishing/environments.md#embed-code))。 如果您對目標或檔案格式進行任何更改，則必須對應用程式進行匹配更新，以使用新引用。

環境分為三種類型（或階段），每種類型對您可擁有的總數有不同的限制：

| 環境類型 | 允許的數字 |
| --- | --- |
| 開發 | （無限制） |
| 預備 | 一 |
| 生產 | 一 |

{style="table-layout:auto"}

這些環境類型具有類似的行為，但會用於 [標籤發佈工作流程](../../ui/publishing/publishing-flow.md).

環境只屬於一個 [屬性](./properties.md).

如需環境的更一般資訊，請參閱 [環境](../../ui/publishing/environments.md) 在發佈檔案中。

## 快速入門

本指南中使用的端點屬於 [Reactor API](https://www.adobe.io/experience-platform-apis/references/reactor/). 繼續之前，請檢閱 [快速入門手冊](../getting-started.md) 以取得如何驗證API的重要資訊。

## 擷取環境清單 {#list}

您可以在GET要求的路徑中加入屬性的ID，以擷取屬性的環境清單。

**API格式**

```http
GET /properties/{PROPERTY_ID}/environments
```

| 參數 | 說明 |
| --- | --- |
| `PROPERTY_ID` | 此 `id` 擁有環境的屬性。 |

{style="table-layout:auto"}

>[!NOTE]
>
>您可以使用查詢參數，根據下列屬性來篩選列出的環境：<ul><li>`archive`</li><li>`created_at`</li><li>`name`</li><li>`stage`</li><li>`token`</li><li>`updated_at`</li></ul>請參閱 [篩選回應](../guides/filtering.md) 以取得更多資訊。

**要求**

```shell
curl -X GET \
  https://reactor.adobe.io/properties/PR97d92a379a5f48758947cdf44f607a0d/environments \
  -H 'Authorization: Bearer {ACCESS_TOKEN}' \
  -H 'x-api-key: {API_KEY}' \
  -H 'x-gw-ims-org-id: {ORG_ID}' \
  -H "Content-Type: application/vnd.api+json" \
  -H 'Accept: application/vnd.api+json;revision=1'
```

## 建立環境 {#create}

您可以提出POST要求，以建立新環境。

**API格式**

```http
POST /properties/{PROPERTY_ID}/environments
```

| 參數 | 說明 |
| --- | --- |
| `PROPERTY_ID` | 此 `id` 的 [屬性](./properties.md) 定義下的環境。 |

{style="table-layout:auto"}

**要求**

下列請求會為指定的屬性建立新環境。 呼叫也會透過將環境與現有主機建立關聯 `relationships` 屬性。 請參閱 [關係](../guides/relationships.md) 以取得更多資訊。

```shell
curl -X POST \
  https://reactor.adobe.io/properties/PR97d92a379a5f48758947cdf44f607a0d/environments \
  -H 'Authorization: Bearer {ACCESS_TOKEN}' \
  -H 'x-api-key: {API_KEY}' \
  -H 'x-gw-ims-org-id: {ORG_ID}' \
  -H 'Content-Type: application/json' \
  -d '{
        "data": {
          "attributes": {
            "name": "Development Environment A",
            "archive": true,
            "archive_passphrase": "pass12345",
            "path": "/development-a",
            "stage": "development"
          },
          "relationships": {
            "host": {
              "data": {
                "id": "HT5d3fbe7bd34d4f65a46fad4598aefd4e",
                "type": "hosts"
              }
            }
          },
          "type": "environments"
        }
      }'
```

| 屬性 | 說明 |
| --- | --- |
| `attributes.name` | **（必要）** 環境的人類看得懂的名稱。 |
| `attributes.archive` | 一個布林值，用於指示其是否為歸檔格式的生成。 |
| `attributes.archive_passphrase` | 可用於解鎖存檔檔案的字串密碼。 |
| `attributes.path` | 來自環境主機URL的路徑。 |
| `attributes.stage` | 環境階段（開發、測試或生產）。 |
| `id` | 此 `id` 的URL。 這應符合 `{ENVIRONMENT_ID}` 值。 |
| `type` | 要更新的資源類型。 對於此端點，值必須是 `environments`. |

{style="table-layout:auto"}

## 更新環境 {#update}

您可以在PATCH請求的路徑中加入環境ID，以更新環境。

**API格式**

```http
PATCH /environments/{ENVIRONMENT_ID}
```

| 參數 | 說明 |
| --- | --- |
| `ENVIRONMENT_ID` | 此 `id` 的URL。 |

{style="table-layout:auto"}

**要求**

下列請求會更新 `name` 的URL區段。

```shell
curl -X PATCH \
  https://reactor.adobe.io/environments/DE3fab176ccf8641838b3da59f716fc42b \
  -H 'Authorization: Bearer {ACCESS_TOKEN}' \
  -H 'x-api-key: {API_KEY}' \
  -H 'x-gw-ims-org-id: {ORG_ID}' \
  -H 'Content-Type: application/json' \
  -d '{
        "data": {
          "attributes": {
            "name": "New Environment Name"
          },
          "id": "ENeb00d8f62d244732bd27765301b1410f",
          "type": "environments"
        }
      }'
```

| 屬性 | 說明 |
| --- | --- |
| `attributes` | 一個對象，其屬性表示要為環境更新的屬性。 可更新下列環境屬性： <ul><li>`archive`</li><li>`archive_passphrase`</li><li>`include_debug_library`</li><li>`name`</li><li>`path`</li></ul> 請參閱的範例呼叫 [建立環境](#create) 以取得屬性清單及其使用案例。 |

| `type` | 要更新的資源類型。 對於此端點，值必須是 `environments`. |

## 刪除環境

您可以在DELETE請求的路徑中加入環境ID，以刪除環境。

**API格式**

```http
DELETE /environments/{ENVIRONMENT_ID}
```

| 參數 | 說明 |
| --- | --- |
| `ENVIRONMENT_ID` | 此 `id` 刪除的環境。 |

{style="table-layout:auto"}

**要求**

```shell
curl -X DELETE \
  https://reactor.adobe.io/environments/ENeb00d8f62d244732bd27765301b1410f \
  -H 'Authorization: Bearer {ACCESS_TOKEN}' \
  -H 'x-api-key: {API_KEY}' \
  -H 'x-gw-ims-org-id: {ORG_ID}'
```

**回應**

成功的回應會傳回HTTP狀態204（無內容），但沒有回應內文，表示環境已遭刪除。

請參閱 [關係指南](../guides/relationships.md) 以取得Reactor API中關係的詳細資訊。

### 查找環境的相關主機 {#host}

您可以借由附加 `/host` 至GET請求的路徑。

>[!NOTE]
>
>您可以透過 [獨立呼叫](#host-relationship).

**API格式**

```http
GET  /environments/{ENVIRONMENT_ID}/host
```

| 參數 | 說明 |
| --- | --- |
| `{ENVIRONMENT_ID}` | 此 `id` 您想要查找的環境。 |

{style="table-layout:auto"}

**要求**

```shell
curl -X GET \
  https://reactor.adobe.io/environments/ENeb00d8f62d244732bd27765301b1410f/host \
  -H 'Authorization: Bearer {ACCESS_TOKEN}' \
  -H 'x-api-key: {API_KEY}' \
  -H 'x-gw-ims-org-id: {ORG_ID}' \
  -H "Content-Type: application/vnd.api+json" \
  -H 'Accept: application/vnd.api+json;revision=1'
```
