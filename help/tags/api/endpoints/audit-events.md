# 審核事件端點

>[!WARNING]
>
>實施 `/audit_events` 隨著功能的添加、刪除和重作，端點處於通量中。

稽核事件是對Reactor API中其他資源進行特定變更的記錄，會在進行變更時產生。 這些是可透過使用 [回撥](./callbacks.md). 此 `/audit_events` reactor API中的端點可讓您以程式設計方式管理experience應用程式中的稽核事件。

審核事件以 `{RESOURCE_TYPE}.{EVENT}`，例如 `build.created` 或 `rule.updated`.

資源類型可以是下列任一類型：

* `property`
* `extension`
* `data_element`
* `rule`
* `rule_component`
* `library`
* `build`
* `environment`
* `host`

每種資源類型均支援下列事件：

* `created`
* `updated`
* `deleted`

## 快速入門

本指南中使用的端點屬於 [Reactor API](https://www.adobe.io/experience-platform-apis/references/reactor/). 繼續之前，請檢閱 [快速入門手冊](../getting-started.md) 以取得如何驗證API的重要資訊。

## 擷取稽核事件清單 {#list}

您可以向 `/audit_events` 端點。

**API格式**

```http
GET /audit_events
```

**要求**

```shell
curl -X GET \
  https://reactor.adobe.io/audit_events \
  -H 'Authorization: Bearer {ACCESS_TOKEN}' \
  -H 'x-api-key: {API_KEY}' \
  -H 'x-gw-ims-org-id: {ORG_ID}' \
  -H "Content-Type: application/vnd.api+json" \
  -H 'Accept: application/vnd.api+json;revision=1'
```

**回應**

## 查找審核事件 {#lookup}

您可以在稽核請求的路徑中提供其ID，借此查詢GET事件。

**API格式**

```http
GET /audit_events/{AUDIT_EVENT_ID}
```

| 參數 | 說明 |
| --- | --- |
| `AUDIT_EVENT_ID` | 此 `id` 查找的審計事件。 |

{style="table-layout:auto"}

**要求**

```shell
curl -X GET \
  https://reactor.adobe.io/audit_events/AEa98742de8ef044d8b86767aa6a15a674 \
  -H 'Authorization: Bearer {ACCESS_TOKEN}' \
  -H 'x-api-key: {API_KEY}' \
  -H 'x-gw-ims-org-id: {ORG_ID}' \
  -H "Content-Type: application/vnd.api+json" \
  -H 'Accept: application/vnd.api+json;revision=1'
```

**回應**

成功的回應會傳回稽核事件的詳細資訊。
