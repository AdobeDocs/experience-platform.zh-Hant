# 建立 [!DNL Amazon S3] UI中的源連接

Adobe Experience Platform的源連接器提供了定期接收外部源資料的能力。 本教程提供建立 [!DNL Amazon S3]  (以下簡稱：[!DNL S3]&quot;)使用平台用戶介面的源連接器。

## 快速入門

本教程需要對Adobe Experience Platform的以下部分進行有效的理解：

- [[!DNL Experience Data Model (XDM)] 系統](../../../../../xdm/home.md):Experience Platform組織客戶體驗資料的標準化框架。
   - [架構組合的基礎](../../../../../xdm/schema/composition.md):瞭解XDM架構的基本構建基塊，包括架構組成中的關鍵原則和最佳做法。
   - [架構編輯器教程](../../../../../xdm/tutorials/create-schema-ui.md):瞭解如何使用架構編輯器UI建立自定義架構。
- [[!DNL Real-Time Customer Profile]](../../../../../profile/home.md):基於來自多個源的聚合資料提供統一、即時的用戶配置檔案。

如果您已經有 [!DNL S3] 連接，您可以跳過本文檔的其餘部分並繼續學習有關 [配置資料流](../../dataflow/batch/cloud-storage.md)。

### 收集所需憑據

為了訪問 [!DNL S3] 平台上的bucket，您需要為以下憑據提供有效值：

| 憑據 | 說明 |
| ---------- | ----------- |
| `s3AccessKey` | 您的訪問密鑰ID [!DNL S3] 桶。 |
| `s3SecretKey` | 您的密鑰ID [!DNL S3] 桶。 |
| `serviceUrl` | （可選）自定義 [!DNL S3] 要連接到的終結點。 當您 [!DNL S3] 桶是特定於區域的。 格式 `serviceUrl` 為： `https://s3.{REGION}.amazonaws.com/)`。 |
| `bucketName` | 的 [!DNL S3] bucket包含您的資料及其相應的描述性元資料。 您 [!DNL S3] 儲存段名稱的長度必須介於3到63個字元之間，且必須以字母或數字開頭和結尾。 儲存桶名稱只能包含小寫字母、數字或連字元(`-`)，並且不能格式化為IP地址。 |
| `folderPath` | 資料夾的路徑 [!DNL S3] 儲存資料的儲存桶。 當用戶具有限制訪問時，此憑據是必需的。 |

有關入門的詳細資訊，請訪問 [本AWS檔案](https://aws.amazon.com/blogs/security/wheres-my-secret-access-key/)。

## 連接 [!DNL S3] 帳戶

在平台UI中，選擇 **[!UICONTROL 源]** 從左導航欄訪問 [!UICONTROL 源] 工作區。 的 [!UICONTROL 目錄] 螢幕顯示可為其建立帳戶的各種源。

可以從螢幕左側的目錄中選擇相應的類別。 或者，您可以使用搜索選項找到要使用的特定源。

在 **[!UICONTROL 雲儲存]** 類別，選擇 **[!UICONTROL AmazonS3]** ，然後選擇 **[!UICONTROL 添加資料]**。

![目錄](../../../../images/tutorials/create/s3/catalog.png)

的 **[!UICONTROL 連接到AmazonS3]** 的子菜單。 在此頁上，您可以使用新憑據或現有憑據。

### 新帳戶

如果使用新憑據，請選擇 **[!UICONTROL 新帳戶]**。 在顯示的輸入表單上，提供名稱、可選說明和 [!DNL S3] 憑據。 完成後，選擇 **[!UICONTROL 連接到源]** 然後再給新連接建立一段時間。

![連接](../../../../images/tutorials/create/s3/connect.png)

### 現有帳戶

要連接現有帳戶，請選擇 [!DNL S3] 要連接的帳戶，然後選擇 **[!UICONTROL 下一個]** 繼續。

![現有](../../../../images/tutorials/create/s3/existing.png)
