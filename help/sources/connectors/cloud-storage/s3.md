# AmazonS3連接器

Adobe Experience Platform提供本地連接，如AWS, [!DNL Google Cloud Platform], [!DNL Azure]。 您可以將資料從這些系統中 [!DNL Platform]。

雲儲存源可以將您自己的資料 [!DNL Platform] 無需下載、格式化或上載。 所攝取的資料可以格式化為XDM JSON、XDM Parke或分隔。 流程的每個步驟都整合到「源」工作流中。 [!DNL Platform] 允許您通過批處理從S3導入資料。

## IP地址允許清單

在使用源連接器之前，必須將IP地址清單添加到允許清單。 如果無法將特定於區域的IP地址添加到允許清單，則在使用源時可能會導致錯誤或效能不佳。 查看 [IP地址允許清單](../../ip-address-allow-list.md) 的子菜單。

## 檔案和目錄的命名約束

以下是命名雲儲存檔案或目錄時必須考慮的約束條件清單。

- 目錄和檔案元件名稱不能超過255個字元。
- 目錄和檔案名不能以正斜槓結尾(`/`)。 如果提供，將自動刪除。
- 必須正確轉義以下保留URL字元： `! ' ( ) ; @ & = + $ , % # [ ]`
- 不允許使用以下字元： `" \ / : | < > * ?`。
- 不允許使用非法的URL路徑字元。 代碼點類似 `\uE000`，但在NTFS檔案名中有效，則不是有效的Unicode字元。 此外，也不允許使用某些ASCII或Unicode字元，如控制字元（0x00到0x1F、\u0081等）。 有關HTTP/1.1中Unicode字串的規則，請參見 [RFC 2616，第2.2節：基本規則](https://www.ietf.org/rfc/rfc2616.txt) 和 [RFC 3987](https://www.ietf.org/rfc/rfc3987.txt)。
- 不允許使用下列檔案名：LPT1、LPT2、LPT3、LPT4、LPT5、LPT6、LPT7、LPT8、LPT9、COM1、COM2、COM3、COM4、COM5、COM6、COM7、COM9、COM9prn、AUX、NUL、CON、CLOCK$、點字元(.)和兩個點字元(..)。

## 先決條件 {#prerequisites}

要使用S3接收單個目錄，必須建立 [!DNL Identity and Access Management] (IAM)用戶，用於S3控制台中的平台，並為以下操作分配權限：

- `s3:GetObject`
- `s3:GetObjectVersion`

在瀏覽和測試連接性時還需要以下權限：

- `s3:ListAllMyBuckets`
- `s3:ListBucket`
- `s3:GetBucketLocation`

檔案路徑 `myBucket/folder/subfolder/subsubfolder/abc.csv` 可能只會讓你 `subsubfolder/abc.csv`。 如果要訪問子資料夾，可以指定 `bucket` 在S3控制台中 `myBucket` 和 `folderPath` 如 `folder/subfolder` 確保檔案探測從 `subfolder` 與 `subsubfolder/abc.csv`。

## 使用臨時安全憑據連接 [!DNL Amazon S3]

您可以連接 [!DNL Amazon S3] 使用臨時安全憑據 `s3SessionToken`。 這允許您連接 [!DNL Amazon S3] 無需使用 [!DNL Amazon Web Services]，或提供訪問 [!DNL Amazon S3] 儲存到不受信任環境中的用戶。

臨時安全憑據與常規的長期訪問密鑰憑據類似，但您可以為臨時憑據配置較短的到期日。 在激活後可以將過期時間設定為幾分鐘，也可以設定為幾小時。 用戶也不包含臨時憑據。 這意味著您必須在臨時憑據過期時請求新的一組憑據。

有關如何生成臨時會話令牌的步驟，請參閱 [[!DNL AWS] 臨時會話令牌的文檔](https://docs.aws.amazon.com/IAM/latest/UserGuide/id_credentials_temp_request.html#api_getsessiontoken)
。
