# 資料使用標籤字彙表 {#data-usage-labels-glossary}

>[!CONTEXTUALHELP]
>id="platform_policies_labeltype"
>title="標籤類型"
>abstract="資料使用標籤有多種類別。Adobe 定義的標籤包括合約標籤、身分識別標籤和敏感標籤。您的組織定義的標籤被歸類為自訂標籤。"
>text="See the data usage labels glossary for more information on these label types."

資料使用情況標籤可讓您根據以下專案將資料集和欄位分類： [治理原則](../policies/overview.md) 和 [存取控制原則](../../access-control/abac/overview.md) 適用於該資料之屬性。 Adobe Experience Platform提供數個現成可用的核心資料使用標籤，您可用來開始將資料分類。

本檔案概述Experience Platform目前提供的核心資料使用標籤。

## 合約標籤

合約「C」標籤可用來對具有合約義務或與貴組織的資料治理原則相關的資料進行分類。

| 標籤 | 定義 |
| --- | --- |
| [C1](#c1) | 資料只能以彙整形式從Adobe Experience Cloud匯出，而不包含個人或裝置識別碼。 |
| [C2](#c2) | 資料無法匯出至第三方。 |
| [C3](#c3) | 資料無法結合或搭配可直接識別的資訊使用。 |
| [C4](#c4) | 資料不可用於任何廣告或內容的目標定位，無論是在網站上還是跨網站。 |
| [C5](#c5) | 資料不可用於基於興趣、跨網站的內容或廣告目標定位。 |
| [C6](#c6) | 資料不可用於網站上的廣告目標定位。 |
| [C7](#c7) | 資料無法用於網站上的內容目標定位。 |
| [C8](#c8) | 資料不可用於衡量貴組織的網站或應用程式。 |
| [C9](#c9) | 資料不能用於資料科學工作流程。 |
| [C10](#c10) | 資料無法用於拼接身分啟用。 |
| [C11](#c11) | 資料無法與區段比對合作夥伴共用。 |
| [C12](#c12) | 資料無法以任何方式匯出。 |

## 身分標籤

身分「I」標籤可用來對可識別或聯絡特定人員的資料進行分類。

| 標籤 | 定義 |
| --- | --- |
| **I1** | 可識別或聯絡特定人員而非裝置的直接可識別資料。 |
| **I2** | 間接可識別的資料，可結合任何其他資料使用來識別或聯絡特定人員。 |

## 敏感標籤 {#sensitive}

敏感「S」標籤可用來對您及貴組織認為敏感的資料進行分類。

您可能認為敏感的其中一種資料型別可能是不同型別的地理資料，但此類別不限於地理資料。

| 標籤 | 定義 |
| --- | --- |
| **S1** | 指定可用於判斷裝置精確位置的經緯度的資料。 |
| **S2** | 可用於判斷廣泛定義的地理柵欄區域的資料。 |
| **PSPD** | 允許的敏感個人資料(PSPD)是指Adobe在合約上允許您上傳的被視為「敏感」、「特殊資料類別」或適用法律使用的類似詞語的資料。 這明確不包括受保護的健康資訊(PHI)和其他受管制的健康資料。 |
| **RHD** | 參考受保護的健康資訊(PHI)或Adobe合約准許您上傳之患者相關資訊的資料。 |

## 附錄

以下小節提供有關可用資料使用標籤的其他資訊。

### 合約標籤詳細資料

以下小節提供與實作特定合約「C」標籤相關的詳細資訊。

#### C1 {#c1}

部分資料只能以彙整形式從Adobe Experience Cloud匯出，而不包含個人或裝置識別碼。 例如，源自社交網路的資料。

#### C2 {#c2}

有些資料提供者的合約中有條款，禁止匯出原本收集的資料。 例如，社交網路合約通常會限制您從合約中接收的資料傳輸。 C2標籤的限制比 [C1](#c1)，只需要彙總和匿名資料，但限制條件卻比 [C12](#c12)，可防止資料完全匯出，無論目的地為何。

#### C3 {#c3}

有些資料提供者在其合約中訂有條款，禁止將該資料與可直接識別的資訊合併或使用。 例如，來自廣告網路、廣告伺服器和第三方資料提供者的資料合約通常包含明確的合約禁令，禁止將此類資料與可直接識別的資料一起使用。

#### C4 {#c4}

C4包含標籤 [C5](#c5)， [C6](#c6)、和 [C7](#c7). 這是限制最嚴格的標籤之一，僅次於以 [C12](#c12).

#### C5 {#c5}

如果符合下列三個條件，就會發生以興趣為根據的目標定位或個人化：網站所收集的資料是(1)用來推斷使用者的興趣，(2)用於其他情境中，例如在另一個網站或應用程式（離站），以及(3)用來根據這些推斷選擇提供哪些內容或廣告。

來自多個網站的資料組合（包括站上資料和站外資料的組合，或來自多個站外來源的資料組合）稱為跨網站資料。 不同的網站代表不同的前後關聯，因此在任何前後關聯中使用跨網站資料的方式與使用原始網站資料的方式不同。 通常會收集和處理跨網站資料，以推斷使用者的興趣。 因此，使用跨網站資料來鎖定目標廣告或內容通常符合興趣型鎖定目標的資格，無論廣告或內容出現在網站上還是網站上。 例如，如果現場資料與非現場資料結合使用，以選擇在組織自己的網站上向使用者顯示的廣告，則該使用將符合興趣型鎖定目標的資格。 再舉一個例子，在網站外將廣告重新目標定位給使用者，也可能符合興趣型目標定位的資格。

單獨使用離站資料進行目標定位也很可能符合以興趣為基礎的定位資格，因為離站資料通常是收集和處理的，以推斷使用者的興趣。

不過，僅使用站上資料的目標定位內容或廣告通常不符合興趣型目標定位的條件。 網站上的目標定位若不符合興趣型目標定位的條件，則會視為兩個不同的標籤。 具體而言，標籤C6針對站上廣告目標定位和報告，並特別說明廣告選擇、傳送和報告，而標籤C7針對站上內容選擇、傳送和報告（站上內容目標定位）。

最終，標籤的詮釋及使用該標籤之資料的使用方式取決於您。 以下提供IAB和DAA架構以供參考：

iab：個人化。 收集和處理關於您使用此服務的資訊，以隨後隨著時間在其他內容中（例如其他網站或應用程式上）為您個人化廣告和/或內容。 通常，網站或應用程式的內容會用於推斷您的興趣，以利日後選取廣告和/或內容。

DATA：線上行為廣告。 從特定電腦或裝置收集有關一段時間及跨非附屬網站網頁檢視行為的資料，以利用這些資料來預測使用者偏好或興趣，以根據從這類網頁檢視行為推斷出的偏好或興趣，向該電腦或裝置傳送廣告。

#### C6 {#c6}

廣告是出現在網站或應用程式上的訊息或通知，包括文字和影像，主要用於促銷商品或服務。 您可以自行決定這類訊息或通知的用途。 廣告與站上內容分開，由標籤涵蓋 [C7](#c7). 具有C6標籤的資料不可用於站上廣告目標定位，包括在您組織的網站或應用程式上選擇和投放廣告，或衡量此類廣告的投放和有效性。 這包括使用先前收集的使用者興趣相關網站資料來選取廣告、處理顯示哪些廣告、何時在何處顯示，以及使用者是否採取任何與廣告相關的動作，例如選取廣告或購買。 一般而言，根據使用者的網站活動來推斷使用者的偏好設定，然後在網站上的廣告目標定位中使用這些偏好設定，將不符合以興趣為基礎的目標定位（也稱為個人化）的資格，因為它不符合以興趣為基礎的目標定位所需的全部三個要求。 *[如需這些需求，請參閱標籤C5。](#c5)*

最終，標籤的詮釋及使用該標籤之資料的使用方式取決於您。 以下提供IAB和DAA架構以供參考：

IAB： 3. 廣告選擇、傳送、報告：收集資訊，並與先前收集的資訊相結合，以選擇並傳送廣告給您，並評估此類廣告的傳送和有效性。 這包括使用先前收集的有關您興趣的資訊來選取廣告、處理有關顯示哪些廣告、顯示頻率、顯示時間和顯示位置以及您是否採取任何與廣告相關的動作的資料，例如選取廣告或進行購買。 這不包括個人化，個人化是收集和處理關於您使用此服務的資訊，隨後隨著時間而在其他內容（例如網站或應用程式）中為您個人化廣告和/或內容。

DAA：線上行為廣告不包含第一方活動、廣告交付或廣告報告，或情境式廣告（即根據被瀏覽網頁內容、消費者目前瀏覽網頁或搜尋查詢的廣告）。

#### C7 {#c7}

網站內容是設計用來提供資訊、教育或娛樂的文字和影像，並非為了促銷商品或服務而建立。 您可以自行決定內容的用途，包括內容是否符合原生廣告的資格。 C7標籤的用途並非涵蓋標籤涵蓋的站上廣告 [C6](#c6). 具有C7標籤的資料無法用於網站上的內容目標定位，包括在您組織的網站或應用程式上選擇和傳遞內容，或測量此類內容的傳遞和有效性。 這包括先前收集的使用者對選取內容之興趣的相關資訊、處理顯示的內容、顯示頻率或時間長度、顯示時間和顯示位置，以及使用者是否採取與內容相關的任何動作，例如選取內容。 一般而言，根據使用者的網站活動來推斷使用者的偏好設定，然後在網站上的內容目標定位中使用這些偏好設定，將不符合興趣型目標定位（也稱為個人化）的資格，因為它不符合興趣型目標定位所需的全部三項需求。 *[如需這些需求，請參閱標籤C5。](#c5)*

最終，標籤的詮釋及使用該標籤之資料的使用方式取決於您。 以下提供IAB和DAA架構以供參考：

IAB： 4. 內容選擇、傳送、報告：資訊的集合，以及先前收集的資訊組合，可讓您選取及傳送內容，並評估此類內容的傳送及有效性。 這包括使用先前收集的有關您興趣的資訊來選取內容、處理有關顯示什麼內容、顯示頻率或時間長度、顯示時間和位置，以及您是否採取與內容相關的任何動作的資料，例如選取內容。 這不包括個人化，個人化是指收集和處理關於您使用此服務的資訊，隨後隨著時間而在其他內容（例如網站或應用程式）中為您個人化內容和/或廣告。

DAA：線上行為廣告不包含第一方活動、廣告交付或廣告報告，或情境式廣告（即根據正在瀏覽的網頁內容、消費者目前造訪的網頁或搜尋查詢的廣告）。

#### C8 {#c8}

資料無法用於測量、瞭解及報告使用者對您組織的網站或應用程式的使用情況。 這不包括以興趣為根據的目標定位（跨網站目標定位），這是您使用此服務之後在其他內容中（例如在網站或應用程式上）個人化內容和/或廣告的資訊集合。

#### C9 {#c9}

有些合約包含明確禁令，禁止將資料用於資料科學。 有時候，這些措辭會禁止將資料用於人工智慧(AI)、機器學習(ML)或模型。

#### C10 {#c10}

有些資料治理原則會限制將拼接的身分資料用於個人化。 如果區段的合併原則使用「私密圖表」選項，C10標籤會自動套用至區段。

#### C11 {#c11}

Adobe Experience Platform區段比對可讓您將第一方區段與隱私權和同意偏好設定進行比對，有助於進行更豐富的設定和下游分析。 C11標籤表示不應在下列位置使用的資料： [!DNL Segment Match] 程式。 在您確定要從「區段比對」排除哪些資料集和/或欄位，並相應地新增C11標籤後，「區段比對」工作流程會自動強制執行標籤。

#### C12 {#c12}

無法以任何方式從Platform匯出具有此標籤的資料。 CSV下載、API使用量和啟用工作流程中會排除C12標籤的欄位。
