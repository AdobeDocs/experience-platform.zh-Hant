# 資料使用標籤辭彙表

資料使用標籤可讓您根據套用至該資料的使用原則來分類資料集和欄位。 Adobe Experience Platform資料管理提供數種現成可用的核心資料使用標籤，讓您用來開始分類資料。

本檔案概述[!DNL Experience Platform]目前提供的核心資料使用標籤。 有關[!DNL Data Governance]的更多資訊，請參閱[資料治理概述](../home.md)。

## 合約標籤

合約「C」標籤可用來分類具有合約義務或與貴組織的資料管理政策相關的資料。

| 標籤 | 定義 |
|---|---|
| **C1** | 資料只能匯出自Adobe Experience Cloud的匯整形式，而不含個別或裝置識別碼。 [更多資訊...](#c1) |
| **C2** | 資料無法匯出至協力廠商。 [更多資訊...](#c2) |
| **C3** | 資料無法結合或與直接可識別的資訊搭配使用。 [更多資訊...](#c3) |
| **C4** | 資料無法用於定位任何廣告或內容，不論是在網站上或跨網站。 [更多資訊...](#c4) |
| **C5** | 資料無法用於內容或廣告的喜好式跨網站定位。 [更多資訊...](#c5) |
| **C6** | 資料無法用於站上廣告定位。 [更多資訊...](#c6) |
| **C7** | 資料無法用於內容的站上定位。 [更多資訊...](#c7) |
| **C8** | 資料無法用於測量貴組織的網站或應用程式。 [更多資訊...](#c8) |
| **C9** | 資料無法用於Data Science工作流程。 [更多資訊...](#c9) |
| **C10** | 資料無法用於銜接身分啟動。 [更多資訊...](#c10) |

## 身分標籤

身分「I」標籤可用來分類可識別或聯絡特定人員的資料。

| 標籤 | 定義 |
|---|---|
| **I1** | 可直接識別或聯絡特定人員（而非裝置）的可識別資料。 |
| **I2** | 可與任何其他資料結合使用的間接可識別資料，以識別或聯絡特定人員。 |

## 敏感標籤

敏感的「S」標籤可用來將您和您的組織認為敏感的資料分類。

一種您可能認為敏感的資料類型可能是不同的地理資料類型；不過，此類別不限於地理資料。

| 標籤 | 定義 |
|---|---|
| **S1** | 指定緯度和經度的資料，可用來判斷裝置的精確位置。 |
| **S2** | 可用來判斷廣泛定義之地理範圍的資料。 |

## 附錄

以下各節提供有關可用資料使用標籤的其他資訊。

### 合約標籤詳細資訊

以下各節提供與具體合同&quot;C&quot;標籤的實施有關的詳細資訊。

#### C1 {#c1}

有些資料只能匯出自Adobe Experience Cloud，匯總格式不含個別或裝置識別碼。 例如，源自社交網路的資料。

#### C2 {#c2}

有些資料提供者在其合約中有條款禁止從原始收集位置匯出資料。 例如，社交網路合約通常會限制您從其接收到的資料傳輸。 C2標籤的限制比[C1](#c1)更嚴格，後者只需要聚合和匿名資料。

#### C3 {#c3}

有些資料提供者在其合約中有條款禁止將資料與直接可識別資訊合併或使用。 例如，源自廣告網路、廣告伺服器和第三方資料供應商的資料合約通常包含針對使用這類資料與直接可識別資料的特定合約禁止。

#### C4 {#c4}

C4是最嚴格的標籤——它包含[C5](#c5)、[C6](#c6)和[C7](#c7)標籤。

#### C5 {#c5}

符合下列三個條件時，就會發生喜好式定位或個人化：現場收集的資料用於(1)對用戶的興趣做出推論，(2)用於其他上下文，例如在其他站點或應用程式（非現場），以及(3)用於根據這些推論選擇所提供的內容或廣告。

數個網站的資料組合（包括現場資料與非現場資料的組合，或數個非現場來源的資料組合）稱為跨網站資料。 不同的網站代表不同的上下文，因此在任何上下文中跨網站資料的使用都與原始內容不同。 跨網站資料通常會收集並處理，以做出使用者興趣的推論。 因此，使用跨網站資料來定位廣告或內容通常符合喜好式定位的資格，不論廣告或內容是在網站上或網站外顯示。 例如，如果將現場資料與非現場資料結合使用，以選擇在組織本身網站上顯示使用者的廣告，則使用該廣告符合喜好定位的資格。 另一個例子是，將廣告重新定位到網站外的使用者，也可能符合喜好定位的資格。

僅使用網站外資料進行定位也可能符合喜好式定位，因為網站外資料通常會收集並處理，以做出使用者興趣的推論。

不過，僅使用現場資料來定位內容或廣告通常不符合喜好定位的資格。 不符合興趣型定位資格的網站定位，會視為兩個不同的標籤。 具體來說，標籤C6可解決現場廣告定位與報告，並特別涉及廣告選擇、傳送與報告，標籤C7可解決現場內容選擇、傳送與報告（現場內容定位）。

最終，標籤的解釋以及如何使用帶有該標籤的資料取決於您。 為參考，IAB和DAA框架如下：

IAB:個人化。 收集和處理您使用本服務的相關資訊，以便隨著時間推移，在其他情境（例如在其他網站或應用程式）中為您個人化廣告和／或內容。 通常，網站或應用程式的內容會用來推斷您的興趣，進而為未來選擇的廣告和／或內容提供資訊。

DAA:線上行為廣告。 從特定電腦或裝置收集有關網路檢視行為的資料，並跨非附屬網站跨時間收集資料，以利用這些資料來預測使用者偏好或興趣，根據此類網路檢視行為推斷的喜好或興趣，向該電腦或裝置提供廣告。

#### C6 {#c6}

廣告是出現在網站或應用程式上的訊息或通知，包括文字和影像，主要用於促銷商品或服務的銷售。 此類訊息或通知的目的由您決定。 廣告與現場內容不同，由[C7](#c7)標籤覆蓋。 具有C6標籤的資料無法用於網站廣告定位，包括在貴組織的網站或應用程式上選擇和發佈廣告，或用於評估此類廣告的發佈和效果。 這包括使用先前收集到的使用者興趣的現場資料來選擇廣告、處理顯示廣告的時間和位置資料，以及使用者是否採取與廣告相關的任何動作，例如選擇廣告或購買。 通常，根據使用者的網站活動來推論使用者的喜好，然後在網站廣告定位中使用這些偏好，並不符合喜好式定位（也稱為個人化），因為這不符合喜好式定位的所有三項必要要求。 *[請參閱標籤C5以瞭解這些需求。](#c5)*

最終，標籤的解釋以及如何使用帶有該標籤的資料取決於您。 為參考，IAB和DAA框架如下：

IAB:3. 廣告選擇、傳送、報告：收集資訊，並結合先前收集的資訊，以選擇和提供廣告，並評估此類廣告的傳遞效果。 這包括使用先前收集到的有關您興趣的資訊來選擇廣告、處理有關顯示廣告的資料、顯示頻率、顯示時間和位置，以及您是否採取與廣告相關的任何動作，包括選擇廣告或購買。 這不包括個人化，個人化是指收集和處理您使用本服務的相關資訊，以便隨著時間推移，在其他情境（例如網站或應用程式）中為您個人化廣告和／或內容。

DAA:線上行為廣告不包括「第一方」、「廣告傳送」或「廣告報告」或情境式廣告（亦即基於所瀏覽網頁內容的廣告、消費者目前瀏覽網頁或搜尋查詢）的活動。

#### C7 {#c7}

現場內容是文字和影像，旨在提供資訊、教育或娛樂，並非為促銷商品或服務的銷售而建立。 內容的目的由您決定，包括內容是否符合原生廣告的資格。 C7標籤不用於覆蓋由標籤[C6](#c6)覆蓋的現場廣告。 具有C7標籤的資料無法用於網站內容定位，包括選擇和傳送組織網站或應用程式上的內容，或用來評估此類內容的傳送和效果。 這包括先前收集到的使用者對選取內容的興趣、處理顯示內容的頻率、顯示時間、位置、使用者是否採取與內容相關的任何動作（例如選取內容）的資料。 通常，根據使用者的網站活動來推論使用者的喜好，然後在網站內容定位中使用這些偏好，並不符合喜好式定位（又稱為個人化），因為這不符合喜好式定位的所有三項必要要求。 *[請參閱標籤C5以瞭解這些需求。](#c5)*

最終，標籤的解釋以及如何使用帶有該標籤的資料取決於您。 為參考，IAB和DAA框架如下：

IAB:4. 內容選擇、傳送、報告：收集資訊，並結合先前收集的資訊，以選擇並提供內容，並評估此類內容的傳遞與效能。 這包括使用先前收集到的有關您興趣的資訊來選取內容、處理有關內容的顯示頻率、顯示時間、顯示時間、位置的資料，以及您是否採取與內容相關的任何動作，包括例如選取內容。 這不包括個人化，個人化是指收集和處理您使用本服務的相關資訊，以便隨時在其他情境（例如網站或應用程式）中為您個人化內容和／或廣告。

DAA:線上行為廣告不包括「第一方」、「廣告傳送」或「廣告報告」或情境式廣告（亦即，以正在瀏覽之網頁內容為基礎的廣告、消費者目前對網頁的瀏覽或搜尋查詢）的活動。

#### C8 {#c8}

資料無法用來測量、瞭解和報告使用者對您組織網站或應用程式的使用情形。 這不包括喜好式定位（跨網站定位），這是您使用本服務後來針對其他情境（例如網站或應用程式）個人化內容和／或廣告的資訊收集，以供您隨時間推移使用。

#### C9 {#c9}

有些合約明確禁止資料科學的資料使用。 有時，這些措辭的措辭會禁止為人工智慧(AI)、機器學習(ML)或模型使用資料。

#### C10 {#c10}

某些資料使用政策會限制使用銜接身分資料進行個人化。 如果C10標籤的合併策略使用「專用圖表」選項，則會自動套用至區段。
