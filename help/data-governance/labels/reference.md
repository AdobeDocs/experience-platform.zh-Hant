# 資料使用標籤辭彙表 {#data-usage-labels-glossary}

>[!CONTEXTUALHELP]
>id="platform_policies_labeltype"
>title="標籤類型"
>abstract="資料使用標籤有多種類別。Adobe 定義的標籤包括合約標籤、身分識別標籤和敏感標籤。您的組織定義的標籤被歸類為自訂標籤。"
>text="See the data usage labels glossary for more information on these label types."

資料使用標籤允許您根據 [治理政策](../policies/overview.md) 和 [訪問控制策略](../../access-control/abac/overview.md) 適用於那些資料。 Adobe Experience Platform提供了幾個現成的核心資料使用標籤，您可以使用這些標籤開始對資料進行分類。

本文檔概述了當前由Experience Platform提供的核心資料使用標籤。

## 合同標籤

合同「C」標籤用於對具有合同義務或與貴組織的資料治理策略相關的資料進行分類。

| 標籤 | 定義 |
| --- | --- |
| [C1](#c1) | 資料只能以聚合形式從Adobe Experience Cloud導出，而不包括單個或設備標識符。 |
| [C2](#c2) | 資料無法導出到第三方。 |
| [C3](#c3) | 資料不能與直接可識別的資訊組合或以其他方式使用。 |
| [C4](#c4) | 資料不能用於針對任何廣告或內容，無論是現場廣告還是跨站點廣告。 |
| [C5](#c5) | 資料不能用於基於興趣的內容或廣告的跨站點目標。 |
| [C6](#c6) | 資料不能用於現場廣告目標。 |
| [C7](#c7) | 資料不能用於內容的現場目標。 |
| [C8](#c8) | 資料不能用於測量您組織的網站或應用。 |
| [C9](#c9) | 資料不能在資料科學工作流中使用。 |
| [C10](#c10) | 資料不能用於縫合標識激活。 |
| [C11](#c11) | 無法與段匹配夥伴共用資料。 |
| [C12](#c12) | 無法以任何方式導出資料。 |

## 標識標籤

標識「I」標籤用於對可識別或聯繫特定人的資料進行分類。

| 標籤 | 定義 |
| --- | --- |
| **I1** | 可直接識別的資料可以識別或聯繫特定人，而不是設備。 |
| **I2** | 可與任何其他資料組合使用的間接可識別資料，以識別或聯繫特定人。 |

## 敏感標籤 {#sensitive}

敏感「S」標籤用於對您和您的組織認為敏感的資料進行分類。

您可能認為敏感的一種資料類型可能是不同類型的地理資料；但是，此類別不限於地理資料。

| 標籤 | 定義 |
| --- | --- |
| **S1** | 指定可用於確定設備精確位置的緯度和經度的資料。 |
| **S2** | 可用於確定廣義地質區的資料。 |
| **PSPD** | 允許的敏感個人資料(PSPD)是指Adobe約定允許您上傳的被視為「敏感」、「特殊類別的資料」或適用法律使用的類似術語的資料。 這特別不包括受保護的健康資訊(PHI)和其他受管理的健康資料。 |
| **RHD** | 指受保護健康資訊(PHI)或Adobe約定允許上傳的有關您的患者的資訊的資料。 |

## 附錄

以下各節提供了有關可用資料使用標籤的其他資訊。

### 合同標籤詳細資訊

以下各節提供了與具體合同&quot;C&quot;標籤的實施有關的詳細資訊。

#### C1 {#c1}

某些資料只能以聚合形式從Adobe Experience Cloud導出，而不包括單個或設備標識符。 例如，源自社交網路的資料。

#### C2 {#c2}

一些資料提供者在其合同中規定了禁止從最初收集的地方導出資料的條款。 例如，社交網路合同通常會限制您從這些合同接收到的資料的傳輸。 C2標籤的限制性比 [C1](#c1)，它只需要聚合和匿名資料，但限制性比 [C12](#c12)，它完全阻止資料導出，而不考慮目標。

#### C3 {#c3}

一些資料提供商在合同中有條款，禁止將資料與直接可識別資訊合併或使用。 例如，從廣告網路、廣告伺服器和第三方資料提供商獲取的資料合同通常包括禁止使用直接可識別資料的此類資料的具體合同。

#### C4 {#c4}

C4包含標籤 [C5](#c5)。 [C6](#c6), [C7](#c7)。 它是限制性最強的標籤之一，僅次於 [C12](#c12)。

#### C5 {#c5}

如果滿足以下三個條件，即基於興趣的目標或個性化：現場收集的資料(1)用於對用戶興趣做出推論，(2)用於另一個上下文，例如在另一個站點或應用（非現場）上，並且(3)用於根據這些推論選擇服務哪些內容或廣告。

來自多個站點的資料的組合，包括現場資料和非現場資料的組合或來自多個非現場源的資料的組合，稱為跨站點資料。 不同的站點代表不同的上下文，因此在任何上下文中跨站點資料的使用都不同於原始上下文。 通常收集和處理跨站點資料以推斷用戶的興趣。 因此，使用跨站點資料來定位廣告或內容通常符合基於興趣的目標，而不管廣告或內容是在現場還是場外顯示。 例如，如果將現場資料與非現場資料結合使用，以選擇在組織自己的站點上顯示用戶的廣告，則使用該廣告將符合基於興趣的目標。 另一個例子是，將廣告重新定向到非現場用戶也可能被認定為基於興趣的目標。

僅將非現場資料用於目標可能也符合基於興趣的目標，因為非現場資料通常被收集和處理以推斷用戶的興趣。

但是，僅使用現場資料定位內容或廣告通常不符合基於興趣的目標。 不屬於基於興趣的目標的現場目標被視為兩個不同的標籤。 具體來說，標籤C6針對現場廣告目標和報告，並具體指廣告選擇、交付和報告，標籤C7針對現場內容選擇、交付和報告（現場內容目標）。

最終，標籤的解釋以及如何強制使用帶有該標籤的資料取決於您。 為參考，IAB和DAA框架如下：

IAB:個性化。 收集和處理有關您使用此服務的資訊，以便隨時間在其它上下文（如其他網站或應用）中對您的廣告和/或內容進行個性化設定。 通常，站點或應用的內容用於推斷您的興趣，這些興趣為將來選擇廣告和/或內容提供資訊。

資料：線上行為廣告。 從特定電腦或設備收集關於隨時間和跨非關聯網站的網頁查看行為的資料，以便使用這些資料來預測用戶偏好或興趣，以基於從這種網頁查看行為推斷的偏好或興趣向該電腦或設備發送廣告。

#### C6 {#c6}

廣告是主要旨在促進商品或服務銷售的網站或應用上出現的消息或通知，包括文本和影像。 由您確定此類消息或通知的用途。 廣告與現場內容分開，由標籤覆蓋 [C7](#c7)。 帶有C6標籤的資料不能用於現場廣告目標，包括在您組織的網站或應用上選擇和傳遞廣告，或衡量此類廣告的傳遞和效果。 這包括使用先前收集的關於用戶興趣的現場資料來選擇廣告、處理關於顯示哪些廣告的資料、顯示時間和地點的資料以及用戶是否採取了與廣告相關的任何行動，如選擇廣告或購買。 通常，根據用戶的現場活動推斷用戶的偏好，然後在現場廣告目標中使用這些偏好並不符合基於興趣的目標（也稱為個性化），因為它不能滿足基於興趣的目標所需的全部三項要求。 *[有關這些要求，請參見標籤C5。](#c5)*

最終，標籤的解釋以及如何強制使用帶有該標籤的資料取決於您。 為參考，IAB和DAA框架如下：

IAB:3. 廣告選擇、交付、報告：資訊的收集，以及與先前收集的資訊的組合，用於為您選擇和傳送廣告，並衡量此類廣告的傳送和效果。 這包括使用先前收集的有關您的興趣的資訊來選擇廣告、處理有關顯示哪些廣告、顯示頻率、顯示時間和位置的資料，以及您是否採取了與廣告相關的任何操作，包括選擇廣告或購買廣告。 這不包括個性化，即收集和處理有關您使用此服務的資訊，以隨時間在其他上下文（如網站或應用）中對您的廣告和/或內容進行個性化。

資料：線上行為廣告不包括第一方、廣告傳遞或廣告報告或上下文廣告的活動（即基於所訪問網頁的內容的廣告、消費者當前對網頁的訪問或搜索查詢）。

#### C7 {#c7}

現場內容是文本和圖片，旨在通知、教育或娛樂，而不是為促進商品或服務的銷售而建立。 由您決定內容的用途，包括內容是否符合本機廣告的要求。 C7標籤不用於覆蓋由標籤覆蓋的現場廣告 [C6](#c6)。 帶有C7標籤的資料不能用於現場內容目標，包括選擇和傳遞您組織網站或應用上的內容，或衡量此類內容的傳遞和有效性。 這包括先前收集的有關用戶對選定內容的興趣的資訊、處理有關顯示內容的資料、顯示時間、顯示時間、位置以及使用者是否採取了與內容相關的任何操作（例如，選擇內容）的資料。 通常，根據用戶的現場活動推斷用戶的偏好，然後在現場內容目標中使用這些偏好將不符合基於興趣的目標（也稱為個性化），因為它不能滿足基於興趣的目標所需的全部三項要求。 *[有關這些要求，請參見標籤C5。](#c5)*

最終，標籤的解釋以及如何強制使用帶有該標籤的資料取決於您。 為參考，IAB和DAA框架如下：

IAB:4. 內容選擇、交付、報告：資訊的收集以及與先前收集的資訊的組合，用於選擇和交付內容，並衡量此類內容的交付和有效性。 這包括使用先前收集的有關您的興趣的資訊來選擇內容、處理有關顯示內容的資料、顯示內容的頻率或時間、顯示時間和位置以及您是否採取了與內容相關的任何操作（例如選擇內容）。 這不包括個性化，即收集和處理有關您使用此服務以隨時間在其他上下文（如網站或應用）中個性化內容和/或廣告的資訊。

資料：線上行為廣告不包括第一方、廣告傳遞或廣告報告或上下文廣告的活動（即基於正在訪問的網頁內容的廣告、消費者當前對網頁的訪問或搜索查詢）。

#### C8 {#c8}

資料不能用於衡量、瞭解和報告用戶對您組織的站點或應用的使用情況。 這不包括基於興趣的目標（跨站點目標），即收集有關您使用此服務以隨後在其他上下文（例如網站或應用）中對您的內容和/或廣告進行個性化的資訊。

#### C9 {#c9}

一些合同明確禁止資料科學的資料使用。 有時，這些措辭是禁止使用人工智慧(AI)、機器學習(ML)或建模的資料。

#### C10 {#c10}

某些資料治理策略限制了使用縫合身份資料進行個性化。 如果C10標籤的合併策略使用「專用圖」選項，則該標籤將自動應用於段。

#### C11 {#c11}

Adobe Experience Platform網段匹配允許您將第一方網段與隱私和同意首選項進行匹配，從而有助於豐富的分析和下游洞察力。 C11標籤表示不應用於 [!DNL Segment Match] 進程。 在確定要從段匹配中排除的資料集和/或欄位並相應添加C11標籤後，該標籤將由段匹配工作流自動強制執行。

#### C12 {#c12}

無法以任何方式從平台導出帶有此標籤的資料。 C12標籤的欄位不包括在CSV下載、API消耗和激活工作流中。
